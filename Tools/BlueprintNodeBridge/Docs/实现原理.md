## 1. T3D 模板缓存系统

### 核心问题

**Q: 复制短代码时是否生成 T3D？在哪里？**

### 设计思路

复制节点时需要解决两个问题：
1. **显示用短代码** - 用于 AI 理解的可读文本
2. **粘贴用模板** - 用于重建节点的完整数据

### 实现原理

```
复制节点流程：
┌─────────────────────────────────────────────────────────┐
│  用户选中节点 → 点击"复制为短代码"                        │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 1: 调用 FEdGraphUtilities::ExportNodesToText()    │
│          生成 T3D 格式字符串                             │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 2: T3D 存储到内存中的 NodeTemplateCache             │
│          格式: TMap<FString, FString>                    │
│          Key: "NodeType_Name" 或 "__CompleteSet__"      │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 3: 生成显示用短代码（可读格式）                      │
│          例: "Branch_1 (inExec;) : (outExec True;)"    │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 4: 短代码复制到剪贴板                              │
│          T3D 保留在内存中                                │
└─────────────────────────────────────────────────────────┘
```

### T3D 存储位置

| 层级 | 位置 | 说明 |
|------|------|------|
| **内存** | `NodeTemplateCache` (TMap) | 运行时缓存，进程存活期间有效 |
| **磁盘** | `Saved/BlueprintNodeBridge/NodeTemplates.json` | 持久化存储，跨会话保留 |

### 内存 vs 磁盘

```
内存缓存 (NodeTemplateCache)
├── 生命周期: 随编辑器进程
├── 访问速度: 最快 (O(1))
├── 容量: 受内存限制
└── 内容: 完整 T3D 字符串

磁盘缓存 (NodeTemplates.json)
├── 生命周期: 永久 (除非删除)
├── 访问速度: 较慢 (需读写文件)
├── 容量: 受磁盘空间限制
└── 内容: JSON 格式 T3D 集合
```

### 缓存 Key 命名规则

```cpp
// 完整节点集 T3D（用于粘贴多个节点）
Key: "__CompleteSet__"

// 按节点类型缓存（首次遇到该类型时生成）
Key: "Branch"
Key: "PrintString"
Key: "SetMyVariable"

// 带序号的唯一标识
Key: "Branch_1"
Key: "PrintString_2"
```

### T3D 内容示例

UE 的 T3D (Text 3D) 格式是节点序列化文本：

```
Begin Map
    Begin Level
        Begin Actor Class=/Script/BlueprintGraph.K2Node_IfThenElse
            NodePosX=0
            NodePosY=0
            NodeGuid=ABC123DEF456...
            InputPins(0)=(PinName="Execution",...)
            OutputPins(0)=(PinName="Then",...)
        End Actor
    End Level
End Map
```

### 如何使用 T3D 代码

**直接访问缓存：**
```cpp
// 在任意模块代码中访问缓存
extern FBlueprintNodeBridgeModule& GetModule();
FString T3D = GetModule().NodeTemplateCache["Branch"];
```

**导出供外部使用：**
1. 使用"管理模板"菜单打开文件夹
2. 编辑 `NodeTemplates.json`
3. 重启编辑器加载修改

## 2. 短代码生成与解析

### 设计思路

短代码是一种**紧凑的文本表示格式**，用于：
- 人类可读
- AI 可理解
- 便于传输和存储

### 格式规范

```
# --- Node Definitions ---
NodeName_ID (InputPin1; InputPin2\(value\);) : (OutputPin1; OutputPin2;)
NodeName_ID (...) : (...)

# --- Links ---
Node1 (outPin) -> Node2 (inPin)
```

### 生成算法 (GenerateShortCode)

```cpp
FString FTextGraphParser::GenerateShortCode(const TArray<UEdGraphNode*>& Nodes, ...)
{
    // 1. 为每个节点生成唯一 ID
    for (int32 i = 0; i < Nodes.Num(); i++)
    {
        FString UniqueName = BaseName + "_" + FString::FromInt(i + 1);
        NodeToID.Add(Node, UniqueName);
    }

    // 2. 收集节点定义
    for (UEdGraphNode* Node : Nodes)
    {
        // 遍历引脚，生成输入/输出格式
        Inputs += "PinName\\(value\\); ";
        Outputs += "PinName; ";
    }

    // 3. 收集连接关系
    for (UEdGraphNode* Node : Nodes)
    {
        for (UEdGraphPin* Pin : Node->Pins)
        {
            for (UEdGraphPin* LinkedPin : Pin->LinkedTo)
            {
                // 生成连接行: Node1 (out) -> Node2 (in)
            }
        }
    }
}
```

### 解析算法 (ParseAndPaste)

```cpp
bool FTextGraphParser::ParseAndPaste(const FString& ShortCode, ...)
{
    // 1. 解析节点定义
    for (Line : Lines)
    {
        if (包含 "->")  // 连接行
            ParseLinkDefinition(Line);
        else            // 节点定义行
            ParseNodeDefinition(Line);
    }

    // 2. 从缓存获取 T3D 并粘贴
    for (Def : Definitions)
    {
        FString T3D = NodeCache[Def.Name];
        FEdGraphUtilities::ImportNodesFromText(Graph, T3D, ImportedNodes);
    }

    // 3. 重建连接
    for (Link : Links)
    {
        FindPin(SourceNode, Link.SourcePin);
        FindPin(TargetNode, Link.TargetPin);
        MakeLinkTo(SourcePin, TargetPin);
    }
}
```

### 转义规则

| 原始字符 | 转义后 | 原因 |
|---------|--------|------|
| `(` | `\(` | 解析时用于分离引脚名和值 |
| `)` | `\)` | 值结束标记 |
| `;` | `\;` | 引脚分隔符 |
| `\` | `\\` | 转义字符本身 |
| `\n` | `\n` | 字符串内的换行 |

## 3. 节点布局算法

### 设计思路

节点布局采用**层级式布局算法**：
- 执行流节点水平排列（层级 = rank）
- 纯函数节点放置在消费者左侧
- 自动处理循环（断开环边）

### 算法流程

```
┌─────────────────────────────────────────────────────────┐
│  Input: 选中的节点数组                                    │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 1: 构建关系图                                       │
│  - 执行流连接 → Children/Parents                         │
│  - 数据流连接 → DataConsumers                            │
│  - 纯函数判断 → 无 exec 引脚                              │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 2: 循环检测与移除                                   │
│  - DFS 遍历                                               │
│  - 遇到已在栈中的节点 → 断开该边                          │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 3: BFS 层级分配                                     │
│  - 从根节点开始（无父节点）                               │
│  - 每层 Rank + 1                                        │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  Step 4: 节点定位                                         │
│  - 执行流节点: Rank × 水平间距                           │
│  - 纯函数节点: 消费者左侧                                 │
└─────────────────────────────────────────────────────────┘
```

### 布局数据结构

```cpp
struct FNodeLayoutInfo
{
    UEdGraphNode* Node;
    int32 Rank;           // 层级 (0 = 根)
    int32 Order;          // 同层顺序
    bool bIsPureFunction; // 是否纯函数
    float X, Y;           // 最终位置

    TArray<FNodeLayoutInfo*> Parents;
    TArray<FNodeLayoutInfo*> Children;     // 执行流子节点
    TArray<FNodeLayoutInfo*> DataConsumers; // 数据流消费者
};
```

### 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `HorizontalSpacing` | 200.0 | 节点间水平间距 |
| `VerticalSpacing` | 60.0 | 节点间垂直间距 |
| `Rank` | 层级深度 | 根节点为 0 |

## 4. 菜单注册机制

### 设计思路

使用 UE5 的 **UToolMenus** 系统扩展编辑器菜单：
- 不修改引擎代码
- 支持多个编辑器类型
- 动态菜单内容

### 注册流程

```cpp
void FBlueprintNodeBridgeModule::RegisterMenus()
{
    // 支持的编辑器工具栏
    TArray<FName> EditorToolbars = {
        "AssetEditor.BlueprintEditor.ToolBar",
        "AssetEditor.WidgetBlueprintEditor.ToolBar",
        "AssetEditor.AnimationBlueprintEditor.ToolBar",
        "AssetEditor.MaterialEditor.ToolBar",
        "AssetEditor.NiagaraScriptEditor.ToolBar"
    };

    for (const FName& ToolbarName : EditorToolbars)
    {
        // 1. 获取或创建扩展菜单
        UToolMenu* Menu = UToolMenus::Get()->ExtendMenu(ToolbarName);

        // 2. 添加菜单节
        FToolMenuSection& Section = Menu->FindOrAddSection("Settings");

        // 3. 添加下拉按钮
        Section.AddEntry(FToolMenuEntry::InitComboButton(
            "BlueprintNodeBridgeActions",
            FUIAction(),
            FOnGetContent::CreateLambda([this]() {
                FMenuBuilder MenuBuilder(true, PluginCommands);
                // 添加各菜单项...
                return MenuBuilder.MakeWidget();
            }),
            "AI Bridge",
            "AI Bridge - Node Tools"
        ));
    }
}
```

### 工具栏名称对应

| 编辑器 | 工具栏注册名 | 实际路径 |
|--------|-------------|----------|
| Blueprint | `AssetEditor.BlueprintEditor.ToolBar` | 主工具栏 |
| Widget Blueprint | `AssetEditor.WidgetBlueprintEditor.ToolBar` | 控件蓝图工具栏 |
| Animation Blueprint | `AssetEditor.AnimationBlueprintEditor.ToolBar` | 动画蓝图工具栏 |
| Material Editor | `AssetEditor.MaterialEditor.ToolBar` | 材质编辑器工具栏 |
| Niagara | `AssetEditor.NiagaraScriptEditor.ToolBar` | Niagara 工具栏 |

### 命令绑定

## 5. 编辑器焦点检测

### 设计思路

需要判断用户当前是否在蓝图编辑器中操作，以启用/禁用菜单项。

### 实现方案

```cpp
bool FBlueprintNodeBridgeModule::IsBlueprintEditorFocused() const
{
    // 1. 获取当前激活窗口
    TSharedPtr<SWindow> ActiveWindow = FSlateApplication::Get().GetActiveTopLevelWindow();

    // 2. 检查窗口标题
    FString WindowTitle = ActiveWindow->GetTitle().ToString();
    return WindowTitle.Contains(TEXT("Blueprint")) ||
           WindowTitle.Contains(TEXT("Kismet"));
}
```

### 获取选中节点

```cpp
TArray<UEdGraphNode*> FBlueprintNodeBridgeModule::GetSelectedNodes() const
{
    // 策略 1: 从 GraphEditor 获取
    TSharedPtr<SGraphEditor> GraphEditor = GetFocusedGraphEditor();
    if (GraphEditor.IsValid())
    {
        FGraphPanelSelectionSet SelectedSet = GraphEditor->GetSelectedNodes();
        for (UObject* Obj : SelectedSet)
        {
            if (UEdGraphNode* Node = Cast<UEdGraphNode>(Obj))
                SelectedNodes.Add(Node);
        }
    }

    // 策略 2: 回退到全局选择
    if (SelectedNodes.Num() == 0 && GEditor)
    {
        USelection* Selection = GEditor->GetSelectedObjects();
        // ...
    }

    return SelectedNodes;
}
```

## 6. 缓存持久化

### 设计思路

使用 JSON 格式将内存中的缓存持久化到磁盘，下次启动时自动加载。

### 生命周期管理

```
┌─────────────────────────────────────────────────────────┐
│  StartupModule()                                         │
│      ├─ LoadCache()  ← 读取磁盘 JSON → 内存              │
│      ├─ RunAllTests()                                    │
│      └─ 注册菜单、命令                                    │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  运行时                                                   │
│      ├─ CopyAsShortCodeClicked()                         │
│      │     └─ 更新 NodeTemplateCache (内存)              │
│      │           └─ SaveCache() (异步)                   │
│      │                                                     │
│      └─ PasteShortCodeClicked()                          │
│            └─ 读取 NodeTemplateCache (内存)               │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  ShutdownModule()                                        │
│      └─ SaveCache()  ← 内存 → 磁盘 JSON                  │
└─────────────────────────────────────────────────────────┘
```

### JSON 格式

```json
{
    "Branch": "Begin Map\n    Begin Actor ...",
    "PrintString": "Begin Map\n    Begin Actor ...",
    "GetPlayerCharacter_1": "Begin Map\n    Begin Actor ...",
    "__CompleteSet__": "Begin Map\n    ..."
}
```

### 性能优化

- **启动时加载**: 首次使用前才加载
- **延迟保存**: 不每次修改都写盘，仅在关键操作后保存
- **仅保存修改**: 使用增量更新（当前实现为全量覆盖）
