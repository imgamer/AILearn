## 核心类

### FBlueprintNodeBridgeModule

主模块类，负责插件生命周期和菜单注册。

**主要方法：**

| 方法 | 功能 |
|------|------|
| `StartupModule()` | 初始化样式、注册命令、加载缓存、运行测试 |
| `ShutdownModule()` | 保存缓存、注销菜单、清理资源 |
| `RegisterMenus()` | 为多种编辑器注册工具栏菜单 |
| `GetSelectedNodes()` | 获取当前选中的节点列表 |
| `GetFocusedGraphEditor()` | 获取当前焦点的 GraphEditor |

**菜单处理：**

```cpp
void PasteShortCodeClicked();           // 解析短代码并粘贴节点
void CopyAsShortCodeClicked();          // 序列化选中节点为短代码
void FormatNodesClicked();              // 调用布局算法整理节点
void CopyAIRulesClicked();              // 复制 AI 规则到剪贴板
void ManageTemplatesClicked();          // 打开模板缓存文件夹
void EditAIRulesClicked();              // 用外部编辑器打开规则文件
```

### FTextGraphParser

节点序列化与反序列化。

**静态方法：**

| 方法 | 功能 |
|------|------|
| `GenerateShortCode()` | 将节点数组转换为短代码字符串 |
| `ParseAndPaste()` | 解析短代码并在图表中创建节点 |
| `ParseNodeDefinition()` | 解析单个节点定义 |
| `ParseLinkDefinition()` | 解析连接定义 |
| `IsValidShortCode()` | 验证短代码格式 |
| `EscapePinValue()` / `UnescapePinValue()` | Pin 值转义/反转义 |

### FNodeGraphFormatter

节点布局算法实现。

**主要算法：**

1. **构建节点关系图** - 区分执行流连接和数据流连接
2. **循环检测与移除** - 防止死循环布局
3. **层级分配 (BFS)** - 从根节点按深度分配 rank
4. **执行流节点布局** - 同层节点水平排列
5. **纯函数节点布局** - 放置在数据消费者左侧

## 菜单注册机制

使用 UE5 ToolMenus 系统注册菜单扩展：

```cpp
void FBlueprintNodeBridgeModule::RegisterMenus()
{
    TArray<FName> EditorToolbars = {
        TEXT("AssetEditor.BlueprintEditor.ToolBar"),
        TEXT("AssetEditor.WidgetBlueprintEditor.ToolBar"),
        TEXT("AssetEditor.AnimationBlueprintEditor.ToolBar"),
        TEXT("AssetEditor.MaterialEditor.ToolBar"),
        TEXT("AssetEditor.NiagaraScriptEditor.ToolBar")
    };
    
    for (const FName& ToolbarName : EditorToolbars)
    {
        UToolMenu* ToolbarMenu = UToolMenus::Get()->ExtendMenu(ToolbarName);
        // ... 添加菜单项
    }
}
```

## 节点序列化格式

### 短代码格式

```
NodeName_UniqueID (InputPin1; InputPin2\;value\;); : (OutputPin1; OutputPin2;)
```

### 数据结构

```cpp
struct FNodeDefinition
{
    FString Name;                    // 节点类型名
    FString ID;                      // 唯一标识符
    TMap<FString, FString> InputPins; // 输入引脚: 名称 -> 值
    TArray<FString> OutputPins;      // 输出引脚列表
};

struct FLinkDefinition
{
    FString SourceNodeID;    // 源节点 ID
    FString SourcePinName;   // 源引脚名
    FString TargetNodeID;    // 目标节点 ID
    FString TargetPinName;   // 目标引脚名
};
```

## 模板缓存系统

节点模板以 T3D 格式缓存在 JSON 文件中：

```cpp
// 缓存结构
TMap<FString, FString> NodeTemplateCache;  // NodeType -> T3D Content

// 保存位置
Saved/BlueprintNodeBridge/NodeTemplates.json
```

**工作流程：**
1. 复制节点时，系统查询缓存是否已有该类型模板
2. 如无模板，从节点生成 T3D 内容并缓存
3. 粘贴时使用缓存的 T3D 内容重建节点

## 扩展指南

### 添加新菜单项

1. 在 `BlueprintNodeBridgeCommands.h` 添加命令声明
2. 在 `RegisterCommands()` 实现中注册命令
3. 在 `RegisterMenus()` 添加菜单项
4. 实现处理函数

### 添加新的编辑器支持

在 `RegisterMenus()` 的 `EditorToolbars` 数组中添加新的工具栏名称：

```cpp
TEXT("AssetEditor.YourEditor.ToolBar")
```

### 修改布局算法

`NodeGraphFormatter.cpp` 中的 `FormatNodes()` 方法实现了主要的布局逻辑：
- `FNodeLayoutInfo` 结构存储每个节点的布局信息
- 通过修改 `HorizontalSpacing` 和 `VerticalSpacing` 调整间距
- 纯函数节点的定位逻辑在"Place pure functions near consumers"部分
