## 1. 文档概述

### 1.1 目的

本文档详细描述 BlueprintAIBridge 插件的六个核心功能的实现设计与技术细节，为后续开发维护提供参考依据。

### 1.2 背景

BlueprintAIBridge 是一个虚幻引擎编辑器插件，旨在建立**人类用户**与**AI模型**之间的蓝图节点通信桥梁。通过这个插件：

- 用户可以将蓝图节点复制为**短代码文本**（人类和 AI 都能理解的格式）
- AI 可以根据短代码生成新的蓝图节点描述
- 用户将 AI 生成的短代码粘贴回蓝图编辑器

### 1.3 功能清单

| 序号 | 功能名称 | 功能描述 |
|------|---------|---------|
| 1 | 复制为短代码 | 将选中的蓝图节点转换为文本格式的短代码 |
| 2 | 粘贴短代码 | 将剪贴板中的短代码文本解析并生成蓝图节点 |
| 3 | 整理节点 | 自动排列选中的蓝图节点位置 |
| 4 | 复制 AI 规则 | 将 AI 生成规则复制到剪贴板 |
| 5 | 管理模板 | 打开节点模板缓存文件夹 |
| 6 | 编辑 AI 规则 | 用系统默认编辑器打开 AI 规则文件 |

#### 3.1.4 短代码文本格式

生成的短代码采用以下格式：

```text
# --- Node Definitions ---
节点名_序号 (输入引脚1(含值); 输入引脚2;) : (输出引脚1; 输出引脚2;)
节点名_序号 (输入引脚(含值);) : (输出引脚;)

# --- Links ---
序号A (输出引脚名) -> 序号B (输入引脚名)
序号A (输出引脚名) -> 序号B (输入引脚名)
```

**示例**：

```text
# --- Node Definitions ---
EventBeginPlay_1 () : (outExec;)
Branch_2 (inExec; Condition(true);) : (True; False;)
PrintString_3 (inExec; InString("Hello");) : (outExec;)
SetActorRotation_4 (Target; NewRotation(Rotator(0, 90, 0));) : (outExec;)

# --- Links ---
EventBeginPlay_1 (outExec) -> Branch_2 (inExec)
Branch_2 (True) -> PrintString_3 (inExec)
Branch_2 (False) -> SetActorRotation_4 (inExec)
```

#### 3.1.5 关键类与方法

| 类 | 方法 | 职责 |
|---|------|------|
| `FBlueprintAIBridgeModule` | `CopyShortCodeClicked()` | 命令入口，获取选中节点，触发生成 |
| `FTextGraphParser` | `GenerateShortCode()` | 核心逻辑：生成短代码文本和缓存 |
| `FTextGraphParser` | `GetNodeBaseName()` | 提取节点基础名称 |
| `FTextGraphParser` | `GetPinValue()` | 获取引脚的默认值 |
| `FTextGraphParser` | `EscapePinValue()` | 转义引脚值中的特殊字符 |
| `FBlueprintAIBridgeModule` | `SaveCache()` | 将缓存写入 JSON 文件 |

#### 3.1.6 输入输出

| 项目 | 说明 |
|------|------|
| **输入** | 用户在蓝图中选中的节点 (`TArray<UEdGraphNode*>`) |
| **处理** | 遍历节点，提取引脚信息，生成短代码文本 |
| **输出** | 系统剪贴板中的短代码文本 + 本地 T3D 模板缓存文件 |

#### 3.1.7 执行引脚别名映射

| 内部名称 | 短代码别名 | 说明 |
|---------|-----------|------|
| `execute` | `inExec` | 执行输入引脚（白色箭头左侧） |
| `then` | `outExec` | 执行输出引脚（白色箭头右侧） |

### 3.2 粘贴短代码

#### 3.2.1 功能概述

从系统剪贴板读取短代码文本，解析后根据缓存的 T3D 模板在当前图表中创建蓝图节点，并重建节点之间的连接。

#### 3.2.2 用户交互流程

```
用户操作                    系统响应
────────                    ────────
1. 在 AI 对话中获得短代码    
2. 在 UE 中点击"粘贴短代码" 
                              → 读取剪贴板文本
                              → 获取当前图表焦点
                              → 解析短代码
                              → 创建节点
                              → 重建连接
3. 看到成功通知              ← "节点粘贴成功！"
```

#### 3.2.4 节点查找策略

粘贴时按以下优先级查找 T3D 模板缓存：

```
1. NodeName_ID (如 "Branch_2")
   └── 如果找到了，直接使用

2. NodeName (如 "Branch")
   └── 如果找到了，使用模板创建

3. 都找不到
   └── 跳过该节点，显示警告日志
```

#### 3.2.5 关键类与方法

| 类 | 方法 | 职责 |
|---|------|------|
| `FBlueprintAIBridgeModule` | `PasteShortCodeClicked()` | 命令入口，读取剪贴板，获取图表 |
| `FTextGraphParser` | `ParseAndPaste()` | 核心逻辑：解析文本，创建节点，重建连接 |
| `FTextGraphParser` | `ParseNodeDefinition()` | 解析节点定义行 |
| `FTextGraphParser` | `ParseLinkDefinition()` | 解析连接定义行 |
| `FTextGraphParser` | `RegenerateT3DGUIDs()` (Lambda) | 替换 T3D 中的 GUID |
| `FBlueprintAIBridgeModule` | `LoadCache()` | 从文件加载模板缓存 |

#### 3.2.6 输入输出

| 项目 | 说明 |
|------|------|
| **输入** | 系统剪贴板中的短代码文本 |
| **处理** | 解析文本 → 查找缓存 → 替换 GUID → 导入节点 → 重建连接 |
| **输出** | 当前图表中新建的蓝图节点及其连接 |

#### 3.2.7 GUID 重新生成算法

**问题**: 直接导入 T3D 时，如果 GUID 已存在，UE 会尝试复用已有节点而非创建新节点。

**解决方案**:
```
1. 扫描 T3D 文本中所有32位连续十六进制字符
2. 对每个匹配，生成新的 FGuid::NewGuid()
3. 格式化为大写无连字符字符串 (如 "A1B2C3D4E5F678901234567890ABCDEF")
4. 原地替换原始 GUID
```

```
                      │         │ NotifyGraphChanged()  │
                      │         └───────────┬───────────┘
                      │                     │
                      ▼                     ▼
                      End                    End
```

#### 3.3.4 布局算法详解

**Step 1 - 节点分类**

| 类型 | 定义 | 布局方式 |
|------|------|---------|
| 执行流节点 | 有 PC_Exec 引脚 | 按 Rank 水平排列 |
| 纯函数节点 | 无 Exec 引脚 | 放置在消费者左侧 |

**Step 2 - 执行流节点布局**

```
Rank 0        Rank 1        Rank 2        Rank 3
┌────────┐
│ Event  │──────►┌────────┐──────►┌────────┐──────►┌────────┐
│ Begin  │        │Branch │        │ Print │        │ Set   │
└────────┘        └───────┘        └───────┘        └───────┘
                         │
                         │ Rank 1 (与 Branch 同层)
                         ▼
                   ┌────────────┐
                   │ Delay      │
                   └────────────┘
```

**Step 3 - 纯函数节点布局**

```
                    ┌────────────────┐
                    │ Get Player Ref │
                    └───────┬────────┘
                            │ (数据流)
                            ▼
┌──────────────┐     ┌──────────────┐
│ Get Health   │     │ Branch       │
└───────┬──────┘     └──────────────┘
        │
        │ (数据流)
        ▼
   ┌────────┐
   │ Print  │
   └────────┘

纯函数节点(左侧) → 消费者(右侧)
```

**Step 4 - 碰撞检测**

当纯函数节点与已有节点位置重叠时，向上偏移：
```
初始位置: PureY = ConsumerY
检测重叠 → PureY -= (Height + Spacing)
重试检测 → 直到无重叠或达到最大尝试次数(10)
```

#### 3.3.5 关键类与方法

| 类 | 方法/结构 | 职责 |
|---|----------|------|
| `FBlueprintAIBridgeModule` | `FormatNodesClicked()` | 命令入口，获取选中节点 |
| `FNodeGraphFormatter` | `FormatNodes()` | 核心布局算法入口 |
| `FNodeGraphFormatter` | `FNodeLayoutInfo` | 布局信息结构体 |
| (内部) | BFS 分层 | 按拓扑序分配层级 |
| (内部) | DFS 环检测 | 检测并移除循环引用 |
| (内部) | AABB 碰撞检测 | 检测并解决位置重叠 |

#### 3.3.6 布局参数

| 参数 | 值 | 说明 |
|------|---|------|
| `HorizontalSpacing` | 200.0px | 节点之间的水平间距 |
| `VerticalSpacing` | 60.0px | 节点之间的垂直间距 |
| `GridSize` | 16px | 位置坐标网格对齐单位 |
| `MaxCollisionAttempts` | 10次 | 碰撞检测最大重试次数 |

#### 3.4.6 AI 规则内容概述

|------|------|
| **全量输出模式** | 无论修改多么微小，必须完整输出整个蓝图代码 |

| **数据流扫描** | 再确保所有非纯函数节点的输入引脚都有数据来源 |
