# Don't Pull — 游戏策划文档

> 基于 Capcom 1991年街机《三个奇迹》(Three Wonders) 第三作《Don't Pull》
> 文档按开发依赖顺序编排，从核心框架到外围系统，程序可直接参照实现

---

## 开发顺序总览

```
第1章  游戏框架与基础系统     ← 一切的地基
第2章  玩家系统              ← 依赖第1章网格与碰撞
第3章  方块系统              ← 依赖第1章网格，与第2章玩家交互
第4章  敌人系统              ← 依赖第1章网格，与第3章方块交互
第5章  关卡系统              ← 依赖第1~4章，组装地图数据
第6章  心形拼合机制           ← 依赖第3章方块系统
第7章  道具与奖励系统         ← 依赖第3章方块、第4章敌人
第8章  玩家状态与生命系统      ← 依赖第2章玩家、第7章道具
第9章  双人模式              ← 依赖第2~8章，扩展为多人
第10章 UI与表现系统          ← 依赖所有前序系统
附录   数据表与配置参考
```

# 第1章 游戏框架与基础系统

## 1.1 网格系统（Grid System）

### 1.1.1 基本参数

| 参数 | 值 | 说明 |
|------|----|------|
| 网格类型 | 正方形网格 | 每格宽高相等 |
| 格子尺寸 | 32×32 像素（参考值，可配置） | 单个格子在世界空间中的大小 |
| 地图宽度 | 可变，典型值 13~17 列 | 含边界墙 |
| 地图高度 | 可变，典型值 11~15 行 | 含边界墙 |
| 坐标系 | 左上角为 (0,0)，X 向右递增，Y 向下递增 | 整数格子坐标 |

### 1.1.2 格子类型枚举

```
enum CellType {
    FLOOR,          // 空地——可通行
    WALL,           // 墙壁——不可通行，不可推动
    MANHOLE,        // 下水道井盖——敌人生成点，玩家可踩踏封堵
    MANHOLE_BLOCKED,// 被封堵的井盖——玩家站在上面时状态
    BORDER_PATH,    // 外周路径——滚石滚动的专用通道
}
```

### 1.1.3 格子数据结构

```
struct Cell {
    cellType: CellType
    isOccupied: Bool         // 是否被实体（方块/玩家/敌人）占据
    occupyingEntity: Entity? // 引用占据该格子的实体
}
```

### 1.1.4 网格管理器接口

```
class GridManager {
    cells: Cell[][]                    // 二维数组存储所有格子
    width: Int
    height: Int

    getCell(x: Int, y: Int): Cell
    setCell(x: Int, y: Int, type: CellType)
    isWalkable(x: Int, y: Int): Bool   // 该格是否可被实体进入
    isPushable(x: Int, y: Int): Bool   // 该格是否可被推入（空地且无实体占据）
    worldToGrid(worldPos: Vector2): Vector2Int
    gridToWorld(gridPos: Vector2Int): Vector2
}
```

## 1.2 游戏循环（Game Loop）

### 1.2.1 主循环流程

```
每帧执行:
    1. 处理输入
    2. 更新游戏逻辑（按以下子系统顺序）
       a. 玩家移动与推块
       b. 方块移动与碰撞
       c. 敌人 AI 与移动
       d. 滚石移动
       e. 道具生成与拾取
       f. 心形拼合检测
       g. 碰撞判定（玩家-敌人、方块-敌人、滚石-实体）
       h. 计时器更新
    3. 检查胜负条件
    4. 渲染
```

### 1.2.2 帧率与时间步进

| 参数 | 值 | 说明 |
|------|----|------|
| 目标帧率 | 60 FPS | 固定帧率 |
| 逻辑步进 | 1/60 秒 | 固定时间步进，避免浮点误差 |
| 移动方式 | 格子对齐移动 | 实体移动以整格为单位，移动动画在格子间插值 |

### 1.2.3 实体移动规范

- 所有实体（玩家、敌人、方块）的**逻辑位置**始终为整数格子坐标
- 移动时先更新逻辑坐标，再播放从旧位置到新位置的**插值动画**
- 移动动画时长：玩家 0.1s，方块 0.08s，敌人 0.15s（可配置）
- 在移动动画期间，实体的逻辑坐标已经更新，碰撞检测基于逻辑坐标

## 1.3 实体基类（Entity Base）

```
class Entity {
    gridX: Int               // 逻辑格子坐标 X
    gridY: Int               // 逻辑格子坐标 Y
    worldX: Float            // 渲染用世界坐标 X（插值）
    worldY: Float            // 渲染用世界坐标 Y（插值）
    entityType: EntityType   // 实体类型枚举
    isMoving: Bool           // 是否正在移动动画中
    moveDir: Direction       // 当前移动方向
    moveProgress: Float      // 0.0~1.0 移动插值进度
    isActive: Bool           // 是否在场景中活跃

    moveTo(dir: Direction): Bool   // 尝试向某方向移动一格
    onMoved()                      // 移动完成回调
    destroy()                      // 从场景中移除
}
```

### 方向枚举

```
enum Direction {
    UP, DOWN, LEFT, RIGHT
}
```

### 实体类型枚举

```
enum EntityType {
    PLAYER,
    BLOCK_NORMAL,
    BLOCK_HEART,
    BLOCK_BOMB,
    BLOCK_STAR,
    ENEMY_JELLY,
    ENEMY_DRAGON,
    BOULDER,
    ITEM_FRUIT,
    ITEM_POWERUP,
}
```

## 1.4 碰撞系统（Collision System）

### 1.4.1 碰撞检测原则

- 基于**格子坐标**检测，不使用连续碰撞
- 每次实体移动后，检查目标格子是否被其他实体占据
- 碰撞优先级：墙 > 方块 > 敌人 > 道具 > 玩家

### 1.4.2 碰撞矩阵

| 碰撞对 | 结果 |
|--------|------|
| 玩家 → 墙壁 | 阻挡，玩家不移动 |
| 玩家 → 方块 | 触发推块逻辑（见第3章） |
| 玩家 → 敌人 | 玩家受伤（无敌状态下敌人死亡） |
| 玩家 → 道具 | 拾取道具 |
| 玩家 → 滚石 | 玩家死亡（过关后仅播放动画） |
| 方块 → 墙壁 | 方块被压碎（普通/炸弹/星星）或停止（心形） |
| 方块 → 方块 | 被推方块压碎（普通/炸弹/星星）或停止（心形） |
| 方块 → 敌人 | 敌人被碾压消灭 |
| 滚石 → 敌人 | 敌人被碾压消灭 |
| 滚石 → 玩家 | 玩家死亡 |
| 滚石 → 方块 | 方块被压碎（所有类型） |

# 第2章 玩家系统

## 2.1 玩家数据结构

```
class Player extends Entity {
    playerID: Int              // 0 = 1P(Don/兔子), 1 = 2P(Pull/松鼠)
    lives: Int                 // 剩余生命数，初始值 3
    score: Int                 // 当前得分
    state: PlayerState         // 当前状态
    invincibleTimer: Float     // 无敌剩余时间（秒）
    isDead: Bool               // 是否已死亡
    respawnTimer: Float        // 重生倒计时
    fruitCount: Int            // 已收集水果数量
    nextLifeFruitCount: Int    // 下一次奖命所需水果数
}
```

### 玩家状态枚举

```
enum PlayerState {
    NORMAL,           // 正常状态
    INVINCIBLE,       // 无敌状态（三心拼合后）
    DAMAGED,          // 受伤无敌帧（短暂闪烁）
    DEAD,             // 死亡
    RESPAWNING,       // 重生中
}
```

## 2.2 玩家移动

### 2.2.1 输入映射

| 输入 | 动作 |
|------|------|
| 8方向摇杆 / 方向键 | 移动（上/下/左/右，4方向，不支持斜向移动） |
| A 按钮 | 推块/碎块 |
| B 按钮 | 推块/碎块（与A功能相同） |

### 2.2.2 移动规则

1. 玩家每帧最多移动一格
2. 移动目标格子判定：
   - **空地（FLOOR）**→ 可移动
   - **墙壁（WALL）**→ 不可移动
   - **方块**→ 触发推块逻辑（见第3章），推块成功则玩家移动，推块失败则玩家不移动
   - **敌人**→ 玩家移动到该格并触发受伤判定
   - **道具**→ 玩家移动到该格并拾取道具
   - **井盖（MANHOLE）**→ 可移动，移动到井盖上时封堵该井盖
3. 玩家不可穿越地图边界

### 2.2.3 移动速度

| 状态 | 移动间隔 | 说明 |
|------|----------|------|
| 正常 | 8 帧/格 | 约 7.5 格/秒 |
| 受伤闪烁 | 8 帧/格 | 与正常相同，但有短暂无敌 |
| 无敌 | 8 帧/格 | 速度不变 |

## 2.3 玩家受伤与死亡

### 2.3.1 受伤判定

玩家接触敌人时触发受伤：
- **正常状态** → 玩家死亡
- **受伤无敌帧** → 无效，不受伤害
- **无敌状态** → 敌人被消灭

### 2.3.2 死亡流程

```
玩家死亡:
    1. 播放死亡动画（不同敌人不同动画）
       - 被史莱姆杀死：史莱姆亲吻，玩家尖叫眩晕倒地
       - 被恐龙杀死：恐龙喷火，玩家被烧黑
       - 被滚石碾死：碾压动画
    2. lives -= 1
    3. 若 lives > 0:
       - 进入 RESPAWNING 状态
       - respawnTimer = 2.0 秒
       - 玩家在出生点重生
       - 重生后获得 2 秒受伤无敌帧
    4. 若 lives <= 0:
       - GAME OVER
```

### 2.3.3 受伤无敌帧

| 触发条件 | 持续时间 | 表现 |
|----------|----------|------|
| 重生后 | 2.0 秒 | 角色闪烁 |
| 被炸弹晕眩的敌人碰到 | 不触发受伤 | — |

# 第3章 方块系统

## 3.1 方块数据结构

```
class Block extends Entity {
    blockType: BlockType       // 方块类型
    isSliding: Bool            // 是否正在滑动中（被推后）
    slideDir: Direction        // 滑动方向
}
```

### 方块类型枚举

```
enum BlockType {
    NORMAL,    // 普通箱子
    HEART,     // 心形箱子
    BOMB,      // 炸弹箱子
    STAR,      // 五角星箱子
}
```

## 3.2 推块机制

### 3.2.1 推块触发条件

1. 玩家面向方块方向按下方向键或推块按钮
2. 玩家与方块相邻（格子坐标差1格）
3. 方块前方（推动方向下一格）满足推入条件

### 3.2.2 推块判定流程

```
玩家尝试推块:
    1. 确定推动方向 dir
    2. 检查方块前方格子 targetCell = block.gridPos + dir
    3. 判断 targetCell 内容:
       a. 空地 + 无实体占据 → 方块滑动到 targetCell
       b. 墙壁 → 方块被压碎（心形箱子除外，见3.3.2）
       c. 另一个方块 → 被推方块压碎（心形箱子除外）
       d. 敌人 → 方块滑动，碾压敌人
       e. 道具 → 方块滑动，道具被摧毁
       f. 滚石 → 方块被压碎
    4. 若方块成功滑动，玩家移动到方块原位置
    5. 若方块被压碎，玩家移动到方块原位置
```

### 3.2.3 方块滑动规则

- 方块被推动后**沿推动方向持续滑动**，直到碰到障碍物
- 滑动过程中每格检测碰撞
- 滑动速度：6 帧/格（比方块推动动画稍慢）
- 滑动过程中碾压路径上所有敌人

```
方块滑动每帧更新:
    1. 计算下一格坐标 nextPos = currentPos + slideDir
    2. 检查 nextPos:
       - 空地 + 无实体 → 继续滑动
       - 敌人 → 碾压敌人，继续滑动
       - 墙壁/方块/边界 → 停止或压碎（见3.3）
       - 滚石 → 方块压碎
    3. 若碾压了敌人，记录碾压数（用于连杀加分）
```

### 3.2.4 连杀加分

| 一次推动碾压敌人数 | 得分倍率 |
|-------------------|----------|
| 1 只 | ×1 |
| 2 只 | ×2 |
| 3 只 | ×3 |
| 4 只及以上 | ×5 |

## 3.3 四种方块详细行为

### 3.3.1 普通箱子（NORMAL）

| 属性 | 值 |
|------|----|
| 可被推动 | 是 |
| 碰到墙壁/方块时 | **被压碎**，从场景中移除 |
| 碾压敌人 | 是 |
| 被压碎后 | 消失，不再出现 |
| 可反复使用 | 否（推到障碍物即碎） |

**设计要点**：普通箱子是最基本的攻防工具，但资源有限。玩家过度推碎箱子会导致掩体减少，容易被围攻。

### 3.3.2 心形箱子（HEART）

| 属性 | 值 |
|------|----|
| 可被推动 | 是 |
| 碰到墙壁/方块时 | **停止**，不会被压碎 |
| 碾压敌人 | 是 |
| 被压碎后 | 不会被压碎 |
| 可反复使用 | 是 |

**设计要点**：心形箱子是不可摧毁的永久掩体，也是拼心机制的核心载体。每关固定3个。

### 3.3.3 炸弹箱子（BOMB）

| 属性 | 值 |
|------|----|
| 可被推动 | 是 |
| 碰到墙壁/方块时 | **爆炸**，箱子消失 |
| 碾压敌人 | 是（滑动中） |
| 被压碎后 | 触发爆炸效果 |
| 可反复使用 | 否 |

**爆炸效果**：
- 爆炸范围：以炸弹格子为中心的 **3×3 区域**（十字形或方形，可配置）
- 爆炸结果：范围内所有敌人进入**晕眩状态**，持续 3 秒
- 晕眩敌人：无法移动，不会攻击，可被推动的方块碾压
- 爆炸不影响玩家（不造成伤害）
- 爆炸不摧毁其他方块

### 3.3.4 五角星箱子（STAR）

| 属性 | 值 |
|------|----|
| 可被推动 | 是 |
| 碰到墙壁/方块时 | **被压碎**，从场景中移除 |
| 碾压敌人 | 是 |
| 被压碎后 | 掉落道具 |
| 可反复使用 | 否 |

**掉落规则**：
- 五角星箱子被推动或被压碎时，**必定**在箱子位置生成一个道具
- 道具类型从道具池中随机选取（见第7章）

# 第4章 敌人系统

## 4.1 敌人数据结构

```
class Enemy extends Entity {
    enemyType: EnemyType       // 敌人类型
    state: EnemyState          // 当前状态
    spawnManholeID: Int        // 从哪个井盖生成
    rageTimer: Float           // 狂暴计时器
    isRaging: Bool             // 是否狂暴
    stunTimer: Float           // 晕眩剩余时间
    moveTimer: Float           // 移动间隔计时器
    targetPlayer: Player?      // 追踪的目标玩家
    attackCooldown: Float      // 攻击冷却时间（恐龙专用）
}
```

### 敌人类型枚举

```
enum EnemyType {
    JELLY,     // 史莱姆/果冻怪
    DRAGON,    // 小恐龙
}
```

### 敌人状态枚举

```
enum EnemyState {
    SPAWNING,    // 从井盖中冒出
    IDLE,        // 待机（短暂停顿）
    CHASING,     // 追踪玩家
    ATTACKING,   // 攻击中（恐龙喷火）
    STUNNED,     // 晕眩（被炸弹炸到）
    RAGING,      // 狂暴状态
    DEAD,        // 死亡
}
```

## 4.2 敌人生成机制

### 4.2.1 井盖系统

```
class Manhole {
    id: Int                    // 井盖唯一ID
    gridX: Int                 // 格子坐标
    gridY: Int
    isBlocked: Bool            // 是否被封堵
    spawnCooldown: Float       // 生成冷却时间
    spawnTimer: Float          // 当前冷却计时器
    spawnQueue: EnemyType[]    // 该井盖的生成队列
}
```

### 4.2.2 生成规则

| 参数 | 值 | 说明 |
|------|----|------|
| 生成方式 | 从井盖中冒出 | 井盖打开动画 → 敌人出现 |
| 生成冷却 | 3~5 秒（可配置） | 同一井盖两次生成间隔 |
| 井盖封堵 | 玩家站在井盖上 | 该井盖停止生成敌人 |
| 封堵解除 | 玩家离开井盖 | 恢复生成 |
| 每关井盖数 | 2~4 个 | 根据关卡配置 |
| 每关敌人总数 | 固定 | 全部消灭即过关 |

### 4.2.3 生成流程

```
每帧更新每个井盖:
    1. 若 isBlocked → 跳过
    2. spawnTimer -= deltaTime
    3. 若 spawnTimer <= 0 且 spawnQueue 非空:
       a. 从 spawnQueue 取出下一个敌人类型
       b. 检查井盖格子是否被占据
       c. 若未被占据:
          - 创建敌人实体
          - 敌人进入 SPAWNING 状态
          - 播放井盖打开动画
          - spawnTimer = spawnCooldown
       d. 若被占据:
          - 延迟 0.5 秒后重试
```

## 4.3 敌人 AI

### 4.3.1 史莱姆（JELLY）AI

| 行为 | 说明 |
|------|------|
| 移动方式 | 跳跃式移动，每 2 格跳跃一次 |
| 移动间隔 | 正常 20 帧/格，狂暴 10 帧/格 |
| 追踪逻辑 | 以 60% 概率向玩家方向移动，40% 随机方向 |
| 攻击方式 | 接触伤害（碰到玩家即造成伤害） |
| 特殊行为 | 无 |

### 4.3.2 小恐龙（DRAGON）AI

| 行为 | 说明 |
|------|------|
| 移动方式 | 步行移动，每格走一步 |
| 移动间隔 | 正常 16 帧/格，狂暴 8 帧/格 |
| 追踪逻辑 | 以 80% 概率向玩家方向移动，20% 随机方向 |
| 攻击方式 | 喷火（远程）+ 接触伤害 |
| 喷火条件 | 与玩家在同一行/列且距离 ≤ 4 格时触发 |
| 喷火冷却 | 3 秒（狂暴 1.5 秒） |
| 喷火范围 | 直线 3 格，火焰占据格子 1.5 秒 |
| 火焰效果 | 火焰占据的格子不可通行，玩家进入则受伤 |

### 4.3.3 追踪算法

```
敌人选择移动方向:
    1. 计算到目标玩家的方向向量
    2. 生成随机数 r (0~1)
    3. 若 r < chaseProbability:
       - 选择使距离最短的方向
       - 若该方向不可行走，选择次优方向
    4. 否则:
       - 从可行走方向中随机选择
    5. 检查目标格子是否可行走:
       - 不可行走 → 原地等待
       - 可行走 → 移动
```

## 4.4 敌人狂暴机制

# 第6章 心形拼合机制

## 6.1 概述

心形拼合是 Don't Pull 最具标志性的核心机制。将3个心形箱子推到**相邻排列**时触发强力奖励。

## 6.2 拼合检测

### 6.2.1 拼合条件

- 场上3个心形箱子**两两相邻**，形成一条直线（横向或纵向）
- 排列形式：`H-H-H`（3格连续）
- 不要求特定顺序，只要最终3个心形箱子连成一线即可

### 6.2.2 检测算法

```
每次心形箱子移动后执行:
    1. 获取所有心形箱子的格子坐标列表 hearts[]
    2. 若 hearts.length < 3 → 返回（不检测）
    3. 检查横向排列:
       - 3个心形是否在同一行
       - X坐标是否连续（排序后差值均为1）
    4. 检查纵向排列:
       - 3个心形是否在同一列
       - Y坐标是否连续（排序后差值均为1）
    5. 若满足任一条件 → 触发心形拼合奖励
```

### 6.2.3 检测时机

- 心形箱子**停止移动时**检测（不是推动瞬间）
- 心形箱子被推到墙壁/方块前停止时也检测

## 6.3 拼合奖励

### 6.3.1 奖励内容

| 奖励 | 数值 | 说明 |
|------|------|------|
| 基础得分 | 5000 分 | 拼合瞬间获得 |
| 无敌时间 | 10 秒 | 玩家进入无敌状态 |
| 水果雨 | 场景中大量生成水果 | 见6.3.2 |

### 6.3.2 水果雨机制

```
心形拼合触发后:
    1. 在场景空地上随机生成 15~25 个水果道具
    2. 水果出现位置必须为空地且无实体占据
    3. 水果分批出现（每0.3秒出现一批，每批3~5个）
    4. 所有敌人进入晕眩状态（持续 = 无敌时间）
    5. 播放特殊BGM（俄罗斯方块变奏）
    6. 无敌倒计时结束后:
       - 敌人恢复行动
       - 水果停止生成
       - BGM恢复
       - 心形箱子保留在原位，可再次拼合
```

### 6.3.3 无敌状态

```
玩家进入无敌状态:
    1. player.state = INVINCIBLE
    2. player.invincibleTimer = 10.0
    3. 无敌期间:
       - 接触敌人 → 敌人死亡
       - 不受滚石伤害
       - 不受火焰伤害
       - 视觉表现：角色发光/闪烁
    4. 倒计时结束:
       - player.state = NORMAL
       - 视觉效果恢复
```

## 6.4 心形箱子的战略意义

- 心形箱子**不可摧毁**，是永久掩体
- 拼心需要精心规划推块路线，过程中容易暴露在敌人攻击下
- 拼心后心形箱子保留原位，可再次拆开/拼合
- 高风险高回报：拼心过程中死亡是常见失败原因
