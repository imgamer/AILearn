# 推砖块压死怪物技能设计文档

## 版本信息
- **引擎版本**：UE 5.7
- **文档日期**：2026/04/29
- **适用场景**：多人联机俯视角/横版动作游戏

---

## 1. 设计目标

设计一个角色主动技能：玩家靠近特定砖块时按键触发，角色播放推动动画，砖块沿直线向前滑动，路径上的怪物被检测并承受碾压伤害。

**核心体验**：
- 简单直观的物理交互
- 一击必杀的爽快感
- 支持多人协作/竞争

---

## 2. 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 技能按键 | Input Action | `IA_PushBlock` | 触发推动的输入绑定 |
| 检测范围 | float | `150.0` | 与砖块的交互距离（单位：厘米） |
| 推动距离 | float | `400.0` | 砖块滑动总距离 |
| 推动时长 | float | `0.8s` | 砖块滑动持续时间 |
| 碾压伤害 | float | `9999.0` | 对怪物造成的伤害值 |
| 冷却时间 | float | `2.0s` | 技能冷却 |
| 检测盒大小 | FVector | `(60,60,60)` | 砖块移动时检测怪物的 Overlap 体积 |

---

## 3. 输出结果

- 角色推动动画 Montage 播放
- 砖块沿直线运动学滑动
- 路径/范围内怪物受到 Server 权威伤害
- 滑动灰尘/碾压血雾粒子效果
- 技能冷却状态更新

---

## 4. 前提假设

- 怪物实现了通用伤害接口 `BPI_Damageable`
- 可推动砖块使用特定基类 `BP_PushBlockBase`
- 关卡碰撞设置合理，砖块不会穿墙
- 角色动画资源包含推动 Montage `AM_PushBlock`
- 数据资产 `DA_PushBlockSettings` 已配置

---

## 5. 实现方案

### 5.1 组件设计

#### `BP_PushBlockKillComponent`（附加到角色）

**变量定义：**

| 名称 | 类型 | 说明 |
|------|------|------|
| `bCanPush` | bool | 技能是否可用 |
| `PushCooldownRemaining` | float | 剩余冷却时间 |
| `TargetBlock` | BP_PushBlockBase | 当前目标砖块 |
| `PushSettings` | DA_PushBlockSettings | 参数数据资产 |

**事件流程：**

##### `TryPushBlock`（绑定到输入 `IA_PushBlock`）

```
1. Branch: bCanPush && PushCooldownRemaining <= 0
2. 在角色位置做 SphereTraceByChannel，半径 = PushSettings.PushRange
3. ForEach 命中结果，过滤 Actor Class == BP_PushBlockBase 且 bIsBeingPushed == false
4. 若有命中，设置 TargetBlock = 第一个有效命中
5. 计算推动方向：GetActorForwardVector -> Vector * (1,1,0) -> Normalize
6. 调用 Server_RequestPushBlock(TargetBlock, PushDirection)
```

##### `Server_RequestPushBlock`（Reliable Server RPC）

```
1. 验证条件：
   - IsValid(TargetBlock)
   - DistanceTo(TargetBlock) <= PushSettings.PushRange
   - TargetBlock.bIsBeingPushed == false
2. TargetBlock.bIsBeingPushed = true
3. 调用 Multicast_PlayPushEffects(TargetBlock, PushDirection)
```

##### `Multicast_PlayPushEffects`（Multicast RPC）

```
1. PlayAnimMontage(AM_PushBlock)
2. TargetBlock -> 调用接口 PushBlock(PushDirection, PushSettings.PushDistance, PushSettings.PushDuration)
```

---

#### `BP_PushBlockBase`（可推动砖块基类）

**变量定义（Replication）：**

| 名称 | 类型 | 默认值 | 同步方式 |
|------|------|--------|----------|
| `bIsBeingPushed` | bool | false | Replicated + RepNotify |
| `PushStartLoc` | FVector | 0,0,0 | 确定性计算，无需同步 |
| `PushTargetLoc` | FVector | 0,0,0 | 确定性计算，无需同步 |
| `PushDuration` | float | 0.8 | 确定性计算，无需同步 |
| `PushElapsed` | float | 0.0 | 本地计时 |

**组件构成：**

| 组件 | 用途 |
|------|------|
| `StaticMeshComponent` | 砖块网格 |
| `BoxCollision` | 碰撞体（覆盖砖块体积） |
| `Timeline`（`TL_BlockPush`） | 推动动画，Float Track: Curve 0->1 |

**事件流程：**

##### `PushBlock(Direction, Distance, Duration)`

```
1. PushStartLoc = GetActorLocation()
2. PushTargetLoc = PushStartLoc + Direction * Distance
3. PushDuration = Duration
4. PushElapsed = 0.0
5. bIsBeingPushed = true
6. SetActorTickEnabled(true)
```

##### Tick 更新（或 Timeline Update）

```
1. PushElapsed += DeltaTime
2. Alpha = PushElapsed / PushDuration
3. Branch: Alpha >= 1.0
   - Alpha = 1.0
   - bIsBeingPushed = false
   - SetActorTickEnabled(false)
   - 调用 FinishPush()
   - Return
4. NewLoc = Lerp(PushStartLoc, PushTargetLoc, Alpha)
5. CheckWallCollision: 从当前位置向 NewLoc 做 LineTrace/Sweep
   - 若命中 WorldStatic，则提前停止
6. SetActorLocation(NewLoc)
7. 调用 CheckCrushDamage()
```

##### `CheckCrushDamage()`（仅在 Server 执行）

```
1. GetOverlappingActors（Class Filter = 怪物基类 / 接口 BPI_Damageable）
2. ForEach 命中 Actor：
   - 过滤条件：DotProduct(PushTargetLoc - PushStartLoc, ActorLocation - GetActorLocation()) > 0
     （只伤害前方，避免后方怪物被误伤）
   - 调用 BPI_Damageable.TakeDamage(Damage=9999, Type=Crushed, Instigator=Pusher, Causer=self)
3. 被伤害的怪物触发：
   - 播放粒子 NS_BlockCrush
   - 播放碾压音效
```

##### `FinishPush()`

```
1. 播放停止撞击音效
2. 若落入 KillZ 或触发坠落区域，可扩展为 Falling 状态
```

---

#### `BPI_Damageable`（伤害接口）

| 函数 | 输入参数 |
|------|----------|
| `TakeDamage` | `DamageAmount: float`, `DamageType: E_DamageType`, `Instigator: Pawn`, `DamageCauser: Actor` |

## 6. 多人同步方案

### 6.1 责任分工

| 方面 | 处理方案 |
|------|----------|
| **权威位置** | Server 控制 PushStartLoc / PushTargetLoc，客户端本地插值 |
| **RPC 流程** | Client -> Server_RequestPushBlock -> Server 验证 -> Multicast_PlayPushEffects -> 砖块移动开始 |
| **状态同步** | `bIsBeingPushed` Replicated + RepNotify，客户端用于启用/禁用交互提示 |
| **伤害判定** | **仅在 Server** 执行 CheckCrushDamage 和 TakeDamage，结果走正常 Gameplay 框架同步 |
| **位置同步** | 移动是确定性动画（起点+方向+时长），各端本地计算位置；Server 结束时校正 |
| **竞争条件** | Server 验证 `bIsBeingPushed == false`，第二个请求自然失败 |

### 6.2 RPC 定义

| RPC 名称 | 类型 | 可靠性 | 调用端 | 执行端 |
|----------|------|--------|--------|--------|
| `Server_RequestPushBlock` | Server | Reliable | Client | Server |
| `Multicast_PlayPushEffects` | Multicast | Reliable | Server | All Clients |

## 8. 测试清单

### 8.1 PIE 单人测试

- [ ] 靠近砖块时出现交互提示，超出范围消失
- [ ] 按技能键 -> 角色播放 `AM_PushBlock`
- [ ] 砖块沿角色朝向直线滑动 `PushDistance`
- [ ] 滑动路径上的怪物被击杀/受 `Crushed` 伤害
- [ ] 砖块撞墙（WorldStatic）提前停止，不穿透
- [ ] 冷却期间无法再次推动
- [ ] 砖块被推动中时其他玩家/自己无法选中它

### 8.2 Dedicated Server 多人测试

- [ ] Client A 推动，Client B 同步看到动画和砖块移动
- [ ] Server 上砖块位置权威，客户端无肉眼可见漂移
- [ ] 两玩家同时按键推同一块砖，仅一人成功（Server 竞争验证）
- [ ] 高延迟（200ms+）下，Client 本地立即播放蒙太奇，无操作粘滞感

### 8.3 边界情况

- [ ] 角色在空中/被击飞时无法触发推动
- [ ] 怪物在砖块后方/侧面，不被碾压
- [ ] 砖块被推到悬崖/坑洞下（若有关卡设计）
- [ ] 推动过程中角色死亡，砖块继续滑行至终点
- [ ] 推动过程中关卡流送卸载砖块 -> 安全处理
- [ ] 斜坡上推动时砖块贴地（Tick 中用 LineTrace 调整 Z 高度）

## 9. 后续扩展建议

1. **多尺寸砖块**：扩展 `BP_PushBlockBase` 子类，支持大/中/小型砖块，不同伤害范围
2. **连锁反应**：大砖块被推动时可能撞碎前方小砖块
3. **环境互动**：砖块落入水中产生波浪，落入岩浆产生蒸汽
4. **成就系统**：记录"一砖双杀""一砖三杀"等成就
5. **物理版本**：可选使用 Physics Simulation 替代 Kinematic，支持更复杂的物理交互
