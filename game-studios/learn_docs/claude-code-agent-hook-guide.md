# Claude Code Agent 与 Hook 机制详解

## 一、Agent 基础概念

### 1.1 Agent 定义文件 vs Agent 实例

| 概念 | 说明 | 类比 |
|------|------|------|
| **Agent 定义文件** (.md) | 角色设定文件，定义职责、工作方式、协作协议 | 类定义 (Class) |
| **Agent 实例** | 运行时创建的对话上下文，可以"扮演"某个角色 | 类实例 (Instance) |

**关键理解**：
- `.claude/agents/*.md` 文件只是**静态配置**，由项目创建者编写
- 运行时"创建 Agent"指的是创建 Agent 实例
- 用户打开 Claude Code → 自动创建一个 Agent 实例
- 使用 Subagent 工具 → 创建一个新的子 Agent 实例

### 1.2 父代理 vs 子代理

#### 关系图解

## 二、Hook 机制详解

### 2.1 Hook 的本质

**Hook 的本质**：在 Claude Code 会话生命周期中的特定节点插入自定义逻辑。

这是 Claude Code 预留的钩子位，就像一个个插槽：
- 打开 Claude 程序时执行一个钩子（SessionStart）
- 保存文件时执行一个钩子（PostToolUse）
- 工具调用前执行一个钩子（PreToolUse）

### 2.2 预定义的 Hook 插槽

```
┌─────────────────────────────────────────────────────────────────┐
│              Claude Code 预定义的钩子插槽（Hook Slots）              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  这些插槽是由 Claude Code 系统预定义的，就像一个个"钩子点"：           │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  插槽名称          触发时机           输入数据类型        │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │  SessionStart      会话开始时         无                  │   │
│  │  Stop              会话结束时         无                  │   │
│  │  PreCompact        上下文压缩前       无                  │   │
│  │  SubagentStart     子代理创建时       Agent信息(JSON)       │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │  PreToolUse        工具调用前         工具输入(JSON)        │   │
│  │  PostToolUse       工具调用后         工具输入/输出(JSON)    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  这些插槽是 Claude Code 系统内置的，就像"插座"一样：                  │
│  · 系统定义了这些插槽的存在                                          │
│  · 每个插槽在特定时机被"激活"                                         │
│  · 你可以把脚本"插"到这些插槽上                                        │
│  · 插槽激活时，会自动执行插上的脚本                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 如何指定 sh 文件插到哪个插槽？

```
┌─────────────────────────────────────────────────────────────────┐
│              通过 settings.json 配置钩子绑定                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  关键文件：.claude/settings.json                                  │
│  这个文件告诉 Claude Code：当某个插槽激活时，执行哪些脚本               │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  配置结构：                                                │   │
│  │                                                           │   │
│  │  {                                                        │   │
│  │    "hooks": {                                             │   │
│  │      // 插槽名称: [脚本1, 脚本2, ...]                        │   │
│  │      "SessionStart": [                                    │   │
│  │        ".claude/hooks/session-start.sh",                  │   │
│  │        ".claude/hooks/detect-gaps.sh"                      │   │
│  │      ],                                                   │   │
│  │      "Stop": [                                            │   │
│  │        ".claude/hooks/session-stop.sh"                     │   │
│  │      ],                                                   │   │
│  │      "PreCompact": [                                      │   │
│  │        ".claude/hooks/pre-compact.sh"                      │   │
│  │      ],                                                   │   │
│  │      "SubagentStart": [                                   │   │
│  │        ".claude/hooks/log-agent.sh"                        │   │
│  │      ],                                                   │   │
│  │      // 带条件的插槽配置                                      │   │
│  │      "PreToolUse": {                                      │   │
│  │        // 只有调用Bash工具时才触发                             │   │
│  │        "Bash": [                                          │   │
│  │          ".claude/hooks/validate-commit.sh",               │   │
│  │          ".claude/hooks/validate-push.sh"                  │   │
│  │        ]                                                  │   │
│  │      },                                                   │   │
│  │      "PostToolUse": {                                     │   │
│  │        // Write或Edit工具调用后触发                          │   │
│  │        "Write": [                                         │   │
│  │          ".claude/hooks/validate-assets.sh"                │   │
│  │        ],                                                 │   │
│  │        "Edit": [                                          │   │
│  │          ".claude/hooks/validate-assets.sh"                │   │
│  │        ]                                                  │   │
│  │      }                                                    │   │
│  │    }                                                      │   │
│  │  }                                                        │   │
│  │                                                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  配置规则：                                                        │
│  · 插槽名称必须是系统预定义的（见上文列表）                            │
│  · 值可以是：                                                      │
│    - 字符串数组：[脚本路径1, 脚本路径2]                             │
│    - 条件对象：{ "工具名": [脚本数组], ... }                        │
│  · 多个脚本按顺序执行                                              │
│  · 脚本路径相对于项目根目录                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 五、总结

### 5.1 Agent 核心要点

| 问题 | 答案 |
|------|------|
| **父代理 vs 子代理的区别** | 父代理是用户直接对话的主会话；子代理是通过工具创建的独立上下文，用于并行/专注任务 |
| **创建方式区别** | 父代理自然创建；子代理通过 `Subagent` 工具调用，需指定 agent_name 和 task |
| **Claude Code 如何区分** | 通过工具调用链：用户消息→父代理→Subagent 工具→新建上下文→子代理执行→结果返回 |
| **子 Agent 核心价值** | 上下文隔离、模块化、降低复杂度、降低心智成本 |

### 5.2 Hook 核心要点

| 问题 | 答案 |
|------|------|
| **钩子是 Claude Code 的机制吗？** | **是的**，钩子是 Claude Code 原生支持的机制，通过 Hook Engine 调度执行 |
| **谁来调度触发？** | Claude Code 的 Hook Engine 监听内部事件（SessionStart、PreToolUse 等），自动触发绑定钩子 |
| **为什么用 .sh 文件？** | 钩子在操作系统层面执行，需要访问文件系统、git、环境变量；shell 脚本是最轻量、最通用的可执行格式 |
| **如何指定 sh 文件插哪个插槽？** | 在 `.claude/settings.json` 的 `hooks` 配置中绑定 |
| **发生事件时肯定会触发吗？** | 是的，只要配置了就会按顺序执行，这是系统级保证 |
| **脚本可以阻止操作吗？** | PreToolUse 钩子可以通过 `exit 2` 阻止工具调用 |

## 四、核心要点速查表

### Agent 机制

| 概念 | 说明 |
|------|------|
| **Agent 定义文件** | 静态角色设定（.md 文件），类比类定义 |
| **Agent 实例** | 运行时创建，类比类实例 |
| **父代理** | 用户直接对话的主会话，或调用 Subagent 的代理 |
| **子代理** | 通过 Subagent 工具创建的独立上下文，用于专注任务 |
| **核心价值** | 上下文隔离、模块化、降低复杂度和心智成本 |

### Hook 机制

| 概念 | 说明 |
|------|------|
| **Hook 插槽** | Claude Code 预定义的生命周期事件点（SessionStart, PreToolUse 等） |
| **Hook 脚本** | 绑定到插槽的 .sh 文件，在特定时机执行 |
| **配置方式** | 在 .claude/settings.json 中绑定脚本到插槽 |
| **触发保证** | 系统级事件驱动，只要配置就会触发 |
| **阻止操作** | PreToolUse 钩子可以通过 exit 2 阻止工具调用 |

## 五、总结

Claude Code 的 Agent 和 Hook 机制设计精妙：

1. **Agent 机制**通过上下文隔离和模块化，让复杂任务变得可管理
2. **Hook 机制**通过生命周期事件插槽，让开发者可以在关键节点插入自定义逻辑
3. **两者结合**，构建了一个既灵活又强大的 AI 辅助开发环境

理解这些机制，将帮助你更好地利用 Claude Code 构建高效的工作流。
