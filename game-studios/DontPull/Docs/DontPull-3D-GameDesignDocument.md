## 2D→3D核心设计原则

| 原则 | 说明 |
|------|------|
| **玩法不变** | 核心规则（推块/碾压/拼心/狂暴）与2D完全一致，3D是表现层升级 |
| **格子对齐不变** | 所有实体仍以整数格子坐标移动，3D世界坐标仅用于渲染插值 |
| **手感优先** | 移动/推块/碾压的操作响应必须与2D原版一致，3D动画不可拖慢逻辑 |
| **信息可读性** | 等轴测3D视角下，玩家必须能一眼看清所有格子状态（方块/敌人/道具） |
| **3D增值** | 3D化必须带来2D无法实现的表现力（立体感/光影/粒子/动画），否则不做 |

# 第1章 3D游戏框架与网格系统

## 1.1 网格系统3D化

### 1.1.1 基本参数变化

| 参数 | 2D原版 | 3D重制 | 说明 |
|------|--------|--------|------|
| 格子尺寸 | 32×32 像素 | 100×100 UE单位（1米） | 3D空间中每格1m²，角色约0.8m高 |
| 坐标系 | 2D (X, Y) | 3D (X, Y, Z)，Z仅用于渲染高度 | 逻辑坐标仍为2D整数，Z轴由实体类型决定 |
| 地图宽度 | 13~17 列 | 同2D | 格子数不变 |
| 地图高度 | 11~15 行 | 同2D | 格子数不变 |

### 1.1.2 格子坐标→世界坐标映射

```text
GridToWorld 映射规则：
  WorldX = GridX × CellSize          // CellSize = 100.0
  WorldY = GridY × CellSize
  WorldZ = 实体类型决定的固定高度
    - 地板: Z = 0
    - 墙壁: Z = 0, 高度 = 150 (1.5m)
    - 方块: Z = 0, 高度 = 80 (0.8m)
    - 玩家: Z = 0, 脚底在地面
    - 道具: Z = 50, 悬浮在半空
    - 井盖: Z = 0, 嵌入地面
```

### 1.1.3 格子类型（与2D一致）

| 类型 | 3D表现 | 说明 |
|------|--------|------|
| FLOOR | 地板贴图平面 | 可通行 |
| WALL | 3D墙壁方块 | 不可通行，高1.5m |
| MANHOLE | 地面圆形井盖 | 敌人生成点，可踩踏封堵 |
| MANHOLE_BLOCKED | 井盖+封堵标记 | 玩家站在上面时状态 |
| BORDER_PATH | 外周轨道 | 滚石专用通道 |

### 1.1.4 网格管理器（3D版）

```cpp
UCLASS()
class UGridManagerComponent : public UActorComponent
{
    UPROPERTY(EditAnywhere)
    int32 GridWidth = 15;

    UPROPERTY(EditAnywhere)
    int32 GridHeight = 13;

    UPROPERTY(EditAnywhere)
    float CellSize = 100.0f;

    UPROPERTY()
    TArray<ECellType> Cells;

    ECellType GetCellType(int32 X, int32 Y) const;
    bool IsWalkable(int32 X, int32 Y) const;
    FVector GridToWorld(int32 X, int32 Y) const;
    FIntPoint WorldToGrid(FVector WorldPos) const;
};
```

**与2D的差异**：数据结构完全相同，仅 `GridToWorld` 输出从 `Vector2` 变为 `FVector`（Z轴由调用方根据实体类型填充）。

## 1.2 游戏循环（与2D一致）

```text
每帧执行:
    1. 处理输入（等轴测方向映射，见第2章）
    2. 更新游戏逻辑（顺序与2D完全一致）
       a. 玩家移动与推块
       b. 方块移动与碰撞
       c. 敌人 AI 与移动
       d. 滚石移动
       e. 道具生成与拾取
       f. 心形拼合检测
       g. 碰撞判定
       h. 计时器更新
    3. 更新3D渲染位置（逻辑坐标→世界坐标插值）
    4. 检查胜负条件
    5. 渲染
```

**3D新增步骤**：步骤3"更新3D渲染位置"是2D没有的。逻辑坐标瞬间更新，3D位置通过插值动画平滑过渡。

## 1.3 实体基类（3D版）

```cpp
UCLASS()
class ADontPullEntity : public AActor
{
    UPROPERTY()
    FIntPoint GridPos;              // 逻辑格子坐标（与2D一致）

    UPROPERTY()
    bool bIsMoving = false;

    UPROPERTY()
    EDirection MoveDirection;

    UPROPERTY()
    float MoveProgress = 0.0f;

    UPROPERTY()
    float MoveDuration = 0.1f;

    UPROPERTY()
    EEntityType EntityType;

    virtual void TickMovement(float DeltaTime);
    virtual void OnMoved();
};
```

**与2D的差异**：
- 2D用 `worldX/worldY` 做渲染插值 → 3D用 `SetActorLocation` 由引擎渲染
- 3D实体有朝向（Yaw旋转），2D用精灵翻转
- 3D实体有SkeletalMesh/StaticMesh组件，2D用Sprite

## 1.4 碰撞系统（与2D逻辑一致）

**核心原则不变**：碰撞判定基于格子坐标，不使用UE物理碰撞。

| 碰撞对 | 2D结果 | 3D结果 | 变化 |
|--------|--------|--------|------|
| 玩家 → 墙壁 | 阻挡 | 阻挡 | 无 |
| 玩家 → 方块 | 推块 | 推块 | 无 |
| 玩家 → 敌人 | 受伤/无敌杀敌 | 受伤/无敌杀敌 | 无 |
| 方块 → 墙壁 | 压碎/停止 | 压碎/停止 | 无 |
| 方块 → 敌人 | 碾压 | 碾压 | 无 |
| 滚石 → 任何 | 碾压 | 碾压 | 无 |

3D碰撞的**辅助用途**：UE碰撞通道用于视觉触发（如玩家走进敌人范围时触发受伤动画），不用于逻辑判定。

# 第2章 等轴测视角与输入映射

> 本章是2D→3D最大的设计变化点

## 2.1 视角方案

### 2.1.1 假正交透视等轴测

| 参数 | 值 | 说明 |
|------|----|------|
| 投影模式 | Perspective | 不用Orthographic |
| FOV | 10° | 近乎正交但有3D立体感 |
| 相机距离 | 3000~5000 UE单位 | 配合FOV覆盖游戏区域 |
| Pitch | -35.264° | 标准等轴测角 |
| Yaw | 45° | 标准等轴测角 |
| 跟随模式 | 固定不动 | 不跟随角色 |

**为什么不用正交投影**：正交投影下3D物体侧面无透视缩放，看起来像"纸片"，缺乏3D立体感。极小FOV透视保留<2%畸变（肉眼不可辨），但侧面/顶面有完整的3D深度信息。

**为什么不用跟随相机**：原版街机是固定视角，整个游戏区域始终可见。3D重制保持这一设计，相机作为关卡级固定Actor。

### 2.1.2 等轴测方向感知

```text
等轴测视角下的方向感知（Yaw=45°, Pitch=-35.264°）：

        屏幕上方
          ↗ 东北
         ╱
  西北 ↖╱        ╲↗ 东北（世界+X方向）
       ╱          ╲
      ╱            ╲
屏幕左 ←————  ————→ 屏幕右
      ╲            ╱
       ╲          ╱
  西南 ↙╲        ╱↘ 东南
         ╲
          ↙ 西南
        屏幕下方

世界坐标映射：
  屏幕"上" → 世界 (-X, -Y) 方向
  屏幕"下" → 世界 (+X, +Y) 方向
  屏幕"左" → 世界 (-X, +Y) 方向
  屏幕"右" → 世界 (+X, -Y) 方向
```

## 2.2 输入方向映射

### 2.2.1 映射方案选择

| 方案 | 说明 | 推荐 |
|------|------|------|
| A. 屏幕相对映射 | 按上→角色向屏幕上方移动（世界斜向） | ✅ 推荐 |
| B. 世界坐标映射 | 按上→角色向世界+X移动（屏幕斜向） | ❌ 反直觉 |
| C. 8方向映射 | 支持4正方向+4斜方向 | ❌ 原版4方向 |

**推荐方案A**：屏幕相对映射。玩家按"上"时角色向屏幕上方移动，符合直觉。内部将屏幕方向转换为世界格子方向。

### 2.2.2 屏幕方向→格子方向转换

```text
输入映射表（Yaw=45°等轴测）：

  玩家按键    屏幕方向    世界移动方向    格子坐标变化
  ─────────  ─────────  ────────────  ────────────
  W / ↑      上         东北(+X, -Y)   GridX+1, GridY-1
  S / ↓      下         西南(-X, +Y)   GridX-1, GridY+1
  A / ←      左         西北(-X, -Y)   GridX-1, GridY-1
  D / →      右         东南(+X, +Y)   GridX+1, GridY+1
```

**关键问题**：等轴测视角下，屏幕的4个方向对应世界的4个对角线方向。但Don't Pull的格子移动是4方向（上下左右），不是8方向。

### 2.2.3 4方向格子移动适配方案

**问题**：等轴测视角下，屏幕的"上/下/左/右"对应世界的对角线方向，而格子移动只有4个正方向。

**方案A：旋转网格45°（✅ 推荐）**

```text
将网格本身旋转45°，使格子的4个方向与屏幕4方向对齐：

  格子方向        屏幕方向
  GridUp    →    屏幕上
  GridDown  →    屏幕下
  GridLeft  →    屏幕左
  GridRight →    屏幕右

实现方式：
  GridToWorld 映射时加入45°旋转：
  WorldX = (GridX - GridY) × CellSize × 0.707
  WorldY = (GridX + GridY) × CellSize × 0.707
```

**优势**：输入直觉完美，按上就向屏幕上方移动，格子逻辑不变

**方案B：保持正交网格，输入映射对角线**

```text
网格不旋转，屏幕方向映射到格子对角线：
  按上 → 尝试 GridX+1, GridY-1（需要两步移动）

问题：一次按键移动两格，手感与原版不同
```

**❌ 不推荐**：破坏了"一次按键移动一格"的原版手感

**方案C：8方向移动**

```text
扩展为8方向格子移动，斜方向也允许

问题：原版只有4方向，8方向会改变玩法平衡
```

**❌ 不推荐**：改变核心玩法

### 2.2.4 推荐方案详细设计（方案A：旋转网格45°）

```text
网格坐标系（旋转45°后）：

  格子坐标 (gx, gy) → 世界坐标 (wx, wy)：
    wx = (gx - gy) × CellSize × 0.707
    wy = (gx + gy) × CellSize × 0.707
    wz = 实体高度

  世界坐标 (wx, wy) → 格子坐标 (gx, gy)：
    gx = round((wx / 0.707 + wy / 0.707) / (2 × CellSize))
    gy = round((-wx / 0.707 + wy / 0.707) / (2 × CellSize))

方向枚举（与屏幕方向对齐）：
  UP    → GridY-1（屏幕上方）
  DOWN  → GridY+1（屏幕下方）
  LEFT  → GridX-1（屏幕左方）
  RIGHT → GridX+1（屏幕右方）
```

**视觉效果**：格子呈菱形排列，与等轴测视角完美对齐。玩家按上，角色沿菱形边向上移动一格。

## 2.3 手柄输入适配

| 手柄输入 | 映射方向 | 说明 |
|----------|----------|------|
| 左摇杆上 | GridUp | 摇杆角度→4方向量化 |
| 左摇杆下 | GridDown | 死区30%，避免误触 |
| 左摇杆左 | GridLeft | — |
| 左摇杆右 | GridRight | — |
| A键 | 推块 | 与2D一致 |
| Start | 暂停 | — |

**摇杆量化规则**：将360°摇杆输入量化为4方向（0°/90°/180°/270°），每方向覆盖90°扇区。不使用8方向。

## 2.4 视角相关设计决策

### 2.4.1 深度遮挡

| 问题 | 方案 |
|------|------|
| 前方墙壁遮挡后方角色 | 墙壁使用半透明材质（被遮挡时自动变透明） |
| 方块堆叠时看不清 | Don't Pull无堆叠，不存在此问题 |
| 角色被方块遮挡 | 方块高度低于角色，不会完全遮挡 |
| 敌人在墙后看不到 | 2D原版也看不到，保持一致 |

### 2.4.2 视野范围

| 参数 | 值 | 说明 |
|------|----|------|
| 可见格子范围 | 约 17×13 格 | FOV 10° + 距离 4000 的覆盖范围 |
| 最小地图 | 13×11 | Beginner Course |
| 最大地图 | 17×15 | Professional Course |
| 相机适配 | 根据地图尺寸自动调整距离 | 大地图拉远，小地图拉近 |

# 第3章 3D玩家系统

## 3.1 角色模型规格

### 3.1.1 角色设计

| 角色 | 2D原版 | 3D重制 | 说明 |
|------|--------|--------|------|
| Don（1P兔子） | 32×32像素精灵 | 约80cm高3D角色 | Q版比例，大头小身 |
| Pull（2P松鼠） | 32×32像素精灵 | 约80cm高3D角色 | Q版比例，与兔子同高 |

### 3.1.2 模型规格

| 参数 | 值 | 说明 |
|------|----|------|
| 角色高度 | 80 UE单位（0.8m） | 比格子小，不遮挡视线 |
| 角色宽度 | 60 UE单位（0.6m） | 格子内居中 |
| 骨骼数 | 约20根 | 简化骨骼，Q版角色不需要复杂骨骼 |
| LOD数 | 2 | 近景全骨骼，远景简化 |
| 面数 | 2000~3000三角面 | Q版角色，低面数即可 |

### 3.1.3 角色朝向

```text
3D角色有8个朝向（Yaw旋转），但移动只有4方向：

  移动方向    角色Yaw旋转
  GridUp      -45°（面向屏幕上方）
  GridDown    135°（面向屏幕下方）
  GridLeft    -135°（面向屏幕左方）
  GridRight   45°（面向屏幕右方）

移动时角色平滑旋转到目标朝向（0.05秒），不是瞬间转向。
推块时角色面向方块方向，不旋转。
```

## 3.2 玩家移动（3D版）

### 3.2.1 移动规则（与2D一致）

| 规则 | 2D | 3D | 变化 |
|------|----|----|------|
| 每次移动一格 | 是 | 是 | 无 |
| 移动动画时长 | 0.1秒 | 0.1秒 | 无 |
| 移动中不可改变方向 | 是 | 是 | 无 |
| 移动逻辑坐标瞬间更新 | 是 | 是 | 无 |

### 3.2.2 3D移动动画

```text
2D：精灵切换（2帧走路动画）
3D：骨骼动画 + 位置插值

移动流程：
  1. 输入触发 → 检查目标格子可行性
  2. 逻辑坐标瞬间更新（GridPos = TargetPos）
  3. 播放走路动画（AnimBP: Walk状态）
  4. 位置插值：从旧世界坐标到新世界坐标（0.1秒）
  5. 角色朝向平滑旋转到移动方向（0.05秒）
  6. 插值完成 → 切回Idle动画
```

### 3.2.3 移动速度（与2D一致）

| 状态 | 移动间隔 | 说明 |
|------|----------|------|
| 正常 | 8帧/格（约0.133秒） | 约7.5格/秒 |
| 受伤闪烁 | 同正常 | 有短暂无敌 |
| 无敌 | 同正常 | 速度不变 |

## 3.3 玩家状态（与2D一致，3D表现不同）

| 状态 | 2D表现 | 3D表现 |
|------|--------|--------|
| NORMAL | 默认精灵 | 默认材质，Idle/Walk动画 |
| INVINCIBLE | 精灵闪烁+光环 | 发光材质（Fresnel+Emissive）+光环粒子 |
| DAMAGED | 精灵快速闪烁 | 半透明脉冲材质，0.2秒间隔 |
| DEAD | 死亡精灵动画 | 骨骼死亡动画（不同死因不同动画） |
| RESPAWNING | 精灵闪烁出现 | 渐入材质+出生点粒子特效 |

## 3.4 玩家受伤与死亡（3D版）

### 3.4.1 受伤判定（与2D一致）

| 情况 | 结果 |
|------|------|
| 正常状态接触敌人 | 玩家死亡 |
| 受伤无敌帧期间 | 不受伤害 |
| 无敌状态接触敌人 | 敌人被消灭 |

### 3.4.2 3D死亡动画

| 死因 | 2D动画 | 3D动画 |
|------|--------|--------|
| 被史莱姆杀死 | 史莱姆亲吻，玩家倒地 | 史莱姆拥抱动画→玩家眩晕倒地骨骼动画 |
| 被恐龙杀死 | 恐龙喷火，玩家烧黑 | 恐龙喷火→玩家烧黑材质+冒烟粒子 |
| 被滚石碾死 | 碾压动画 | 碾压扁平化骨骼动画+灰尘粒子 |
| 被火焰烧到 | 烧黑动画 | 烧黑材质+着火粒子 |

### 3.4.3 3D重生表现

```text
重生流程：
  1. 玩家在出生点出现（逻辑坐标瞬间设置）
  2. 播放渐入动画（0.5秒，从透明到不透明）
  3. 播放出生点粒子特效（光环扩散）
  4. 进入2秒受伤无敌帧（闪烁材质）
```

# 第4章 3D方块系统

## 4.1 方块3D模型规格

### 4.1.1 方块外观设计

| 方块类型 | 2D外观 | 3D外观 | 模型规格 |

| NORMAL | 棕色木箱 | 3D木箱，有木纹贴图 | 80×80×80cm，StaticMesh |
