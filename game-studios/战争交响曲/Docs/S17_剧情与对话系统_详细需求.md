# S17 剧情与对话系统 - 详细需求文档

## 1. 系统概述

剧情与对话系统采用视觉小说式呈现方式，连接章节推进、角色羁绊、世界观展开三大叙事维度。系统覆盖30章主线剧情与15段羁绊剧情，约2000条对话文本，支持角色立绘表情切换、选择分支、CG演出、历史回看等核心功能。

| 模块 | 功能 | 规模 |
|------|------|------|
| 剧情触发 | 章节/战斗/羁绊/条件触发 | 30章主线+15段羁绊 |
| 对话呈现 | 立绘+对话框+逐字显示+自动推进 | ~2000条对话 |
| 选择分支 | 2~3选项+条件+属性影响 | ~150个选择点 |
| CG系统 | 关键剧情CG+图鉴收集 | ~40张CG |
| 主角创建 | 性别+名字+9问属性 | 1次流程 |

## 4. 剧情触发系统

### 4.1 触发类型

| 触发点 | 触发时机 | 示例 |
|--------|---------|------|
| 章节开始前 | 进入关卡前 | 第1章开始→播放序章剧情 |
| 章节结束后 | 关卡结算后 | 第1章通关→播放结算剧情 |
| 战斗中事件 | Boss出现/角色死亡 | Boss登场→播放Boss出场台词 |
| Home Base | 羁绊剧情入口 | 与角色对话→触发羁绊剧情 |
| 条件触发 | 特定角色在队/章节通关 | 拥有角色A+通关第5章→触发隐藏剧情 |

### 4.2 触发检测流程

```
CheckStoryTrigger(trigger_type, context):
    chapters = TB_StoryChapter.Where(ch => MatchesTrigger(ch.trigger_condition, trigger_type, context))

    FOR EACH chapter IN chapters:
        IF NOT chapter.is_completed AND NOT PlayerHasWatched(chapter.chapter_id):
            IF EvaluateCondition(chapter.trigger_condition, context):
                RETURN chapter  // 返回第一个满足条件的章节

    RETURN null  // 无触发

function MatchesTrigger(condition_json, trigger_type, context):
    condition = ParseJson(condition_json)
    IF condition.type != trigger_type:
        RETURN false
    RETURN EvaluateCondition(condition_json, context)

function EvaluateCondition(condition_json, context):
    condition = ParseJson(condition_json)
    SWITCH condition.type:
        CASE "chapter_start":
            RETURN context.current_chapter == condition.chapter
        CASE "chapter_end":
            RETURN context.completed_chapter == condition.chapter
        CASE "battle_event":
            RETURN context.battle_event == condition.event
        CASE "bond_talk":
            RETURN context.character_id == condition.character_id
        CASE "condition":
            RETURN EvaluateComplexCondition(condition, context)
    RETURN false
```

### 4.3 条件触发详细规则

### 4.4 触发优先级

- 同一触发点多个剧情满足条件时，按 sort_weight 降序触发（权重高的优先）
- 主线剧情优先级 > 羁绊剧情
- 已观看过的剧情不重复触发（除非手动回看）

## 5. 对话呈现系统

### 5.1 角色立绘

| 属性 | 说明 |
|------|------|
| 位置 | 左侧(left) / 中央(center) / 右侧(right) |
| 表情切换 | normal / angry / sad / happy / surprised / thinking / shy |
| 入场动画 | fade_in / slide_in_left / slide_in_right / none |
| 退场动画 | fade_out / slide_out_left / slide_out_right / none |
| 灰显 | 回忆/幻影场景中角色灰显处理 |

```
UpdateCharacterPortrait(character_id, expression, position):
    1. 获取角色立绘组件 portrait = GetPortrait(character_id)
    2. 切换表情
       IF portrait.expression != expression:
           portrait.CrossFadeExpression(expression, duration=0.3s)
    3. 更新位置
       IF portrait.position != position:
           portrait.MoveTo(GetPositionX(position), duration=0.3s, easing=EaseInOut)
    4. 高亮当前说话者
       FOR EACH char IN currentScene.characters:
           IF char.character_id == currentSpeaker:
               char.SetAlpha(1.0)
           ELSE:
               char.SetAlpha(0.6)
```

### 5.2 对话框

- 底部半透明黑色遮罩（透明度70%），高度占屏幕25%
- 角色名显示在对话框左上角，颜色按角色品质/阵营区分
- 对话文本逐字显示，默认间隔50ms/字
- 点击屏幕任意位置：若文字未显示完→立即显示全部；若已显示完→推进下一条

### 5.3 文字推进模式

| 模式 | 说明 | 操作 |
|------|------|------|
| 手动 | 点击推进 | 点击屏幕/按空格 |
| 自动 | 自动推进 | 点击自动按钮切换，每条对话显示完后等待2秒自动推进 |
| 2倍速 | 文字显示速度加倍 | 点击2x按钮切换，逐字间隔变为25ms/字 |

```
ProcessDialogueDisplay(dialogue):
    1. 设置说话者名称
       SetSpeakerName(GetCharacterName(dialogue.speaker_id))
    2. 设置立绘表情和位置
       UpdateCharacterPortrait(dialogue.speaker_id, dialogue.portrait_expression, dialogue.position)
    3. 逐字显示文本
       delay = dialogue.text_delay > 0 ? dialogue.text_delay : DEFAULT_TEXT_DELAY  // 0.05s
       IF is_double_speed:
           delay *= 0.5
       FOR EACH char IN dialogue.text:
           AppendChar(char)
           Wait(delay)
           IF user_clicked:
               ShowFullText()
               BREAK
    4. 等待用户操作
       IF auto_mode:
           Wait(AUTO_ADVANCE_DELAY)  // 2秒
       ELSE:
           WaitForClick()
    5. 处理选择分支或推进
       IF dialogue.choices.Count > 0:
           ShowChoices(dialogue.choices)
       ELSE:
           AdvanceToDialogue(dialogue.next_dialogue_id)
```

### 5.4 历史回看

```
ShowDialogueHistory():

       - 每条显示：角色名 + 对话文本（缩略）
```

## 6. 选择分支系统

### 6.1 分支显示

```
ShowChoices(choices):
    1. 过滤不可见选项
       visible_choices = choices.Where(c => EvaluateChoiceCondition(c.condition_json))
    2. 显示选项面板
       - 2~3个选项按钮，从下到上排列
       - 按钮样式：半透明背景+白色文字+边框

       selected = WaitForChoice(visible_choices)
    4. 执行选择结果
       ApplyChoiceResult(selected)
    5. 跳转到对应对话
       AdvanceToDialogue(selected.next_dialogue_id)
```

### 6.4 主角属性选择（9个哲学问题）

9个问题对应原版《战争交响曲》的角色创建属性分配，每个问题3个选项分别影响不同属性成长方向：

| 问题序号 | 选项A | 选项B | 选项C |
|---------|-------|-------|-------|
| 1 | +STR | +SKL | +MAG |
| 2 | +STR | +LDR | +MAG |
| 3 | +SKL | +STR | +LDR |
| 4 | +MAG | +SKL | +STR |
| 5 | +LDR | +MAG | +SKL |
| 6 | +STR | +MAG | +LDR |
| 7 | +SKL | +LDR | +STR |
| 8 | +MAG | +STR | +SKL |
| 9 | +LDR | +SKL | +MAG |

- 每个选项+2点对应属性
- 9题总计+18点属性
- 属性影响：STR→物理攻击, SKL→暴击/命中, MAG→魔法攻击, LDR→带兵量/士气

## 8. 剧情跳过功能

### 8.1 跳过规则

| 条件 | 是否可跳过 |
|------|-----------|
| 首次观看的剧情 | 不可跳过 |
| 已观看完成的剧情 | 可跳过 |
| 主角创建流程（9问） | 不可跳过 |
| 羁绊剧情（已看过） | 可跳过 |

### 8.2 跳过流程

```
RequestSkipStory():
    1. 检查是否可跳过
       IF NOT PlayerHasWatched(current_chapter.chapter_id):
           RETURN  // 首次观看，不显示跳过按钮
    2. 显示确认弹窗
       "确定要跳过此剧情吗？"
       [确定] [取消]
    3. 玩家点击确定
       SkipToChapterEnd(current_chapter)
```

```
SkipToChapterEnd(chapter):
    1. 遍历该章节所有对话，收集所有选择分支的默认结果
       FOR EACH dialogue IN chapter.all_dialogues:
           IF dialogue.choices.Count > 0:
               // 选择第一个可见选项作为默认
               default_choice = dialogue.choices.First(c => EvaluateChoiceCondition(c.condition_json))
               ApplyChoiceResult(default_choice)
    2. 标记章节为已观看
       chapter.is_completed = true
       player.watched_chapters.Add(chapter.chapter_id)
    3. 关闭剧情界面
       CloseStoryUI()
    4. 触发章节完成回调
       OnStoryChapterCompleted(chapter.chapter_id)
```

## 9. 主角创建流程

### 9.1 流程步骤

```
PlayerCreationFlow():
    // Step 1: 选择性别
    ShowGenderSelection()
    gender = WaitForSelection()  // "male" / "female"

    // Step 2: 输入名字
    ShowNameInput(default_name=gender=="male" ? "雷昂" : "莉娜")
    name = WaitForNameInput()
    // 校验：2~8字符，不含敏感词
    IF NOT ValidateName(name):
        ShowError("名字不合法，请重新输入")
        GOTO Step 2

    // Step 3: 9个哲学问题
    attr_bonus = {STR:0, SKL:0, MAG:0, LDR:0}
    FOR i = 1 TO 9:
        question = TB_CreationQuestion[i]
        ShowQuestion(question.text, question.option_a, question.option_b, question.option_c)
        answer = WaitForAnswer()
        SWITCH answer:
            CASE "A": ApplyAttrBonus(attr_bonus, question.bonus_a)
            CASE "B": ApplyAttrBonus(attr_bonus, question.bonus_b)
            CASE "C": ApplyAttrBonus(attr_bonus, question.bonus_c)

    // Step 4: 确认
    ShowCreationConfirm(gender, name, attr_bonus)
    confirmed = WaitForConfirm()
    IF NOT confirmed:
        GOTO Step 1  // 重新开始
```

### 9.3 名字校验

```
ValidateName(name):
    // 长度校验
    IF name.Length < 2 OR name.Length > 8:
        RETURN false

    // 字符合法性：中文/英文/数字
    FOR EACH char IN name:
        IF NOT (IsChinese(char) OR IsLetter(char) OR IsDigit(char)):
            RETURN false

    // 敏感词过滤
    IF ContainsSensitiveWord(name):
        RETURN false

    RETURN true
```

## 10. 剧情播放主流程

### 10.1 完整播放流程

```
PlayStoryChapter(chapter_id):
    1. 获取章节数据 chapter = TB_StoryChapter[chapter_id]
    2. 初始化播放状态
       current_scene_index = 0
       current_dialogue_index = 0
    3. 加载第一个场景
       LoadScene(chapter.scenes[0])
    4. 进入对话循环
       WHILE current_scene_index < chapter.scenes.Count:
           scene = TB_StoryScene[chapter.scenes[current_scene_index]]
           PlayScene(scene)
           current_scene_index += 1
    5. 标记章节完成
       chapter.is_completed = true
       player.watched_chapters.Add(chapter_id)
       SavePlayerData(player)
    6. 触发完成回调
       OnStoryChapterCompleted(chapter_id)
```

```
PlayScene(scene):
    1. 加载背景图
       LoadBackground(scene.background_id)
    2. 播放BGM
       PlayBGM(scene.bgm_id, fade_in=true)
    3. 加载场景角色
       FOR EACH character IN scene.characters:
           ShowCharacter(character)
    4. 播放对话序列
       FOR EACH dialogue IN scene.dialogues:
           PlayDialogue(dialogue)
    5. 场景结束处理
       FadeOutBGM()
```

```
PlayDialogue(dialogue):
    1. 更新说话者立绘
       UpdateCharacterPortrait(dialogue.speaker_id, dialogue.portrait_expression, dialogue.position)
    2. 显示对话框和文本
       ShowDialogueBox(dialogue.speaker_id, dialogue.text)
    3. 播放语音（如有）
       IF dialogue.voice_clip != null:
           PlayVoice(dialogue.voice_clip)
    4. 播放CG（如有）
       IF dialogue.cg_id != null:
           PlayCG(dialogue.cg_id)
    5. 等待推进
       IF dialogue.choices.Count > 0:
           ShowChoices(dialogue.choices)
           // 选择后由ChoiceHandler处理跳转
       ELSE:
           WaitForAdvance()
           AdvanceToDialogue(dialogue.next_dialogue_id)
```
