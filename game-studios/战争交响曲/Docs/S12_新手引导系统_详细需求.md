## 1. 系统概述

### 1.1 核心定位

7天渐进式新手引导系统，让玩家从零基础逐步掌握"组队2小时，战斗5分钟"的编队策略深度。引导不是简单的UI指路，而是通过精心设计的节奏让玩家理解"为什么这样编队"。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 编队深度优先 | 引导重点放在编队策略而非战斗操作，让玩家理解编队是核心乐趣 |
| 渐进式解锁 | 7天内逐步解锁系统，避免信息过载 |
| 强制→弱引导过渡 | 前3章全强制引导，4-8章混合，9章后全弱引导 |
| 玩家自主性 | 弱引导阶段尊重玩家选择，仅提示不强制 |
| 可跳过设计 | 完成前2章强制引导后可跳过剩余引导 |

### 1.3 关键参数总览

| 参数 | 值 | 备注 |
|------|----|------|
| 引导总天数 | 7天 | 从首次登录开始计算 |
| 引导步骤总数 | ~50步 | 覆盖前5章核心系统 |
| 强制引导章节 | 第1-3章 | 灰色遮罩+不可跳过 |
| 混合引导章节 | 第4-8章 | 强制+弱引导混合 |
| 弱引导章节 | 第9章起 | 红点+气泡提示 |
| 好友上限 | 50人 | - |
| 可跳过条件 | 完成前2章 | 设置中提供跳过按钮 |

## 2. 数据结构定义（C# class）

### 2.1 TutorialState — 引导状态

```csharp
public class TutorialState
{
    public string current_step;                     // 当前引导步骤ID，如 "day1_ch1_move"
    public List<string> completed_steps;            // 已完成步骤ID列表
    public bool is_guide_active;                    // 引导是否激活中
    public GuideType guide_type;                    // 当前引导类型
    public int guide_day;                           // 当前引导天数(1-7)
    public long first_login_time;                   // 首次登录时间戳(Unix秒)
    public bool skip_requested;                     // 玩家是否请求跳过
    public int forced_chapters_completed;           // 已完成的强制引导章节数
    public Dictionary<string, bool> ui_first_opened;// UI首次打开记录 key=UI名 value=已打开
    public Dictionary<string, bool> condition_flags;// 条件标记 key=条件名 value=已满足
    public List<string> skipped_steps;              // 被跳过的步骤ID列表
}

public enum GuideType
{
    None,                                           // 无引导
    Forced,                                         // 强制引导（灰色遮罩+不可跳过）
    Weak,                                           // 弱引导（红点+气泡+可关闭）
    AutoFormation                                   // 自动编队推荐引导
}
```

### 2.2 TutorialStep — 引导步骤

```csharp
public class TutorialStep
{
    public string step_id;                          // 步骤ID，如 "day1_ch1_move"
    public int day;                                 // 所属天数(1-7)
    public int chapter;                             // 关联章节(0=不关联章节)
    public string trigger_condition;                // 触发条件ID，关联TB_TutorialTrigger
    public string target_ui_element;                // 目标UI元素路径，如 "Canvas/Home/BtnBattle"
    public ActionType action_type;                  // 玩家需要执行的动作类型
    public string description;                      // 引导描述文字
    public string next_step_id;                     // 下一步骤ID，""表示本日引导结束
    public bool is_forced;                          // 是否强制引导
    public float delay_seconds;                     // 触发后延迟执行秒数
    public string finger_start_element;             // 手指动画起点UI元素路径
    public string finger_end_element;               // 手指动画终点UI元素路径
    public HighlightShape highlight_shape;          // 高亮形状
    public string bubble_text;                      // 气泡文字内容
    public BubblePosition bubble_position;          // 气泡位置
    public string reward_id;                        // 完成奖励ID，""表示无奖励
    public int reward_count;                        // 奖励数量
}

public enum ActionType
{
    Click,                                          // 点击指定UI元素
    Drag,                                           // 拖拽操作
    LongPress,                                      // 长按操作
    Swipe,                                          // 滑动操作
    Wait,                                           // 等待（仅展示信息）
    SelectOption,                                   // 选择指定选项
    Confirm,                                        // 确认操作
    AutoComplete                                    // 自动完成（系统代操作）
}

public enum HighlightShape
{
    Rect,                                           // 矩形高亮
    Circle,                                         // 圆形高亮
    Ellipse,                                        // 椭圆高亮
    Polygon                                         // 多边形高亮（自定义形状）
}

public enum BubblePosition
{
    Top,                                            // 目标上方
    Bottom,                                         // 目标下方
    Left,                                           // 目标左侧
    Right,                                          // 目标右侧
    TopLeft,                                        // 目标左上方
    TopRight,                                       // 目标右上方
    BottomLeft,                                     // 目标左下方
    BottomRight                                     // 目标右下方
}
```

### 2.3 GuideHighlight — 高亮遮罩

```csharp
public class GuideHighlight
{
    public Rect target_rect;                        // 高亮目标区域（屏幕坐标）
    public HighlightShape highlight_shape;          // 高亮形状
    public float padding;                           // 高亮区域外扩像素数
    public FingerAnimationType finger_animation;    // 手指动画类型
    public string text_bubble;                      // 提示气泡文字
    public BubblePosition bubble_pos;               // 气泡位置
    public float mask_alpha;                        // 遮罩透明度(0.0~1.0)，强制引导=0.85，弱引导=0.0
    public bool click_through;                      // 是否允许点击穿透（弱引导=true）
    public float highlight_pulse_speed;             // 高亮区域脉冲动画速度(0=无脉冲)
    public float duration;                          // 高亮持续时间(秒)，0=直到玩家操作
}

public enum FingerAnimationType
{
    None,                                           // 无手指动画
    Point,                                          // 点击动画（手指按下→抬起）
    Drag,                                           // 拖拽动画（手指从起点拖到终点）
    Circle,                                         // 画圈动画（手指围绕目标画圈）
    Bounce,                                         // 弹跳动画（手指在目标上方弹跳）
    Swipe                                           // 滑动动画（手指从一侧滑到另一侧）
}
```

## 4. 7天引导计划详细设计

### 4.1 Day1（第1-2章）：基础操作与义务兵招募

**主题**：学会战斗基本操作，理解"移动→攻击→占领"循环

| 步骤 | step_id | 教学内容 | 引导类型 | 详细流程 |
|------|---------|---------|---------|---------|
| 1 | day1_ch1_move | 移动操作 | 强制 | 高亮移动按钮→手指点击动画→蓝色可达范围展示→等待玩家点击目标格子 |
| 2 | day1_ch1_select_tile | 选择移动目标 | 强制 | 高亮可达格子中的一个推荐格子→手指从移动按钮拖到目标格子→等待玩家点击 |
| 3 | day1_ch1_attack | 攻击操作 | 强制 | 小队移动到敌人附近后→高亮攻击按钮→手指点击动画→等待玩家点击 |
| 4 | day1_ch1_select_target | 选择攻击目标 | 强制 | 高亮攻击范围内的敌方小队→手指指向敌方→等待玩家点击 |
| 5 | day1_ch1_occupy | 占领据点 | 强制 | 战斗胜利后→高亮建筑格子→手指移动动画→等待玩家移动到建筑上 |
| 6 | day1_ch1_end_turn | 结束回合 | 强制 | 高亮结束回合按钮→手指点击动画→等待玩家点击 |
| 7 | day1_ch2_forced_battle | 强制战斗教学 | 强制 | 进入第2章→说明战力碾压可自动结算→高亮自动结算按钮 |
| 8 | day1_ch2_select_squad | 选择小队 | 强制 | 高亮小队选择→等待玩家选择 |
| 9 | day1_ch2_auto_resolve | 自动结算 | 强制 | 高亮自动结算按钮→等待点击→展示结算结果 |
| 10 | day1_ch2_recruit_conscript | 义务兵招募 | 强制 | 战后引导到招募界面→高亮义务兵招募按钮→等待点击→展示招募结果 |

**Day1完成奖励**：金币×500、义务兵×3

### 4.2 Day2（第3-4章）：阵型编辑与前排阻挡

**主题**：理解阵型的重要性，学会前排阻挡概念

| 步骤 | step_id | 教学内容 | 引导类型 | 详细流程 |
|------|---------|---------|---------|---------|
| 1 | day2_ch3_formation | 阵型编辑入口 | 强制 | 高亮阵型编辑按钮→手指点击动画→打开阵型编辑界面 |
| 2 | day2_ch3_front_row | 前排放重步 | 强制 | 说明前排阻挡概念→高亮重步单位→手指拖拽动画到前排格子→等待玩家拖拽 |
| 3 | day2_ch3_back_row | 后排放远程 | 强制 | 说明后排安全输出→高亮弓手→手指拖拽到后排→等待玩家拖拽 |

## 6. 引导触发条件系统

### 6.1 触发条件检查流程

```csharp
function CheckTutorialTriggers(eventType, eventParam):
    // 1. 获取所有匹配当前事件的触发器
    matchingTriggers = TB_TutorialTrigger.Where(t =>
        t.trigger_type == eventType &&
        (t.trigger_param == "" || t.trigger_param == eventParam)
    )

    // 2. 按优先级排序
    matchingTriggers.Sort((a, b) => a.priority - b.priority)

    // 3. 逐个检查
    foreach trigger in matchingTriggers:
        // 检查是否已触发过（一次性触发器）
        if trigger.one_time && tutorialState.completed_steps.Any(s =>
            TB_TutorialStep[s].trigger_condition == trigger.trigger_id):
            continue

        // 检查冷却时间
        if trigger.cooldown_seconds > 0:
            lastTriggerTime = GetLastTriggerTime(trigger.trigger_id)
            if Time.time - lastTriggerTime < trigger.cooldown_seconds:
                continue

        // 检查前置步骤是否完成
        step = TB_TutorialStep.FirstOrDefault(s =>
            s.trigger_condition == trigger.trigger_id)
        if step == null:
            continue

        // 检查前置步骤
        if not ArePrerequisitesMet(step):
            continue

        // 触发引导
        TriggerGuideStep(step)
        break  // 一次只触发一个引导
```

### 6.2 各触发类型详细说明

#### 6.2.1 章节开始触发

```
触发时机：玩家进入新章节，场景加载完成后
触发参数：章节号
检查逻辑：
1. 当前章节 == trigger_param
2. 该章节对应的引导步骤未完成
3. 前置章节引导已完成
示例：chapter_start_1 → 进入第1章时触发day1_ch1_move
```

#### 6.2.2 UI首次打开触发

```
触发时机：玩家首次打开指定UI界面
触发参数：UI名称
检查逻辑：
1. tutorialState.ui_first_opened[ui_name] == false
2. 标记ui_first_opened[ui_name] = true
3. 查找关联的引导步骤
示例：ui_first_open_formation → 首次打开阵型编辑界面时触发
```

#### 6.2.3 条件满足触发

```
触发时机：特定游戏条件满足时
触发参数：条件名
支持的条件：
- first_obtain_ssr：首次获得SSR品质单位
- first_promotion：首次完成转职
- first_tech_research：首次完成科技研究
- first_artifact_equip：首次装备神器
- enemy_low_morale：战斗中敌方士气≤30
- squad_defeated：己方小队被击败
检查逻辑：
1. tutorialState.condition_flags[condition_name] == false
2. 游戏事件触发时设置condition_flags[condition_name] = true
3. 查找关联的引导步骤
```

#### 6.2.4 时间延迟触发

```
触发时机：指定延迟时间后
触发参数：延迟秒数
检查逻辑：
1. 上一步完成时间 + delay_seconds <= 当前时间
2. 玩家当前不在战斗中
3. 玩家当前不在其他引导中
示例：day1_ch1_move完成后延迟0.5秒触发day1_ch1_select_tile
```

#### 6.2.5 步骤完成触发

```
触发时机：上一步引导步骤完成时
触发参数：上一步step_id
检查逻辑：
1. tutorialState.completed_steps.Contains(trigger_param)
2. 当前无其他引导激活
3. 查找next_step_id对应的步骤
这是最常见的触发方式，构成引导链
```

### 6.3 触发条件冲突处理

```
优先级规则（从高到低）：
1. StepComplete（步骤完成触发）→ 最高优先级，保证引导链连续性
2. ChapterStart（章节开始触发）→ 章节级引导入口
3. ConditionMet（条件满足触发）→ 特殊事件引导
4. UIFirstOpen（UI首次打开）→ 界面级引导
5. BattleEnd（战斗结束触发）→ 战后引导
6. TimeDelay（时间延迟触发）→ 最低优先级

冲突处理：
- 同一时刻多个触发器满足条件时，按优先级执行最高优先级的
- 同一优先级按priority字段升序排列
- 一次只执行一个引导步骤
- 引导执行期间，新的触发器排队等待
```

### 7.4 详细评分算法

```csharp
function CalcFormationScore(squad, enemyInfo):
    score = 0.0f

    // 1. 前排阻挡评分 (权重30%)
    frontBlockers = squad.Units.Where(u =>
        u.IsAlive && u.IsHeavyUnit && u.FormationRow == 0).Count()
    if frontBlockers >= 2:
        score += 30.0f
    elif frontBlockers == 1:
        score += 20.0f
    else:
        score += 0.0f  // 无前排阻挡，严重扣分

    // 2. 攻击顺序分散评分 (权重25%)
    priorities = squad.Units.Where(u => u.IsAlive)
        .Select(u => u.AttackPriority).Distinct().Count()
    if priorities >= 3:
        score += 25.0f
    elif priorities == 2:
        score += 18.0f
    else:
        score += 5.0f  // 全队同序，集火溢出风险

    // 3. 穿甲覆盖评分 (权重25%)
    hasArmorPen = squad.Units.Any(u =>
        u.IsAlive && u.ArmorPenetration >= 0.5f)
    hasMagicPen = squad.Units.Any(u =>
        u.IsAlive && u.ArmorPenetration >= 1.0f)
    heavyArmorEnemyRatio = enemyInfo.HeavyArmorCount /
        (float)enemyInfo.TotalEnemyCount

    if heavyArmorEnemyRatio > 0.5f:
        // 重甲敌人多
        if hasMagicPen: score += 25.0f
        elif hasArmorPen: score += 18.0f
        else: score += 5.0f
    else:
        // 轻甲敌人多
        if hasArmorPen || hasMagicPen: score += 20.0f
        else: score += 15.0f

    // 4. 治疗配置评分 (权重20%)
    healers = squad.Units.Where(u =>
        u.IsAlive && u.CanHeal).Count()
    if healers >= 1:
        score += 20.0f
    else:
        score += 5.0f

    return score
```

## 8. 跳过引导功能

### 8.1 跳过规则

| 条件 | 是否可跳过 | 说明 |
|------|-----------|------|
| 未完成第1章 | ❌ | 核心操作必须学会 |
| 未完成第2章 | ❌ | 基础战斗必须掌握 |
| 已完成第2章 | ✅ | 设置中出现跳过按钮 |
| 已跳过的步骤 | - | 记录到skipped_steps，不再触发 |

### 8.2 跳过流程

```
1. 玩家打开设置界面
2. 如果forced_chapters_completed >= 2：
   a. 显示"跳过新手引导"按钮
   b. 点击后弹出确认对话框：
      "跳过后将无法重新体验引导内容，是否确认？"
   c. 确认后：
      - tutorialState.skip_requested = true
      - tutorialState.is_guide_active = false
      - 将所有未完成步骤加入skipped_steps
      - 关闭当前所有引导UI（遮罩/手指/气泡）
      - 保存状态
3. 如果forced_chapters_completed < 2：
   a. "跳过新手引导"按钮灰化
   b. 显示提示"完成前2章后可跳过"
```

### 8.3 跳过后处理

```
1. 清除所有引导遮罩和手指动画
2. 清除所有红点提示
3. 关闭所有引导气泡
4. is_guide_active = false
5. 不再触发任何引导步骤
6. 弱引导系统仍可正常工作（红点提示独立于引导系统）
```

### 9.2 引导步骤完成流程

```csharp
function OnGuideStepComplete(stepId):
    // 1. 记录完成
    if not state.completed_steps.Contains(stepId):
        state.completed_steps.Add(stepId)

    // 2. 发放奖励
    step = TB_TutorialStep[stepId]
    if step.reward_id != "":
        GrantReward(step.reward_id, step.reward_count)

    // 3. 隐藏当前引导UI
    HideCurrentGuide()

    // 4. 检查下一步
    if step.next_step_id != "":
        nextStep = TB_TutorialStep[step.next_step_id]
        // 如果有延迟，设置定时器
        if nextStep.delay_seconds > 0:
            StartCoroutine(DelayedTrigger(nextStep, nextStep.delay_seconds))
        else:
            TriggerGuideStep(nextStep)
    else:
        // 本日引导结束
        state.is_guide_active = false
        SaveState()

    // 5. 保存状态
    SaveState()
```

## 10. 边界条件与测试用例

### 10.1 边界条件

| 场景 | 处理方式 |
|------|---------|
| 引导步骤中玩家强制退出游戏 | 下次登录恢复到最近完成的步骤，未完成步骤重新触发 |
| 引导步骤中目标UI元素不存在 | 跳过该步骤，记录错误日志，继续下一步 |
| 多个触发器同时满足 | 按优先级执行，一次只执行一个 |
| 引导进行中触发新引导 | 排队等待，当前引导完成后执行 |
| 玩家在强制引导中点击非目标区域 | 点击无效，手指动画加速提示（0.5秒后加速1次） |
| 弱引导气泡自动关闭后玩家想重新查看 | 在设置中提供"查看引导提示"按钮，重新展示当日弱引导 |
| 跨天登录（如第1天登录后第5天才登录） | 按实际天数推进引导，不补中间天数的引导 |
| 自动编队推荐时可用单位不足 | 提示"单位不足，建议先招募"，展示最优可用方案 |
| 网络延迟导致引导步骤不同步 | 引导状态本地存储，不依赖网络 |
| 引导步骤配置错误（next_step_id指向不存在的步骤） | 跳过该步骤，记录错误日志，尝试查找同天下一个未完成步骤 |
