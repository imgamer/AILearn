## 1. 系统概述

资源与经济系统管理游戏内所有货币、转职资源、招募机制和市场交易。系统包含6种转职资源、2种货币（金币+钻石）、招募系统（义务兵+佣兵）、市场刷新机制和竞技场。

**核心设计原则：**
- 免费玩家可通过时间积累获得所有资源，不设付费墙
- 付费加速但不破坏平衡，钻石主要用于便利性和外观
- 经济系统与战斗、养成系统紧密耦合
- 市场刷新机制提供每日目标感

**资源总览：**

| 类型 | 资源 | 用途 | 获取难度 |
|------|------|------|---------|
| 基础货币 | 金币 | 招募、购买、投资 | 低 |
| 高级货币 | 钻石 | 抽卡、体力、刷新 | 高 |
| 转职资源 | 铁锭 | 重甲系转职 | 中 |
| 转职资源 | 宝石 | 法师系转职 | 中 |
| 转职资源 | 马匹 | 骑兵系转职 | 中 |
| 转职资源 | 太阳石 | 高级转职(龙系/特殊) | 高 |
| 养成资源 | 亲和之书 | 亲和升级 | 中 |
| 养成资源 | 天赋之书 | 天赋升级 | 中 |
| 养成资源 | 指挥书 | 指挥官经验 | 中 |

## 2. 数据结构

### 2.1 PlayerResources - 玩家资源

```csharp
[Serializable]
public class PlayerResources
{
    public int gold;                // 金币
    public int diamond;             // 钻石

    public int iron;                // 铁锭 - 重甲系转职
    public int gem;                 // 宝石 - 法师系转职
    public int horse;               // 马匹 - 骑兵系转职
    public int sunstone;            // 太阳石 - 高级转职

    public int affinity_book;       // 亲和之书
    public int talent_book;         // 天赋之书
    public int cp_book;             // 指挥书(Command Point Book)

    public Dictionary<string, int> daily_acquisition;  // 每日获取记录
    public string last_daily_reset;                    // 上次每日重置日期

    public bool TrySpend(string resourceType, int amount)
    {
        int current = GetResource(resourceType);
        if (current < amount) return false;
        SetResource(resourceType, current - amount);
        return true;
    }

    public int GetResource(string type)
    {
        return type switch
        {
            "gold" => gold,
            "diamond" => diamond,
            "iron" => iron,
            "gem" => gem,
            "horse" => horse,
            "sunstone" => sunstone,
            "affinity_book" => affinity_book,
            "talent_book" => talent_book,
            "cp_book" => cp_book,
            _ => 0
        };
    }

    public void SetResource(string type, int value)
    {
        switch (type)
        {
            case "gold": gold = value; break;
            case "diamond": diamond = value; break;
            case "iron": iron = value; break;
            case "gem": gem = value; break;
            case "horse": horse = value; break;
            case "sunstone": sunstone = value; break;
            case "affinity_book": affinity_book = value; break;
            case "talent_book": talent_book = value; break;
            case "cp_book": cp_book = value; break;
        }
    }
}
```

## 7. 招募系统

### 7.1 义务兵

**特征：**
- 市场前4个栏位
- 固定价格：100金币
- 品质范围：普通(70%) / 精良(25%) / 稀有(5%)
- 属性随机（低区间）
- 无自带装备和天赋
- 无限量供应（每次刷新4个）

**义务兵生成逻辑：**

```csharp
public class ConscriptGenerator
{
    public static RecruitUnit Generate(int slotId)
    {
        var unit = new RecruitUnit();
        unit.unit_id = System.Guid.NewGuid().ToString();
        unit.is_from_mercenary = false;
        unit.price = 100;

        // 按权重随机品质
        unit.quality = WeightedRandom(UnitQuality.Common, 70,
                                       UnitQuality.Uncommon, 25,
                                       UnitQuality.Rare, 5);

        // 按权重随机职业
        unit.class_id = WeightedRandomClass();

        // 按品质生成属性
        TB_RecruitPool pool = ConfigManager.GetRecruitPool(unit.quality);
        float minMul = pool.stat_multiplier_min;
        float maxMul = pool.stat_multiplier_max;
        float mul = Random.Range(minMul, maxMul);

        unit.base_stats_json = GenerateStatsJson(unit.class_id, mul);
        unit.talents = new List<int>();
        unit.equipment_id = "";

        // 生成名称
        unit.name = GenerateRandomName(unit.class_id, unit.quality);

        return unit;
    }
}
```

### 7.2 佣兵

**特征：**
- 市场后续8个栏位
- 价格范围：500~5000金币（按品质和属性）
- 品质范围：普通(10%) / 精良(30%) / 稀有(35%) / 史诗(20%) / 传说(5%)
- 属性随机（高区间）
- 可能自带装备（按品质概率）
- 可能自带天赋（1~3个）

**佣兵价格计算：**

```csharp
public class MercenaryPricer
{
    public static int CalculatePrice(RecruitUnit unit)
    {
        float basePrice = unit.quality switch
        {
            UnitQuality.Common => 500,
            UnitQuality.Uncommon => 1000,
            UnitQuality.Rare => 2000,
            UnitQuality.Epic => 3500,
            UnitQuality.Legendary => 5000,
            _ => 500
        };

        // 天赋加价
        basePrice += unit.talents.Count * 200;

        // 自带装备加价
        if (!string.IsNullOrEmpty(unit.equipment_id))
        {
            basePrice *= 1.2f;
        }

        // 属性浮动（±10%）
        float fluctuation = Random.Range(0.9f, 1.1f);
        return (int)(basePrice * fluctuation);
    }
}
```

### 7.3 市场刷新机制

```csharp
public class MarketManager
{
    private List<MarketSlot> currentSlots = new List<MarketSlot>();
    private long lastRefreshTime;
    private const int REFRESH_INTERVAL_HOURS = 8;
    private const int PREMIUM_REFRESH_COST = 50;

    public void CheckAutoRefresh()
    {
        long now = GetCurrentUnixTime();
        long interval = REFRESH_INTERVAL_HOURS * 3600;

        if (now - lastRefreshTime >= interval)
        {
            RefreshMarket(isAutoRefresh: true);
            lastRefreshTime = now;
        }
    }

    public bool TryPremiumRefresh(PlayerResources resources)
    {
        if (!resources.TrySpend("diamond", PREMIUM_REFRESH_COST))
            return false;

        RefreshMarket(isAutoRefresh: false);
        return true;
    }

    private void RefreshMarket(bool isAutoRefresh)
    {
        currentSlots.Clear();

        TB_MarketConfig config = ConfigManager.GetMarketConfig();
        int conscriptSlots = config.conscript_slots;
        int mercenarySlots = config.mercenary_slots;
        int equipmentSlots = config.equipment_slots;
        int artifactSlots = config.artifact_slots;

        // 贸易特使科技加成
        if (TechManager.IsUnlocked("trade_envoy"))
        {
            mercenarySlots += config.trade_envoy_extra_slots;
        }

        int slotIndex = 0;

        // 生成义务兵
        for (int i = 0; i < conscriptSlots; i++)
        {
            var unit = ConscriptGenerator.Generate(slotIndex);
            currentSlots.Add(new MarketSlot
            {
                slot_id = slotIndex++,
                item_type = MarketSlot.MarketItemType.Conscript,
                item_id = unit.unit_id,
                price = unit.price,
                is_purchased = false,
                refresh_time = GetCurrentUnixTime()
            });
            RecruitPoolManager.StoreUnit(unit);
        }

        // 生成佣兵
        for (int i = 0; i < mercenarySlots; i++)
        {
            var unit = MercenaryGenerator.Generate(slotIndex);
            // 贸易特使品质提升
            if (TechManager.IsUnlocked("trade_envoy"))
            {
                unit = MercenaryGenerator.ApplyQualityBoost(unit);
            }
            currentSlots.Add(new MarketSlot
            {
                slot_id = slotIndex++,
                item_type = MarketSlot.MarketItemType.Mercenary,
                item_id = unit.unit_id,
                price = MercenaryPricer.CalculatePrice(unit),
                is_purchased = false,
                refresh_time = GetCurrentUnixTime()
            });
            RecruitPoolManager.StoreUnit(unit);
        }

        // 生成装备
        for (int i = 0; i < equipmentSlots; i++)
        {
            var equip = EquipmentGenerator.Generate();
            currentSlots.Add(new MarketSlot
            {
                slot_id = slotIndex++,
                item_type = MarketSlot.MarketItemType.Equipment,
                item_id = equip.equipment_id,
                price = EquipmentPricer.CalculatePrice(equip),
                is_purchased = false,
                refresh_time = GetCurrentUnixTime()
            });
        }

        // 生成神器
        for (int i = 0; i < artifactSlots; i++)
        {
            var artifact = ArtifactGenerator.Generate();
            if (artifact != null)
            {
                currentSlots.Add(new MarketSlot
                {
                    slot_id = slotIndex++,
                    item_type = MarketSlot.MarketItemType.Artifact,
                    item_id = artifact.artifact_id,
                    price = ArtifactPricer.CalculatePrice(artifact),
                    is_purchased = false,
                    refresh_time = GetCurrentUnixTime()
                });
            }
        }
    }

    public bool TryPurchase(PlayerResources resources, int slotId)
    {
        var slot = currentSlots.Find(s => s.slot_id == slotId);
        if (slot == null || slot.is_purchased) return false;

        if (!resources.TrySpend("gold", slot.price)) return false;

        slot.is_purchased = true;
        return true;
    }
}
```

public class ArenaScoring
{
    public static int CalculatePointsChange(bool isVictory, int opponentRank, int myRank)
    {
        if (isVictory)
        {
            int rankDiff = myRank - opponentRank;
            if (rankDiff > 0)
                return 20 + Math.Min(rankDiff / 10, 30); // 击败排名更高的对手获得更多积分
            else
                return Math.Max(10, 20 + rankDiff / 10);  // 击败排名更低的对手获得较少积分
        }
        else
        {
            return -10; // 失败固定扣10分
        }
    }
}

### 9.3 经济平衡结论

- 免费玩家金币产出约75000~105000，消耗约54000~189000
- 金币处于**紧平衡**状态，需要玩家合理规划支出
- 太阳石极度稀缺（30天仅3~9个），高级转职需要长期积累
- 钻石免费获取有限（450~750），仅够2~4次单抽或少量体力购买

**设计原则：**
- 付费不提供免费玩家无法获取的内容
- 付费主要节省时间，不提供数值碾压
- 太阳石不直接售卖（保持稀缺性）

## 10. 边界条件与测试用例

### 测试用例 1：金币不足购买

**场景：** 玩家金币50，尝试购买价格100的义务兵  
**预期：** 购买失败，金币不变，提示"金币不足"  
**验证：** `resources.gold == 50`，`TrySpend` 返回 false

### 测试用例 2：资源达到上限

**场景：** 铁锭当前9998，矿洞产出5  
**预期：** 铁锭设为9999（max_stack），超出部分丢失  
**验证：** `resources.iron == 9999`

### 测试用例 3：市场自动刷新

**场景：** 距上次刷新已过8小时1分钟  
**预期：** 市场自动刷新，所有栏位重新生成  
**验证：** `currentSlots` 全部为新数据，`lastRefreshTime` 更新

### 测试用例 4：钻石手动刷新市场

**场景：** 玩家钻石50，手动刷新市场  
**预期：** 钻石变为0，市场刷新  
**验证：** `resources.diamond == 0`，`currentSlots` 为新数据

### 测试用例 5：钻石不足手动刷新

**场景：** 玩家钻石30，尝试手动刷新（需50）  
**预期：** 刷新失败，钻石不变  
**验证：** `resources.diamond == 30`

### 测试用例 6：竞技场每日次数耗尽

**场景：** 今日已挑战3次，尝试第4次挑战  
**预期：** 挑战失败，提示"今日挑战次数已用完"  
**验证：** `arenaRecord.daily_challenges == 3`，无法发起挑战

### 测试用例 7：招募池品质权重验证

**场景：** 生成1000个义务兵，统计品质分布  
**预期：** 普通约700，精良约250，稀有约50，无史诗和传说  
**验证：** 品质分布与配置权重偏差不超过5%

### 测试用例 8：贸易特使科技效果

**场景：** 解锁贸易特使科技后刷新市场  
**预期：** 佣兵栏位从8增至10，20%概率品质提升  
**验证：** `currentSlots` 中佣兵栏位数为10，多次刷新统计品质分布有提升

### 测试用例 9：竞技场赛季结算

**场景：** 赛季结束，玩家排名25  
**预期：** 获得3000金币+7经验书+3太阳石+60钻石，积分重置  
**验证：** 资源增加正确，`season_points` 重置

### 测试用例 10：出售单位回收金币

**场景：** 出售价格为2000金币的佣兵  
**预期：** 回收1000金币（50%），单位从队伍移除  
**验证：** `resources.gold += 1000`，单位不在队伍列表中
