## 1. 系统概述

### 1.1 核心定位

士气（Morale）是连接 **战斗系统**、**羁绊系统**、**投降系统** 的核心隐藏变量。它影响单位的攻击力、行动次数，并决定敌方是否可能投降。士气系统让战斗不仅仅是数值对撞，而是融入了心理博弈和战术选择。

### 1.2 系统关系图

```
┌──────────┐    影响     ┌──────────┐
│  羁绊系统 │ ────────→ │  士气系统 │
└──────────┘           └────┬─────┘
                            │
        ┌───────────────────┼───────────────────┐
        ↓                   ↓                   ↓
  ┌──────────┐       ┌──────────┐        ┌──────────┐
  │ 攻击修正 │       │ 再动判定 │        │ 投降判定 │
  │ ATK×1.15 │       │ 20%几率 │        │ 概率公式 │
  │ ATK×0.75 │       │ 额外行动 │        │          │
  └──────────┘       └──────────┘        └──────────┘
```

### 1.3 士气值范围

| 属性 | 值 |
|------|-----|
| 初始值 | 50 |
| 最大值 | 100 |
| 最小值 | 0 |
| 高士气阈值 | ≥ 80 |
| 低士气阈值 | ≤ 20 |

## 2. 数据结构

### 2.1 MoraleState - 士气状态

```csharp
/// <summary>
/// 士气状态，每个小队维护一个实例
/// </summary>
[Serializable]
public class MoraleState
{
    public float current_value;                    // 当前士气值 [0, 100]
    public List<MoraleModifier> modifiers_list;    // 当前生效的修正列表
    public string last_change_reason;              // 最近一次士气变化原因（用于UI显示）
    public float last_change_amount;               // 最近一次变化量

    public bool IsHighMorale => current_value >= 80f;
    public bool IsLowMorale => current_value <= 20f;
    public bool IsSurrenderThreshold => current_value <= 30f;

    public MoraleState()
    {
        current_value = 50f; // TB_MoraleConfig.initial_value
        modifiers_list = new List<MoraleModifier>();
        last_change_reason = null;
        last_change_amount = 0f;
    }

    public float ApplyChange(float delta, string reason)
    {
        float oldValue = current_value;
        current_value = Mathf.Clamp(current_value + delta, 0f, 100f);
        last_change_reason = reason;
        last_change_amount = current_value - oldValue;
        return current_value;
    }
}

[Serializable]
public class MoraleModifier
{
    public string modifier_id;       // 修正源ID
    public string source_name;       // 来源名称（如"羁绊-刘备关羽"）
    public float value;              // 修正值（正/负）
    public int remaining_turns;      // 剩余回合数，-1表示永久
    public bool is_active;           // 是否生效
}
```

### 2.2 SurrenderContext - 投降判定上下文

```csharp
/// <summary>
/// 投降判定上下文，每次劝降时构建
/// </summary>
public class SurrenderContext
{
    public float attacker_threat;         // 攻击方威胁等级
    public float defender_threat;         // 防守方威胁等级
    public float defender_morale;         // 防守方当前士气
    public int surround_count;            // 包围我方小队数量
    public float tech_bonus;              // 科技加成（百分比，如0.10表示10%）
    public float leader_talent_bonus;     // 队长天赋加成（百分比）

    // 计算结果（由 SurrenderCalculator 填充）
    public float surrender_prob;          // 最终投降概率
    public float threat_ratio_component;  // 威胁差贡献
    public float morale_component;        // 士气贡献
    public float surround_component;      // 包围贡献
}
```

## 4. 士气变化事件完整列表

### 4.1 战斗事件

#### 击败敌方小队 (+10, Team)

- **触发条件**：敌方一个小队所有单位被击败
- **影响范围**：击败方所有小队
- **叠加规则**：同一回合击败多个小队，士气叠加（击败2队 = +20）
- **边界**：士气不超过上限100

#### 小队损失单位 (-15, Self)

- **触发条件**：小队内任一非队长单位HP降为0
- **影响范围**：仅损失单位的小队
- **叠加规则**：同一回合损失多个单位，士气叠加（损失2单位 = -30）
- **边界**：士气不低于下限0

#### 队长倒下 (-30, Self)

- **触发条件**：小队队长HP降为0
- **影响范围**：仅队长所在小队
- **叠加规则**：不叠加（队长只有一个）
- **特殊**：此事件后该小队士气通常跌至极低，极易触发投降

### 4.2 羁绊事件

#### 羁绊英雄同场 (+5/回合, Team)

- **触发条件**：每回合开始时，检查羁绊双方是否都在战场上且存活
- **影响范围**：羁绊相关的所有小队
- **叠加规则**：多个羁绊独立计算（A-B羁绊 +5, A-C羁绊 +5 = +10）
- **边界**：士气不超过上限100

### 4.3 包围事件

#### 包围敌人 (+5/+10/+15, Self)

- **触发条件**：每回合开始时，统计与目标敌方小队相邻（6方向）的不同我方小队数量
- **影响范围**：参与包围的我方小队
- **叠加规则**：按包围数量取最大值，不叠加

| 包围我方小队数 | 士气加成 |
|----------------|----------|
| 1 | 0（不算包围） |
| 2 | +5 |
| 3 | +10 |
| 4+ | +15（上限） |

### 4.4 天气事件

#### 天气有利 (+3/回合, Team)

- **触发条件**：每回合开始时，当前天气对该阵营有利
- **影响范围**：有利阵营的所有小队
- **有利天气判定**：
  - 雨天 → 水系阵营有利
  - 晴天 → 火系阵营有利
  - 大风 → 风系阵营有利

#### 天气不利 (-3/回合, Team)

- **触发条件**：每回合开始时，当前天气对该阵营不利
- **影响范围**：不利阵营的所有小队

#### 热浪无防护 (-10/回合, Self)

- **触发条件**：每回合开始时，热浪天气 + 小队无防暑装备
- **影响范围**：无防暑装备的小队
- **防暑装备**：遗物标签含 "heat_resistant" 的遗物

### 4.5 神力事件

#### 神力-恐惧 (-30, All敌方)

- **触发条件**：释放"恐惧"神力技能
- **影响范围**：目标区域内所有敌方小队
- **叠加规则**：不叠加，同一回合多次恐惧只取最大值
- **抗性**：部分单位/遗物可提供恐惧抗性（减免百分比）

## 5. 士气效果

### 5.1 高士气效果 (≥80)

```csharp
public class MoraleEffectProcessor
{
    private TB_MoraleConfig config;

    public float GetAttackMultiplier(MoraleState morale)
    {
        if (morale.IsHighMorale)
            return 1f + config.high_atk_bonus;  // 1.15
        if (morale.IsLowMorale)
            return 1f - config.low_atk_penalty;  // 0.75
        return 1f;
    }

    public bool CheckRetrigger(MoraleState morale)
    {
        if (!morale.IsHighMorale)
            return false;

        float roll = UnityEngine.Random.Range(0f, 100f);
        return roll < config.retrigger_chance * 100f;  // 20%几率
    }
}
```

**高士气效果汇总：**

| 效果 | 数值 | 说明 |
|------|------|------|
| 攻击加成 | ATK × 1.15 | 所有伤害+15% |
| 再动几率 | 20% | 每次行动结束后判定，通过则额外行动一次 |

**再动判定流程：**

```
单位行动结束
    ↓
士气 ≥ 80? → 否 → 结束
    ↓ 是
Roll(0~100) < 20? → 否 → 结束
    ↓ 是
触发再动 → 单位获得额外行动 → 行动结束 → 再次判定（可连续再动，无上限）
```

**注意：** 再动可以连续触发。理论上士气≥80时每次行动后都有20%几率再动，但连续触发的概率递减（20% → 4% → 0.8% → ...）。

### 5.2 低士气效果 (≤20)

| 效果 | 数值 | 说明 |
|------|------|------|
| 攻击惩罚 | ATK × 0.75 | 所有伤害-25% |
| 无再动 | - | 不触发再动判定 |

### 5.3 中间士气 (21~79)

无额外效果，攻击力为基准值。

## 6. 投降概率公式

### 6.1 完整公式

```
surrender_prob = MIN(80, MAX(0,
    (attacker_threat - defender_threat) / defender_threat * 50
    + (50 - defender_morale) / 50 * 30
    + surround_bonus
    + tech_bonus
    + leader_talent_bonus
))
```

### 6.2 公式拆解

| 分项 | 公式 | 范围 | 说明 |
|------|------|------|------|
| 威胁差贡献 | `(atk_threat - def_threat) / def_threat * 50` | (-∞, +∞) | 攻击方越强，概率越高 |
| 士气贡献 | `(50 - def_morale) / 50 * 30` | [-30, +30] | 士气越低，概率越高 |
| 包围贡献 | `surround_bonus` | [0, 15] | 2队=5, 3队=10, 4+队=15 |
| 科技加成 | `tech_bonus` | [0, +∞) | 科技研究提供的百分比加成 |
| 队长天赋加成 | `leader_talent_bonus` | [0, +∞) | 队长天赋提供的百分比加成 |

### 6.3 包围贡献映射

| 包围我方小队数 | surround_bonus |
|----------------|----------------|
| 0~1 | 0 |
| 2 | 5 |
| 3 | 10 |
| 4+ | 15 |

### 6.4 投降概率计算器

```csharp
public class SurrenderCalculator
{
    public static SurrenderContext Calculate(SurrenderContext ctx)
    {
        // 1. 威胁差贡献
        if (ctx.defender_threat <= 0f)
            ctx.defender_threat = 0.01f; // 防除零

        ctx.threat_ratio_component =
            (ctx.attacker_threat - ctx.defender_threat) / ctx.defender_threat * 50f;

        // 2. 士气贡献
        ctx.morale_component = (50f - ctx.defender_morale) / 50f * 30f;

        // 3. 包围贡献
        ctx.surround_component = ctx.surround_count switch
        {
            >= 4 => 15f,
            3    => 10f,
            2    => 5f,
            _    => 0f
        };

        // 4. 汇总
        float raw = ctx.threat_ratio_component
                   + ctx.morale_component
                   + ctx.surround_component
                   + ctx.tech_bonus
                   + ctx.leader_talent_bonus;

        ctx.surrender_prob = Math.Min(80f, Math.Max(0f, raw));

        return ctx;
    }
}
```

### 6.5 计算示例

**示例1：势均力敌，士气正常，无包围**

| 参数 | 值 |
|------|-----|
| attacker_threat | 100 |
| defender_threat | 100 |
| defender_morale | 50 |
| surround_count | 0 |
| tech_bonus | 0 |
| leader_talent_bonus | 0 |

```
threat_ratio = (100-100)/100 * 50 = 0
morale_comp  = (50-50)/50 * 30 = 0
surround     = 0
surrender_prob = MIN(80, MAX(0, 0+0+0+0+0)) = 0%
```

## 7. 投降流程状态机

### 7.1 状态机定义

```
                    ┌──────────────────────────────────────────┐
                    ↓                                          │
┌─────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌─┴──────┐
│ 检查士气 │──→│ 弹出劝降 │──→│ 计算概率 │──→│ Roll判定 │──→│ 结果处理│
└─────────┘   └──────────┘   └──────────┘   └──────────┘   └────────┘
     │                                              │              │
     │ 士气>30                                      │              │
     ↓                                              ↓              ↓
  跳过劝降                                    成功            失败
  正常攻击                                    ↓              ↓
                                           投降成功流程    投降失败流程
```

### 7.2 状态枚举

```csharp
public enum SurrenderPhase
{
    None,              // 未进入投降流程
    CheckMorale,       // 检查敌方士气
    ShowOption,        // 弹出劝降选项
    CalculateProb,     // 计算概率
    RollJudge,         // Roll判定
    ProcessSuccess,    // 处理成功
    ProcessFailure     // 处理失败
}
```

### 7.3 详细流程

```csharp
public class SurrenderFlowManager
{
    private SurrenderPhase currentPhase = SurrenderPhase.None;
    private SurrenderContext context;

    public SurrenderPhase TryInitiateSurrender(
        SquadData attacker,
        SquadData defender,
        MoraleState defenderMorale,
        BattleContext battleCtx)
    {
        currentPhase = SurrenderPhase.CheckMorale;

        // Step 1: 检查敌方士气 ≤ 30
        if (defenderMorale.current_value > 30f)
        {
            currentPhase = SurrenderPhase.None;
            return SurrenderPhase.None; // 士气过高，不可劝降
        }

        // Step 2: 弹出劝降选项
        currentPhase = SurrenderPhase.ShowOption;
        // UI层显示劝降对话框，玩家选择"劝降"或"继续攻击"
        return currentPhase;
    }

    public SurrenderPhase PlayerChooseSurrender(bool chooseSurrender)
    {
        if (!chooseSurrender)
        {
            currentPhase = SurrenderPhase.None;
            return SurrenderPhase.None; // 玩家选择继续攻击
        }

        // Step 3: 计算概率
        currentPhase = SurrenderPhase.CalculateProb;
        SurrenderCalculator.Calculate(context);

        // Step 4: Roll判定
        currentPhase = SurrenderPhase.RollJudge;
        float roll = UnityEngine.Random.Range(0f, 100f);
        bool success = roll < context.surrender_prob;

        if (success)
        {
            currentPhase = SurrenderPhase.ProcessSuccess;
            ProcessSurrenderSuccess();
        }
        else
        {
            currentPhase = SurrenderPhase.ProcessFailure;
            ProcessSurrenderFailure();
        }

        return currentPhase;
    }

    private void ProcessSurrenderSuccess()
    {
        // 敌方退出战斗
        // 我方获得1.5倍经验
        // 30%概率获得佣兵
        float mercRoll = UnityEngine.Random.Range(0f, 100f);
        bool getMercenary = mercRoll < 30f;
        // ... 触发对应事件
    }

    private void ProcessSurrenderFailure()
    {
        // 我方少1次攻击
        // 我方必定后手（下回合行动顺序最后）
        // ... 触发对应事件
    }
}
```

### 7.4 投降成功效果

| 效果 | 说明 |
|------|------|
| 敌方退出战斗 | 目标小队立即从战场移除，不计入击杀 |
| 1.5倍经验 | 我方参与劝降的小队获得1.5倍经验奖励 |
| 30%概率获得佣兵 | Roll(0~100) < 30 时，获得敌方1个随机单位作为佣兵 |

### 7.5 投降失败惩罚

| 惩罚 | 说明 |
|------|------|
| 少1次攻击 | 发起劝降的小队本回合少1次攻击机会 |
| 必定后手 | 下回合该小队行动顺序排到最后 |
| 敌方士气+5 | 敌方因抵抗劝降而士气小幅回升 |

## 8. 威胁等级计算

### 8.1 公式

```
threat = SUM(unit.level * class_coefficient * quality_coefficient) + artifact_total_value / 100
```

### 8.2 系数表

**class_coefficient（兵种系数）：**

| 兵种等级 | 系数 | 示例 |
|----------|------|------|
| T1 | 1.0 | 民兵、轻步兵 |
| T2 | 1.5 | 正规步兵、弓手 |
| T3 | 2.2 | 精锐骑兵、重步兵 |
| T4 | 3.5 | 将军卫队、特殊单位 |

**quality_coefficient（品质系数）：**

| 品质 | 系数 |
|------|------|
| N（普通） | 1.0 |
| R（稀有） | 1.1 |
| SR（超稀有） | 1.3 |
| SSR（传说） | 1.6 |

### 8.3 计算器

```csharp
public class ThreatCalculator
{
    public static float CalculateSquadThreat(SquadData squad, UnitDataManager unitMgr)
    {
        float threat = 0f;
        float artifactValue = 0f;

        // 队长
        var leader = unitMgr.GetUnit(squad.leader_unit_id);
        threat += CalculateUnitThreat(leader);

        // 成员
        foreach (var unitId in squad.member_list)
        {
            var unit = unitMgr.GetUnit(unitId);
            threat += CalculateUnitThreat(unit);
        }

        // 遗物
        foreach (var artId in squad.artifact_list)
        {
            var artifact = unitMgr.GetArtifact(artId);
            artifactValue += artifact.total_value;
        }

        threat += artifactValue / 100f;
        return threat;
    }

    private static float CalculateUnitThreat(Unit unit)
    {
        float classCoeff = unit.class_tier switch
        {
            1 => 1.0f,
            2 => 1.5f,
            3 => 2.2f,
            4 => 3.5f,
            _ => 1.0f
        };

        float qualityCoeff = unit.quality switch
        {
            UnitQuality.N   => 1.0f,
            UnitQuality.R   => 1.1f,
            UnitQuality.SR  => 1.3f,
            UnitQuality.SSR => 1.6f,
            _ => 1.0f
        };

        return unit.level * classCoeff * qualityCoeff;
    }
}
```

### 8.4 计算示例

**小队：1队长(SSR T3 Lv15) + 3成员(R T2 Lv10) + 1成员(N T1 Lv5) + 遗物总值500**

```
队长: 15 * 2.2 * 1.6 = 52.8
成员1: 10 * 1.5 * 1.1 = 16.5
成员2: 10 * 1.5 * 1.1 = 16.5
成员3: 10 * 1.5 * 1.1 = 16.5
成员4:  5 * 1.0 * 1.0 = 5.0
遗物: 500 / 100 = 5.0
─────────────────────────
总威胁 = 52.8 + 16.5*3 + 5.0 + 5.0 = 111.3
```

## 9. 包围判定算法

### 9.1 算法描述

统计与目标敌方小队相邻（6方向）的不同我方小队数量。6方向指六边形网格的6个邻居方向。

```
        [NW]  [NE]
       [W]  目标  [E]
        [SW]  [SE]
```

### 9.2 判定实现

```csharp
public class SurroundChecker
{
    // 六方向偏移（奇偶行不同，此处以奇数行为基准）
    private static readonly (int dx, int dy)[] EvenRowOffsets = {
        (-1, -1), (0, -1),   // NW, NE
        (-1,  0), (1,  0),   // W,  E
        (-1,  1), (0,  1)    // SW, SE
    };

    private static readonly (int dx, int dy)[] OddRowOffsets = {
        (0, -1), (1, -1),    // NW, NE
        (-1, 0), (1,  0),    // W,  E
        (0,  1), (1,  1)     // SW, SE
    };

    public static int CountSurroundingSquads(
        HexCoord targetCoord,
        List<SquadData> friendlySquads,
        BattleMap battleMap)
    {
        var surroundingSquadIds = new HashSet<string>();

        var offsets = targetCoord.y % 2 == 0 ? EvenRowOffsets : OddRowOffsets;

        foreach (var (dx, dy) in offsets)
        {
            var neighborCoord = new HexCoord(targetCoord.x + dx, targetCoord.y + dy);
            var squadAtPos = battleMap.GetSquadAt(neighborCoord);

            if (squadAtPos != null && friendlySquads.Contains(squadAtPos))
            {
                surroundingSquadIds.Add(squadAtPos.squad_id);
            }
        }

        return surroundingSquadIds.Count;
    }

    public static float GetSurroundBonus(int surroundCount)
    {
        return surroundCount switch
        {
            >= 4 => 15f,
            3    => 10f,
            2    => 5f,
            _    => 0f
        };
    }
}
```

### 9.3 包围判定规则

| 规则 | 说明 |
|------|------|
| 只计不同小队 | 同一小队的多个单位不算多次包围 |
| 必须相邻 | 仅6方向直接相邻的小队计入 |
| 友军判定 | 只有与攻击方同阵营的小队算包围方 |
| 动态更新 | 每回合开始时重新计算（小队可能移动） |

### 9.4 包围示例

```
地图布局（X=敌方, A/B/C=我方小队）:

     [A]
  [B] [X] [C]

包围数 = 3 (A, B, C)
surround_bonus = 10
```

## 10. 边界条件与测试用例

### 10.1 边界条件

| 场景 | 处理方式 |
|------|----------|
| 士气为0 | 锁定在0，不再下降；攻击力×0.75 |
| 士气为100 | 锁定在100，不再上升；持续享受高士气效果 |
| 防守方威胁为0 | defender_threat 设为 0.01 防除零，投降概率极高 |
| 投降概率计算为负值 | MAX(0, ...) 确保不低于0% |
| 投降概率计算超过80% | MIN(80, ...) 确保不超过80% |
| 同时多个士气事件 | 按事件触发顺序依次应用，每个事件独立计算 |
| 羁绊一方死亡 | 羁绊加成立即失效，不再提供士气 |
| 热浪+防暑装备 | 热浪惩罚无效，不触发-10 |
| 劝降时防守方只有队长 | 正常计算，队长倒下后小队全灭 |
| 连续再动 | 每次行动后独立判定20%，无硬性上限但概率递减 |
| 同一回合多次恐惧 | 只取最大值-30，不叠加为-60 |
| 包围方小队被消灭 | 下回合重新计算，包围数减少 |
