## 1. 小队编制系统

### 1.1 基本定义

| 概念 | 说明 | 数据类型 |
|------|------|----------|
| 军队 | 玩家拥有的所有角色集合 | 集合 |
| 小队 | 出战的基本单位，由多个角色组成 | 结构体 |
| 队长 | 每个小队必须有一个队长，提供领导力 | 角色ID |
| 规模 | 小队内所有角色的规模消耗总和 | 整数 |
| 领导力上限 | 队长领导力决定的最大允许规模 | 整数 |

### 1.2 小队数据结构

```
小队 {
  小队ID: int
  队长ID: int
  成员ID列表: int[]  // 最多9个成员，包含队长
  当前总规模: int
  小队名称: string
  经验加成: float
}
```

### 1.3 规模消耗计算规则

**基础规则：**

1. 每个角色有基础规模消耗：
   - 普通单位：基础规模 = 10
   - 特殊单位（龙、巨兽）：基础规模 = 15

2. 忠诚度修正：
   ```
   忠诚度 = 0 → 新兵，规模 +2 → 12
   忠诚度 = 1 → 信赖，规模 +0 → 10
   忠诚度 = 2 → 忠心，规模 -2 → 8
   ```

3. 队长技能修正：
   - 如果队长有对应兵种规模减免技能，该类兵种规模 -2
   - 例如：队长有"弓兵专精"，队中所有弓兵规模 -2

4. 羁绊关系修正：
   - 如果两个角色有羁绊关系，同时编入小队，每个角色规模 -1
   - 多个羁绊关系可叠加

5. 位置递增惩罚：
   ```
   第1-5个单位：无额外惩罚
   第6个单位：+2规模
   第7个单位：+2规模
   第8个单位：+2规模
   第9个单位：+2规模
   
   难度修正：
   - 简单难度：第6个开始 +1
   - 普通难度：第6个开始 +2
   - 困难难度：第6个开始 +3
   - 疯狂难度：第6个+3，第7个+6，第8个+9，第9个+12
   ```

6. 最终计算：
   ```
   最终规模 = 基础规模 + 忠诚度修正 + 队长技能修正 + 羁绊修正 + 位置惩罚
   最终规模 最小 = 6，不允许更低
   ```

**示例计算：**
- 普通弓兵，忠心，队长弓兵专精，有羁绊
- 计算：10 -2(忠诚) -2(队长) -1(羁绊) = 5 → 限制到最小6点

### 1.4 小队编制界面需求

1. **左侧**：显示玩家所有可用角色列表，可筛选（按职业、属性、忠诚度）
2. **中间**：当前编辑小队，显示9个格子，拖放调整位置
3. **右侧**：显示当前规模/上限，违规标红
4. **自动计算**：每次调整实时重新计算总规模
5. **保存条件**：总规模 ≤ 领导力上限才能保存
6. **预设功能**：保存多个编组预设，快速切换

### 1.5 出队数量限制

- 每章战斗根据关卡设定，允许派出N个小队（通常5-8个）
- 每个小队必须满足：
  - 有且仅有一个队长
  - 至少1个成员（可以只有队长）
  - 总规模 ≤ 队长领导力
- 不符合条件的小队不能上阵

### 1.6 验收标准

- [ ] 规模计算正确，符合上述公式
- [ ] 各种修正（忠诚、队长技能、羁绊）都生效
- [ ] 最小规模限制正确
- [ ] 不同难度的位置惩罚正确
- [ ] 编制界面拖放操作流畅
- [ ] 超规模提示明显

## 2. 战斗系统

### 2.1 回合结构

```
回合流程：
1. 我方回合
   - 玩家可以任意选择一个我方小队行动
   - 该小队完成移动+攻击/指令 → 该小队本回合结束
   - 重复直到所有我方小队都行动完毕
2. 敌方回合
   - AI按顺序每个敌方小队依次行动
   - 所有敌方行动完毕 → 回到我方回合
```

**特殊规则：**
- 有些特性允许小队额外行动（比如骑兵攻击后可以再移动）
- 有些道具可以让已行动小队恢复行动点数

### 2.2 攻击顺序定义

**全局攻击顺序表（从小到大，先打）：**

| 顺序值 | 兵种分类 | 说明 |
|--------|----------|------|
| 10 | 攻城炮 | 最先出手 |
| 20 | 火枪 | |
| 20 | 龙骑兵 | |
| 30 | 法师 | |
| 30 | 龙 | |
| 40 | 女术士 | |
| 50 | 弓箭 | |
| 50 | 弩 | |
| 50 | 游侠 | |
| 60 | 轻步兵 | |
| 60 | 轻骑兵 | |
| 60 | 剑圣 | |
| 60 | 巨兽 | |
| 70 | 重步兵 | |
| 70 | 重骑兵 | |
| 70 | 女武神 | |
| 70 | 圣骑士 | |
| 80 | 学徒 | 治疗辅助 |
| 80 | 祭司 | 治疗辅助 |

**排序规则：**
- 攻击前，将攻击方所有存活成员按顺序值从小到大排序
- 按顺序依次出手，一个打完下一个再打
- 如果有多个同一顺序值，按在小队中的位置顺序（从上到下）

**战术意义：**
- 高顺序单位先出手，能快速击杀敌人，减少敌人反击
- 低顺序单位后出手，适合收割残血
- 混合编组避免伤害溢出

### 2.3 自动战斗流程

当玩家选择"A小队攻击B小队"后，自动战斗按以下流程进行：

```
自动战斗流程：
1. 初始化：双方存活列表，攻击者顺序排序
2. 攻击者按顺序依次出手：
   a. 根据攻击者职业和位置，选择攻击目标
      - 近战优先打前排，远程优先打血少的
   b. 计算命中：命中 = 攻击者SKL - 目标DEF + 各种加成 + 地形修正 + 随机数
   c. 未命中 → 跳转到下一个攻击者
   d. 命中 → 计算伤害，扣除目标HP
   e. 如果目标HP ≤ 0 → 目标死亡，从存活列表移除
   f. 检查攻击者是否有连击特性，是否触发额外攻击
   g. 攻击者下一个
3. 攻击者全部出手完 → 换防御方存活者反击
   a. 防御方按顺序值排序
   b. 同样依次出手，攻击攻击方
   c. 同上计算伤害
4. 战斗结束 → 统计伤亡，结算经验，更新士气
5. 如果一方全队死亡 → 战斗结束
```

**目标选择AI规则：**

| 攻击者类型 | 目标优先级 |
|------------|-----------|
| 近战 | 前排存活(最近位置) → 血多的 → 任意 |
| 远程 | 血少残血 → 低防御 → 任意 |
| 法师/范围 | 人多的位置 → 血少的 |
| 治疗 | 血百分比最低的队友 |

### 2.4 士气系统

**士气数据：**
```
每个小队有士气值：0-100
初始士气：50
```

**士气影响：**
```
士气 > 80 → 高昂
- 伤害 +20%
- 每回合15%概率触发额外攻击一次

士气 < 20 → 低落
- 伤害 -20%
- 每回合行动前20%概率直接投降
- 如果投降 → 战斗直接结束，该小队投降
```

**士气变化规则：**

战斗中事件增减士气：

| 事件 | 士气变化 |
|------|----------|
| 击败敌方单位 | +2 每人 |
| 本方单位死亡 | -3 每人 |
| 队长被击杀 | -15 |
| 附近有羁绊队友（三格内） | 每回合开始 +2 |
| 占领建筑 | +5 |
| 被包围（周围四格都是敌人） | 每回合 -3 |

**投降判定：**

满足以下全部条件，触发投降：
```
1. 敌方士气 < 30
2. 我方战力 > 敌方战力 × 2.0
3. 被我方包围（至少三面被围）
4. 随机判定：概率 = (100 - 敌方士气) × 0.01 × 包围加成
   - 三面包围：×1.5
   - 四面包围：×2.0
```

**投降收益：**
- 玩家获得金币 = 投降单位总等级 × 10
- 玩家获得星星 +2（正常击杀+1）
- 有概率可以感化招募投降单位
- 投降单位全部存活被俘虏，不是死亡

### 2.5 地形系统

**地形属性表：**

| 地形 | 防御加成 | 闪避加成 | 移动消耗 |
|------|----------|----------|----------|
| 平地 | 0 | 0 | 1 |
| 高地 | +10%DEF | 0 | 1 |
| 树林 | 0 | +10%闪避 | 2 |
| 山地 | +5%DEF | +5%闪避 | 3 |
| 要塞 | +30%DEF | 0 | 1 |
| 河流 | 0 | 0 | 3 |
| 城墙 | +25%DEF | 0 | 2 |

**效果说明：**
- 防御加成百分比，直接乘到DEF上
- 闪避加成：被命中概率降低对应百分比

### 2.6 指令系统

每个小队行动时，可以选择以下指令之一：

| 指令 | 效果 | 消耗 |
|------|------|------|
| 正常攻击 | 无修正，正常伤害 | 无 |
| 追击 | 伤害 +15%，命中 +10% | 无 |
| 防御 | 受到伤害 -30% | 无 |
| 休整 | 不攻击，恢复20%伤兵 | 无 |
| 劝降 | 降低敌方士气 15点，本回合攻击次数 -1，最后出手 | 无 |

**劝降规则：**
- 使用劝降指令后，本回合只有一次攻击机会，且排在最后
- 战后敌方士气强制降低15点，提高后续投降概率

### 2.7 伤兵系统

**规则：**
- 战斗中损失的HP，一部分变成伤兵，一部分死亡
- 伤兵比例 = 领导力影响，队长LDR越高，伤兵比例越高
```
伤兵比例 = 0.3 + (LDR × 0.01)
最大伤兵比例 = 0.8
```
- 战斗结束后，伤兵可以通过金币治疗恢复
- 完全死亡的单位需要重新招募，费用更高

### 2.8 验收标准

- [ ] 攻击顺序正确，按顺序值从小到大出手
- [ ] 自动战斗流程正确，不需要玩家操作每个单位
- [ ] 士气变化计算正确，高昂/低落效果生效
- [ ] 投降判定正确，投降收益正确
- [ ] 地形加成正确应用
- [ ] 各个指令效果正确
- [ ] 伤兵计算正确

## 3. 角色属性与成长系统

### 3.1 角色数据结构

```
角色 {
  角色ID: int
  姓名: string
  角色类型: 剧情/精英/佣兵
  职业ID: int
  当前等级: int
  当前XP: int
  当前CP: int  // 职业熟练度
  
  // 六元素属性（固定，不随等级变化）
  光: int  // 0-10
  暗: int  // 0-10
  水: int  // 0-10
  火: int  // 0-10
  电: int  // 0-10
  土: int  // 0-10
  
  // 基础属性（随等级成长）
  STR: int  // 力量，影响物理伤害
  MAG: int  // 魔力，影响法术伤害
  SKL: int  // 技巧，影响命中暴击
  DEF: int  // 防御，减少受到伤害
  LDR: int  // 领导力，影响小队规模上限
  HP: int   // 生命值，最大血量
  
  // 其他属性
  忠诚度: int  // 0=新兵 1=信赖 2=忠心
  经验加成: float
  状态: 正常/受伤/俘虏/死亡
  
  // 羁绊
  羁绊关系列表: [角色ID, 好感度]
  配偶ID: int  // 0表示未婚
  
  // 转职历史
  职业列表: 历史职业ID[]
  是否晋升: bool
}
```

### 3.2 六元素属性转换

六元素直接转换成基础属性加成：

```
STR += 火 + 暗
MAG += 水 + 暗
SKL += 电
DEF += 土
LDR += 光
HP  += 土 × 5
```

**设计意图：**
- 光属性 = 好队长，因为加LDR
- 暗属性 = 输出高，兼顾物理法术
- 水属性 = 法系
- 火属性 = 物理
- 电属性 = 敏捷系（弓、剑圣、火枪）
- 土属性 = 肉盾

### 3.3 成长规则

每次升级，基础属性按职业成长率成长：

```
新的基础属性 = 旧属性 + 职业成长率 × (1 + 元素加成)
```

**成长率**：每个职业预先定义STR/MAG/SKL/DEF/LDR/HP的成长率

例如战士成长率：
```
STR: 1.5
MAG: 0.2
SKL: 0.8
DEF: 1.2
LDR: 0.1
HP:  3.0
```

### 3.4 双经验条

**XP（等级经验）：**
- 战斗结束后，根据击杀敌人等级获得XP
- XP满足升级要求 → 等级+1，属性成长，XP清零
- 没有上限，满级后继续获得XP

**CP（职业熟练度）：**
- 战斗获得，比例：CP = XP × 0.8
- 当前职业CP满了才能转职
- 晋升需要三阶CP满

### 3.5 角色类型

| 类型 | 特点 |
|------|------|
| 剧情角色 | 固定姓名，固定六元素，固定成长，不能解雇 |
| 精英佣兵 | 固定六元素，商店随机刷出，可以解雇 |
| 普通佣兵 | 六元素随机，商店刷出，可以解雇 |

**随机六元素生成规则（普通佣兵）：**
- 总点数：30点随机分配到六个属性
- 每个属性最小1点，最大10点
- 随机分配，所以可能出现极品（全点一个属性）也可能垃圾

### 3.6 验收标准

- [ ] 六元素正确转换为基础属性
- [ ] 升级成长计算正确
- [ ] XP和CP独立计算，互不干扰
- [ ] 随机佣兵六元素生成正确
- [ ] 不同角色类型属性处理正确

## 4. 转职系统

### 4.1 转职树结构

```
一阶职业（初始）
    ↓
二阶职业（多种分支选择）
    ↓
三阶职业（顶级）
    ↓
晋升（强化，不能再转）
```

**设计：** 每个一阶职业可以转多个二阶，每个二阶可以转多个三阶，自由选择路线。

### 4.2 转职条件

满足所有条件才能转职：

1. 当前职业CP已满
2. 关键属性达到要求（例如：剑圣需要SKL ≥ 52）
3. 消耗对应资源：
   - 步兵系：消耗铁矿 × 数量
   - 骑兵系：消耗铁矿 + 马匹 × 数量
   - 火枪系：消耗铁矿 + 火药 × 数量
   - 龙：消耗特殊龙石

### 4.3 转职流程

```
1. 检查条件 → 不满足提示
2. 扣除资源
3. 改变角色职业为新职业
4. 保留当前等级和XP
5. CP清零，重新积累
6. 新职业成长率生效，下次升级按新成长率成长
7. 添加新职业到转职历史
```

### 4.4 晋升系统

**条件：**
- 已经是三阶职业
- 当前三阶CP已满
- 消耗一个"华彩之证"

**效果：**
- 所有基础属性 +10%
- 获得晋升特性
- 晋升后不能再转职，职业固定
- 角色边框变色，区分普通单位

### 4.5 自由转职规则

- 只要满足条件，可以任意转任何开放职业
- 例如：战士可以转法师，只要属性达标资源够
- 支持洗build，玩家可以尝试不同组合

### 4.6 主要职业列表（参考）

#### 一阶
- 新兵、学徒、猎人、骑兵、佣兵

#### 二阶分支示例
- 新兵 → 轻步兵 / 重步兵 / 弓箭手
- 学徒 → 法师 / 祭司
- 猎人 → 游侠 / 狙击手
- 骑兵 → 轻骑兵 / 骠骑兵

#### 三阶分支示例
- 轻步兵 → 剑圣 / 百夫长
- 重步兵 → 卫士 / 重甲骑士
- 法师 → 火法 / 冰法 / 雷法
- 弓箭手 → 神射手 / 盾弩手

### 4.7 验收标准

- [ ] 转职条件检查正确
- [ ] 资源扣除正确
- [ ] 职业变更后成长率正确
- [ ] CP清零正确
- [ ] 晋升属性加成正确
- [ ] 晋升后不能再转职

## 5. 科技树系统

### 5.1 基本结构

三大独立分支，玩家自由点：

1. **战争学院** - 单位强化，解锁特殊单位
2. **战术指挥** - 战术指令，全队加成
3. **工匠技术** - 装备强化，火器强化

每个分支有三列，每列若干层级，总共约30个科技点。

### 5.2 科技数据结构

```
科技 {
  科技ID: int
  分支: 战争学院/战术指挥/工匠
  位置坐标: (列, 行)
  前置科技: 科技ID[]
  资源需求: {资源类型: 数量}
  效果描述: string
  效果代码: 触发效果的参数
  是否解锁: bool
}
```

### 5.3 点科技规则

```

## 7. 婚姻与生育系统

### 7.1 结婚条件

- 两个异性角色

### 7.2 结婚效果

- 解锁生育功能

### 7.3 生育规则

- 结婚后，每两章有概率怀孕
- 怀孕后一章，生下后代
- 后代是全新角色
- 父母等级越高，后代属性越好

### 7.4 后代属性遗传规则

```
后代六元素 = 父母六元素加权平均 + 随机扰动
公式：
for each 元素 in [光,暗,水,火,电,土]:
  后代值 = (父元素 + 母元素) / 2 + 随机(-2, +2)
  限制：后代值 ∈ [1, 10]
总点数限制：后代总点数 = 父母平均总点数 + 随机(-3, +3)
```

**成长率遗传：**
- 后代成长率 = 父母职业成长率随机混合

### 7.5 后代特点

- 后代可以正常培养、转职、晋升
- 后代和父母之间天生有羁绊
- 父母如果死亡，后代有复仇加成
- 理论上可以培养多代，属性越来越强

## 13. AI系统设计

### 13.1 敌方回合AI流程

```
敌方回合开始：
1. 按顺序逐个敌方小队行动
2. AI决策：
   a. 扫描地图，找到最近的我方小队
   b. 判断是否能攻击到
   c. 如果能攻击 → 选择血量最低/最弱的目标攻击
   d. 如果不能攻击 → 向最近的我方移动
   e. 如果周围都是己方 → 向我方推进
3. 完成行动 → 下一个敌方小队
```

### 13.2 目标选择优先级

AI选择攻击目标优先级：

1. **残血我方小队**（HP < 30%）→ 优先击杀
2. **法师/远程小队** → 优先干掉输出
3. **最近的** → 如果都差不多打最近的
4. **士气低的** → 更容易击杀/劝降

### 13.3 移动AI

- 如果敌方有小队在射程内 → 直接攻击
- 如果没有 → 向最近的我方小队移动
- 如果被包围 → 尝试突围，往包围圈缺口走
- 如果首领AI → 保护首领，躲在后面

### 13.4 难度AI调整

| 难度 | AI强度 | 额外加成 |
|------|--------|----------|
| 简单 | 保守 | 敌方属性 -10% |
| 普通 | 正常 | 无加成 |
| 困难 | 激进 | 敌方属性 +10% |
| 疯狂 | 非常激进 | 敌方属性 +20%，获得额外技能 |

### 13.5 验收标准

- [ ] AI会主动寻找并攻击我方
- [ ] 目标选择符合优先级
- [ ] 不同难度AI强度正确
- [ ] 不会卡住，不会原地不动

- 资源点：铁矿/马场/工坊，占领后每章给资源
- 怪物巢穴：反复刷新，刷经验金币
```

### 15.2 移动规则

- 玩家队伍在大地图自由移动
- 进入地点触发战斗或事件
- 每个区域有迷雾，探索后解锁
- 击败区域Boss解锁下一个区域

### 15.3 随机事件

在大地图移动有概率触发随机事件：

- 遭遇强盗 → 战斗，获得金币
- 商人来访 → 特殊商店
- 佣兵招募 → 免费获得一个佣兵
- 陷阱 → 损失一些金币/伤兵
- 宝藏 → 获得随机装备

### 15.4 资源点产出

- 占领资源点后，每回合（每章）自动产出资源
- 可以升级资源点，增加产出
- 需要派兵驻守，不驻守不产出

### 15.5 验收标准

- [ ] 大地图移动流畅
- [ ] 迷雾解锁正确
- [ ] 随机事件概率正确
- [ ] 资源点自动产出正确

## 16. 竞技场系统

### 16.1 基本规则

- 准备阶段可以选择挑战竞技场
- 每次挑战花费金币
- 挑战对手是随机生成的AI队伍
- 击败对手获得额外经验和金币

### 16.2 难度递增

- 五场连胜后，可以选择继续挑战
- 难度越来越高，奖励越来越好
- 有概率出极品装备

### 16.3 奖励规则

```
金币奖励 = 对手等级总和 × 10
经验奖励 = 对手等级总和 × 5
第N场连胜奖励翻倍 × (N/2)
```

### 16.4 验收标准

- [ ] 对手生成正确，难度递增
- [ ] 奖励计算正确
- [ ] 花费金币正确
