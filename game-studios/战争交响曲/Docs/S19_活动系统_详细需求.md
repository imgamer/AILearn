## 一、系统概述

### 1.1 核心定位

限时活动系统是运营节奏的核心载体，驱动玩家回流和付费。通过周期性活动（限时关卡、羁绊UP、资源双倍、登录奖励、公会战、赛季、节日）维持游戏活跃度，提供短期目标和稀缺资源获取渠道。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 运营节奏驱动 | 活动排期形成规律节奏，玩家可预期，运营可调控 |
| 轻度参与保底 | 即使轻度参与也能获得基础奖励，重度参与获得额外奖励 |
| 不强制付费 | 所有活动奖励可通过时间投入获取，付费仅加速 |
| 数据驱动 | 每个活动的参与率/完成率/付费转化均有统计，指导后续排期 |
| 配置化 | 活动类型、时间、奖励均通过配置表定义，无需发版即可调整 |

### 1.3 活动类型总览

| 类型 | 频率 | 持续时间 | 核心目标 | 需求ID前缀 |
|------|------|---------|---------|-----------|
| 限时关卡活动 | 每2周 | 7天 | 提供挑战内容+稀缺资源 | EVT-LV |
| 羁绊活动 | 每月 | 7天 | 角色培养+抽卡驱动 | EVT-BD |
| 资源双倍活动 | 每周轮换 | 1天 | 日常活跃刺激 | EVT-RS |
| 登录活动 | 每月 | 7天(需连续7天) | 回流+留存 | EVT-LG |
| 公会战活动 | 每2周 | 3天 | 社交+竞争 | EVT-GW |
| 赛季活动 | 每6周 | 42天 | 长线目标 | EVT-SS |
| 节日活动 | 特殊 | 7-14天 | 限定内容+付费高峰 | EVT-HD |

## 四、活动类型详细设计

### 4.1 限时关卡活动 (EVT-LV)

#### 4.1.1 核心流程

```
活动开始 → 玩家进入活动关卡 → 选择难度 → 挑战关卡 → 结算积分 → 累计积分
→ 达到积分阈值 → 领取积分奖励 → 活动结束 → 排名结算 → 发放排名奖励
```

#### 4.1.2 难度与积分

| 难度 | 关卡特点 | 通关积分 | 每日次数 | 推荐战力 |
|------|---------|---------|---------|---------|
| 普通 | 标准敌人配置 | 10分 | 3次 | 5000 |
| 困难 | 精英敌人+特殊机制 | 30分 | 3次 | 15000 |
| 地狱 | Boss级敌人+地形限制 | 80分 | 2次 | 30000 |

#### 4.1.3 积分奖励档位

| 档位 | 积分阈值 | 奖励 |
|------|---------|------|
| 第1档 | 100分 | 金币×10000 |
| 第2档 | 500分 | 经验书×10 + 钻石×50 |
| 第3档 | 1000分 | SR碎片自选×5 + 抽卡券×1 |
| 第4档 | 2000分 | 活动限定皮肤碎片×10 + 钻石×200 |
| 第5档 | 5000分 | SSR碎片自选×5 + 活动限定头像框 |

#### 4.1.4 排名奖励

| 排名 | 奖励 |
|------|------|
| 第1名 | SSR碎片自选×10 + 钻石×500 + 限定称号 |
| 第2-3名 | SSR碎片自选×5 + 钻石×300 |
| 第4-10名 | SR碎片自选×10 + 钻石×200 |
| 第11-50名 | SR碎片自选×5 + 钻石×100 |
| 第51-100名 | 钻石×50 + 抽卡券×1 |

#### 4.1.5 代码实现

```csharp
public class LimitedLevelEventHandler
{
    public void OnStageClear(int event_id, int stage_id, int difficulty)
    {
        var progress = GetProgress(event_id);
        var config = GetConfig<LimitedLevelConfig>(event_id);

        int score = difficulty switch
        {
            1 => config.normal_score,
            2 => config.hard_score,
            3 => config.hell_score,
            _ => 0
        };

        progress.total_score += score;
        progress.progress_dict["daily_attempt_" + difficulty] =
            progress.progress_dict.GetValueOrDefault("daily_attempt_" + difficulty, 0) + 1;

        CheckScoreRewards(event_id, progress);
        SaveProgress(progress);
    }

    public bool CanChallenge(int event_id, int difficulty)
    {
        var config = GetConfig<LimitedLevelConfig>(event_id);
        var progress = GetProgress(event_id);

        int daily_limit = difficulty switch
        {
            1 => config.normal_score > 0 ? config.daily_attempt_limit : 0,
            2 => config.daily_attempt_limit,
            3 => config.daily_attempt_limit,
            _ => 0
        };

        if (daily_limit <= 0) return true;

        int used = progress.progress_dict.GetValueOrDefault("daily_attempt_" + difficulty, 0);
        return used < daily_limit;
    }

    private void CheckScoreRewards(int event_id, EventProgress progress)
    {
        var rewards = TB_EventReward.GetByEventId(event_id)
            .Where(r => r.reward_type == "score_threshold")
            .OrderBy(r => r.threshold);

        foreach (var reward in rewards)
        {
            if (progress.total_score >= reward.threshold
                && !progress.claimed_rewards.Contains(reward.reward_id))
            {
                NotifyRewardAvailable(event_id, reward.reward_id);
            }
        }
    }

    public void OnEventEnd(int event_id)
    {
        var rankings = CalculateRankings(event_id);
        DistributeRankingRewards(event_id, rankings);
    }
}
```

### 4.2 羁绊活动 (EVT-BD)

#### 4.2.1 核心流程

```
活动开始 → 指定2-3个角色羁绊经验×2 → UP池同步开放 → 专属羁绊剧情限时开放
→ 玩家使用指定角色战斗 → 羁绊经验双倍获取 → 活动结束恢复正常
```

#### 4.2.2 详细规则

| 规则 | 说明 |
|------|------|
| 加成角色数 | 2-3个(配置决定) |
| 经验倍率 | ×2.0 |
| UP池 | 与活动角色对应的抽卡UP池同步开放 |
| 羁绊剧情 | 指定角色的专属羁绊剧情限时解锁，活动后锁定 |
| 叠加规则 | 不与其他羁绊经验加成叠加，取最高倍率 |

#### 4.2.3 代码实现

```csharp
public class BondUpEventHandler
{
    private HashSet<int> _bonus_unit_ids = new HashSet<int>();
    private float _exp_multiplier = 1.0f;

    public void OnEventStart(int event_id)
    {
        var config = GetConfig<BondUpConfig>(event_id);
        _bonus_unit_ids = new HashSet<int>(config.bonus_unit_ids);
        _exp_multiplier = config.bond_exp_multiplier;

        // 开放UP池
        foreach (int pool_id in config.gacha_pool_ids)
        {
            GachaSystem.ActivatePool(pool_id);
        }

        // 解锁羁绊剧情
        foreach (int story_id in config.story_ids)
        {
            StorySystem.UnlockStory(story_id, StoryUnlockType.EventLimited);
        }
    }

    public float GetBondExpMultiplier(int unit_id)
    {
        if (_bonus_unit_ids.Contains(unit_id))
        {
            return _exp_multiplier;
        }
        return 1.0f;
    }

    public void OnEventEnd(int event_id)
    {
        var config = GetConfig<BondUpConfig>(event_id);

        // 关闭UP池
        foreach (int pool_id in config.gacha_pool_ids)
        {
            GachaSystem.DeactivatePool(pool_id);
        }

        // 锁定羁绊剧情(已读不回收，未读锁定)
        foreach (int story_id in config.story_ids)
        {
            StorySystem.LockStory(story_id, StoryUnlockType.EventLimited);
        }

        _bonus_unit_ids.Clear();
        _exp_multiplier = 1.0f;
    }
}
```

### 4.3 资源双倍活动 (EVT-RS)

#### 4.3.1 每周轮换排期

| 星期 | 资源类型 | 倍率 | 说明 |
|------|---------|------|------|
| 周一 | 金币 | ×2 | 金币关卡收益翻倍 |
| 周三 | 转职资源 | ×2 | 转职材料关卡收益翻倍 |
| 周五 | 经验 | ×2 | 经验关卡收益翻倍 |
| 周日 | 全资源 | ×1.5 | 所有资源关卡收益1.5倍 |

#### 4.3.2 代码实现

```csharp
public class ResourceDoubleEventHandler
{
    private Dictionary<string, float> _active_multipliers = new Dictionary<string, float>();

    public void OnEventStart(int event_id)
    {
        var config = GetConfig<ResourceDoubleConfig>(event_id);
        _active_multipliers[config.resource_type] = config.multiplier;
    }

    public float GetResourceMultiplier(string resource_type)
    {
        // 全资源类型特殊处理
        if (_active_multipliers.ContainsKey("all"))
        {
            float all_mult = _active_multipliers["all"];
            float specific_mult = _active_multipliers.GetValueOrDefault(resource_type, 1.0f);
            return Mathf.Max(all_mult, specific_mult); // 取最高倍率，不叠加
        }

        return _active_multipliers.GetValueOrDefault(resource_type, 1.0f);
    }

    public void OnEventEnd(int event_id)
    {
        var config = GetConfig<ResourceDoubleConfig>(event_id);
        _active_multipliers.Remove(config.resource_type);
    }
}
```

### 4.4 登录活动 (EVT-LG)

#### 4.4.1 核心流程

```
活动开始 → 玩家每日首次登录 → 记录登录天数 → 达到对应天数 → 可领取奖励
→ 连续7天全部领取 → 获得全部奖励 → 活动结束
```

#### 4.4.2 每日奖励

| 天数 | 奖励 | 价值评估 |
|------|------|---------|
| Day1 | 金币×10000 | 基础资源 |
| Day2 | 经验书×5 | 培养资源 |
| Day3 | 钻石×100 | 稀缺货币 |
| Day4 | SR碎片自选×5 | 角色培养 |
| Day5 | 抽卡券×1 | 抽卡机会 |
| Day6 | 转职资源包 | 进阶资源 |
| Day7 | SSR碎片自选×5 | 高价值奖励 |

#### 4.4.3 规则说明

| 规则 | 说明 |
|------|------|
| 连续性 | 必须连续7天登录，中断则从断点继续(不重置) |
| 领取条件 | 当天登录即可领取对应天数奖励 |
| 补领 | 不支持补领，错过即错过 |
| 活动时间 | 活动持续7天，与登录天数一一对应 |

#### 4.4.4 代码实现

```csharp
public class LoginBonusEventHandler
{
    public void OnDailyLogin(string player_id, int event_id)
    {
        var progress = GetProgress(player_id, event_id);
        if (progress == null)
        {
            progress = CreateProgress(player_id, event_id);
            progress.first_enter_time = GetCurrentTimestamp();
        }

        long today = GetTodayDateKey();
        long last_login_date = progress.progress_dict.GetValueOrDefault("last_login_date", 0);

        if (last_login_date < today)
        {
            progress.login_day++;
            progress.progress_dict["last_login_date"] = today;
            progress.progress_dict["login_day"] = progress.login_day;

            SaveProgress(progress);
            NotifyLoginDayAvailable(event_id, progress.login_day);
        }
    }

    public bool ClaimDayReward(string player_id, int event_id, int day)
    {
        var progress = GetProgress(player_id, event_id);

        if (progress.login_day < day)
        {
            return false; // 还未达到该天数
        }

        if (progress.claimed_days.Contains(day))
        {
            return false; // 已领取
        }

        var config = GetConfig<LoginBonusConfig>(event_id);
        var day_reward = config.day_rewards.Find(r => r.day == day);
        if (day_reward == null)
        {
            return false;
        }

        progress.claimed_days.Add(day);
        GrantReward(player_id, day_reward.reward_json);
        SaveProgress(progress);

        return true;
    }

    private long GetTodayDateKey()
    {
        return DateTimeOffset.UtcNow.ToUnixTimeSeconds() / 86400;
    }
}
```

### 5.1 调度引擎

```csharp
public class EventScheduler
{
    private List<EventSchedule> _schedules;
    private Dictionary<int, EventData> _active_events;

    public void Initialize()
    {
        _schedules = TB_EventSchedule.GetAll();
        _active_events = new Dictionary<int, EventData>();

        CheckAndActivateEvents();
    }

    public void OnTimerTick()
    {
        long now = GetCurrentTimestamp();

        // 检查需要开始的活动
        foreach (var schedule in _schedules)
        {
            if (schedule.start_date <= now && schedule.end_date > now)
            {
                if (!_active_events.ContainsKey(schedule.event_id))
                {
                    ActivateEvent(schedule.event_id);
                }
            }
        }

        // 检查需要结束的活动
        List<int> to_remove = new List<int>();
        foreach (var kvp in _active_events)
        {
            var schedule = _schedules.Find(s => s.event_id == kvp.Key);
            if (schedule != null && schedule.end_date <= now)
            {
                to_remove.Add(kvp.Key);
            }
        }

        foreach (int event_id in to_remove)
        {
            DeactivateEvent(event_id);
        }
    }

    private void ActivateEvent(int event_id)
    {
        var event_data = TB_EventConfig.Get(event_id);
        event_data.is_active = true;
        _active_events[event_id] = event_data;

        // 通知各活动Handler
        switch (event_data.type)
        {
            case EventType.LimitedLevel:
                LimitedLevelEventHandler.OnEventStart(event_id);
                break;
            case EventType.BondUp:
                BondUpEventHandler.OnEventStart(event_id);
                break;
            case EventType.ResourceDouble:
                ResourceDoubleEventHandler.OnEventStart(event_id);
                break;
            case EventType.LoginBonus:
                LoginBonusEventHandler.OnEventStart(event_id);
                break;
            case EventType.GuildWar:
                GuildWarEventHandler.OnEventStart(event_id);
                break;
            case EventType.Season:
                SeasonEventHandler.OnEventStart(event_id);
                break;
            case EventType.Holiday:
                HolidayEventHandler.OnEventStart(event_id);
                break;
        }

        // 推送通知
        PushNotification(event_data);
    }

    private void DeactivateEvent(int event_id)
    {
        var event_data = _active_events[event_id];

        // 结束前24小时提醒(在ActivateEvent中注册)
        // 通知各活动Handler
        switch (event_data.type)
        {
            case EventType.LimitedLevel:
                LimitedLevelEventHandler.OnEventEnd(event_id);
                break;
            case EventType.BondUp:
                BondUpEventHandler.OnEventEnd(event_id);
                break;
            case EventType.ResourceDouble:
                ResourceDoubleEventHandler.OnEventEnd(event_id);
                break;
            case EventType.LoginBonus:
                LoginBonusEventHandler.OnEventEnd(event_id);
                break;
            case EventType.GuildWar:
                GuildWarEventHandler.OnEventEnd(event_id);
                break;
            case EventType.Season:
                SeasonEventHandler.OnEventEnd(event_id);
                break;
            case EventType.Holiday:
                HolidayEventHandler.OnEventEnd(event_id);
                break;
        }

        event_data.is_active = false;
        _active_events.Remove(event_id);
    }

    private void CheckAndActivateEvents()
    {
        long now = GetCurrentTimestamp();
        foreach (var schedule in _schedules)
        {
            if (schedule.start_date <= now && schedule.end_date > now)
            {
                ActivateEvent(schedule.event_id);
            }
        }
    }
}
```

| 期望 | 已领取的皮肤保留，未领取的奖励消失(活动结束前有提醒) |

### 测试用例10：活动排期冲突处理

| 项目 | 值 |
|------|-----|
| 前提 | 两个公会战活动排期重叠 |
| 操作 | 调度引擎检测到冲突 |
| 期望 | 按priority高的活动优先执行，低优先级活动自动延后 |

## 附录A：活动排期模板

### A.1 月度排期模板

```
第1周:  限时关卡活动(7天) + 资源双倍-金币(周一)
第2周:  公会战活动(3天) + 资源双倍-转职(周三)
第3周:  限时关卡活动(7天) + 资源双倍-经验(周五)
第4周:  羁绊活动(7天) + 登录活动(7天) + 资源双倍-全资源(周日)

每月1日: 登录活动重置
每6周:  赛季活动开始/结束
特殊:   节日活动按实际日期排期
```

### A.2 排期冲突优先级

| 优先级 | 活动类型 | 说明 |
|--------|---------|------|
| 1(最高) | 节日活动 | 不可调整 |
| 2 | 赛季活动 | 不可调整 |
| 3 | 公会战活动 | 尽量不调整 |
| 4 | 羁绊活动 | 可微调 |
| 5 | 限时关卡活动 | 可微调 |
| 6 | 登录活动 | 可微调 |
| 7(最低) | 资源双倍活动 | 可灵活调整 |
