## 一、项目技术概览

### 1.1 项目基本信息

| 项目属性 | 值 |
|----------|-----|
| 项目类型 | 2D回合制战术RPG手游（F2P） |
| 引擎版本 | Unreal Engine 5.7 |
| 核心渲染方案 | Paper2D + PaperZD插件 |
| 目标平台 | iOS / Android / PC（跨平台） |
| 网络架构 | 客户端-服务端，服务端权威 |
| 目标帧率 | 移动端30fps / PC 60fps |
| 屏幕方向 | 横屏（Landscape） |
| 画面风格 | 像素风2D + 3D特效混合 |

### 1.2 技术栈总览

```
┌──────────────────────────────────────────────────────┐
│                    表现层 (Presentation)               │
│  UMG/CommonUI │ Paper2D Sprites │ Niagara 3D FX      │
├──────────────────────────────────────────────────────┤
│                    游戏逻辑层 (Gameplay)               │
│  GAS(技能/属性/Buff) │ 回合制状态机 │ AI行为树         │

│                    数据层 (Data)                       │
│  DataTable │ PrimaryDataAsset │ Data Registry         │
├──────────────────────────────────────────────────────┤
│                    网络层 (Network)                    │
│  服务端权威 │ RepNotify │ RPC │ 热更新                 │
├──────────────────────────────────────────────────────┤
│                    引擎层 (Engine)                     │
│  UE5.7 │ Enhanced Input │ Substrate │ MegaLights      │
└──────────────────────────────────────────────────────┘
```

### 1.3 核心技术决策理由

| 决策 | 理由 |
|------|------|
| 选择UE5.7而非Unity | GAS系统成熟度、Niagara特效能力、服务端架构能力、长期维护保障 |
| 选择Paper2D而非纯3D | 像素风格需求、美术管线简洁、移动端性能友好、小团队可驾驭 |
| 选择PaperZD补充 | Paper2D缺少AnimBP，54职业×5状态无法手动管理 |
| 选择GAS而非自研技能系统 | GAS提供完整的属性/技能/Buff/Tag体系，省去大量底层开发 |
| 选择CommonUI而非原生UMG | 跨平台输入适配（触屏+键鼠+手柄）、焦点管理、模板化组件 |
| 选择Enhanced Input而非旧输入 | UE5.7标准、支持运行时重映射、动作映射与轴映射分离 |
| 关闭Nanite/Lumen/TSR | 2D渲染不需要，开启会严重拖累移动端性能和包体 |

## 二、Paper2D 系统深度分析

### 2.1 Paper2D核心资产

#### 2.1.1 Sprite（精灵图）

**在本项目中的用途**：
- 角色像素小人（54种职业×4方向×6帧 = 约1296个Sprite）
- UI元素（图标、按钮装饰、边框等）
- 战斗特效帧（部分2D特效用帧序列而非Niagara）
- 地形装饰物

**导入流程**：

```
原始像素图(PNG) 
  → Import to UE as Texture2D
  → Sprite Sheet导入模式：Paper2D Sprite Sheet
  → Extract Sprites：批量提取单个Sprite
  → Apply Paper 2D Import Settings：优化像素风格
```

**Sprite Editor功能**：

| 编辑模式 | 用途 | 本项目应用 |
|----------|------|-----------|
| View | 预览Sprite | 日常检查 |
| Edit Source Region | 调整Sprite在纹理中的裁切区域 | 精确裁切像素角色帧 |
| Edit Collision | 编辑2D碰撞多边形 | 小队网格碰撞体定义 |
| Edit RenderGeom | 编辑渲染几何体 | 修正像素角色渲染边界 |

**像素风格优化设置**：

```
Texture Settings:
  - Filtering: Nearest（最近邻采样，保持像素锐利）
  - Mip Gen Settings: NoMipmaps（禁用Mipmap，避免缩小模糊）
  - Compression: UserInterface2D（UI/2D专用压缩）
  - sRGB: true（UI Sprite）/ false（非UI Sprite）
  - Never Stream: true（小Sprite不应被流式卸载）

Sprite Settings:
  - Pixels Per Unreal Unit: 1.0（1像素=1UE单位，精确控制）
  - Pivot Point: 底部中心（角色）/ 中心（特效）
```

**Sprite资产命名规范**：

```
SP_[类别]_[职业/类型]_[方向]_[帧号]
例：
SP_Unit_Warrior_S_00    // 战士·南向·第0帧
SP_Unit_Mage_E_03       // 法师·东向·第3帧
SP_UI_Icon_Class_Warrior // 职业图标
SP_FX_Fire_00           // 火焰特效帧
```

#### 2.1.2 Flipbook（翻页动画）

**在本项目中的用途**：
- 角色动画：待机(Idle)/移动(Move)/攻击(Attack)/受击(Hit)/死亡(Death)
- 总量：54职业 × 5动画状态 = 约270个Flipbook
- 部分职业有额外动画（如施法Cast、劝降Surrender等）

**Flipbook属性**：

| 属性 | 说明 | 本项目设置 |
|------|------|-----------|
| Frames | 帧序列 | 按动画状态组织 |
| Frame Rate | 帧率 | 8-12fps（像素动画典型帧率） |
| Loop | 是否循环 | Idle/Move循环，Attack/Hit/Death不循环 |
| Play Rate | 播放速率 | 1.0（可被技能加速/减速效果修改） |

**Flipbook资产命名规范**：

```
FB_[职业]_[动画状态]
例：
FB_Warrior_Idle
FB_Warrior_Move
FB_Warrior_Attack
FB_Warrior_Hit
FB_Warrior_Death
FB_Mage_Cast        // 法师额外施法动画
FB_Commander_Surrender // 指挥官劝降动画
```

**Flipbook在蓝图中的动态切换**：

```cpp
// C++方式
UPaperFlipbookComponent* FlipbookComp;
FlipbookComp->SetFlipbook(NewFlipbook);
FlipbookComp->Play();
FlipbookComp->SetLooping(true);

// 蓝图方式
// 使用PaperZD的AnimBP替代手动切换（见2.3节）
```

#### 2.1.3 TileSet（图块集）

**在本项目中的用途**：
- 地形图块：草地/沙地/森林/山地/水域/城墙/道路/桥梁
- 变体：每种地形6个视觉变体，避免重复感
- 总量：8种地形 × 6变体 = 48个Tile定义

**TileSet编辑器**：

| 功能 | 说明 | 本项目应用 |
|------|------|-----------|
| 从纹理提取Tile | 将大纹理切割为单个Tile | 地形素材切割 |
| 定义Tile碰撞 | 设置每个Tile的碰撞形状 | 地形通行性判定 |
| Tile属性 | 设置物理材质、渲染属性 | 不同地形类型标记 |

**Tile碰撞与地形属性映射**：

```
TileSet中每个Tile设置：
  - Collision: 可通行/不可通行/半通行(仅远程)
  - 地形类型Tag: Terrain.Grass / Terrain.Forest / Terrain.Mountain ...
  - 高度信息: 0(平地) / 1(低地) / 2(高地)
  - 掩护等级: 0(无掩护) / 1(轻掩护) / 2(重掩护)
```

**TileSet资产命名规范**：

```
TS_[地形类型]_[变体号]
例：
TS_Grass_00 ~ TS_Grass_05
TS_Forest_00 ~ TS_Forest_05
TS_Mountain_00 ~ TS_Mountain_05
TS_Water_00 ~ TS_Water_05
```

#### 2.1.4 TileMap（瓦片地图）

**在本项目中的用途**：
- 战场地图：30章 × 1地图 = 30个TileMap资产
- 每个TileMap对应一个战场关卡

**TileMap多层结构**：

| 层名 | 用途 | 碰撞 |
|------|------|------|
| Ground | 地面层（草地/沙地/水域） | 定义通行性 |
| Structure | 建筑层（城墙/桥梁/掩体） | 定义不可通行/掩护 |
| Decoration | 装饰层（花草/碎石/旗帜） | 无碰撞 |
| Overlay | 覆盖层（高亮格子/移动范围） | 无碰撞，运行时动态 |

**TileMap与战场数据关联**：

```
TileMap资产（纯视觉）
    ↕ 关联
DT_ChapterMap（配置表，存储地形属性/初始部署点/增援点/胜利条件）
    ↕ 关联
BP_BattleGrid（运行时Actor，管理格子逻辑）
```

**TileMap资产命名规范**：

```
TM_Ch[章节号]_[地图名]
例：
TM_Ch01_Plains
TM_Ch05_Fortress
TM_Ch10_Swamp
TM_Ch30_FinalBattle
```

### 2.2 Paper2D的局限性（关键风险）

#### 2.2.1 无内置Animation Blueprint

**问题描述**：Paper2D原生的Flipbook切换需要手动在蓝图中用Switch/Select节点管理，54职业×5状态=270个Flipbook的手动管理会导致蓝图极其臃肿、难以维护。

**影响等级**：🔴 严重

**应对方案**：使用PaperZD插件（见2.3节），提供与3D AnimBP类似的2D动画状态机。

#### 2.2.2 无骨骼动画

**问题描述**：Paper2D仅支持Flipbook帧动画，不支持骨骼变形动画。角色无法做局部动画（如仅手臂挥动），每帧都需要完整绘制。

**影响等级**：🟡 中等

**应对方案**：
- 接受纯帧动画方案，美术管线按帧绘制
- 如需局部动画，可使用SpriteStudio插件（可选）
- 部分角色可拆分为多层Sprite（身体+武器），通过代码控制武器层旋转

#### 2.2.3 无2D物理引擎

**问题描述**：UE物理引擎(Chaos)是3D的，Paper2D没有专用2D物理。2D碰撞检测需要用3D碰撞体近似。

**影响等级**：🟡 中等（本项目回合制不需要实时物理）

**应对方案**：
- 本项目是回合制，不需要实时物理模拟
- 网格碰撞使用自定义AABB/网格系统，不依赖物理引擎
- 移动范围计算使用BFS/A*算法，不走物理

#### 2.2.4 TileMap编辑器功能有限

**问题描述**：Paper2D的TileMap编辑器不支持大型地图的高级功能（如自动地形过渡、随机刷、条件刷等）。

**影响等级**：🟡 中等

**应对方案**：
- 小型战场（如16×12格）手动编辑可接受
- 如需自动地形过渡，自研Editor Utility Widget辅助
- 地形属性通过DT_ChapterMap配置表管理，不依赖TileMap资产本身

#### 2.2.5 无内置2D光照系统

**问题描述**：Paper2D没有2D法线贴图光照系统，无法实现2D法线光照效果。

**影响等级**：🟢 低（像素风格通常不需要2D光照）

**应对方案**：
- 像素风格使用Unlit材质，不需要动态光照
- 天气/昼夜效果通过后处理/颜色叠加实现
- 如需2D光照，可用自定义Sprite材质+2D法线贴图方案

#### 2.2.6 移动端DrawCall优化

**问题描述**：每个Sprite是独立DrawCall，大量Sprite同时渲染会导致DrawCall过高，移动端性能瓶颈。

**影响等级**：🔴 严重

**应对方案**：
- Sprite Atlas：将同类型Sprite合并到同一纹理，减少DrawCall
- 按需加载：只加载当前战场需要的Flipbook
- 合批渲染：使用材质实例而非材质
- 详见第十章移动端优化

### 2.3 PaperZD插件（强烈推荐）

#### 2.3.1 插件概述

PaperZD是Paper2D的第三方扩展插件，为本项目提供关键的2D动画管理能力。没有PaperZD，54种职业的动画管理将极其混乱。

**插件来源**：GitHub开源，MIT协议  
**UE5.7兼容性**：需确认最新版本兼容性，必要时Fork维护

#### 2.3.2 Animation Blueprint for 2D

**核心价值**：为2D角色提供与3D AnimBP类似的状态机工作流。

**本项目状态机设计**：

```
                    ┌──────────┐
          ┌────────→│  Idle    │←────────┐
          │         └────┬─────┘         │
          │              │ MoveTrigger   │ StopTrigger
          │              ↓               │
          │         ┌──────────┐         │
          │    ┌───→│  Move    │─────────┘
          │    │    └────┬─────┘
          │    │         │ AttackTrigger
          │    │         ↓
          │    │    ┌──────────┐
          │    │    │  Attack  │─────────┐
          │    │    └────┬─────┘         │
          │    │         │ HitTrigger    │ AttackEnd
          │    │         ↓               │
          │    │    ┌──────────┐         │
          │    │    │   Hit    │─────────┘
          │    │    └────┬─────┘
          │    │         │ DeathTrigger
          │    │         ↓
          │    │    ┌──────────┐
          │    └────│  Death   │ (终态，不返回)
          │         └──────────┘
          │
          │  (AttackEnd/HitEnd → Idle)
          └────────────────────────────────
```

**状态转换规则**：

| 源状态 | 目标状态 | 触发条件 | 说明 |
|--------|----------|----------|------|
| Idle | Move | 开始移动 | 循环播放 |
| Move | Idle | 停止移动 | 循环播放 |
| Idle | Attack | 发起攻击 | 单次播放 |
| Attack | Idle | 攻击动画结束 | 自动转换 |
| Any | Hit | 受到伤害 | 打断当前动画 |
| Hit | Idle | 受击动画结束 | 自动转换 |
| Any | Death | HP归零 | 单次播放，终态 |
| Idle | Cast | 施法（法师类） | 单次播放 |
| Cast | Idle | 施法动画结束 | 自动转换 |

**AnimBP资产命名**：

```
ABP_[职业类别]
例：
ABP_Warrior      // 战士类AnimBP
ABP_Mage         // 法师类AnimBP
ABP_Archer       // 弓手类AnimBP
ABP_Commander    // 指挥官类AnimBP
```

### 2.4 Paper2D渲染管线

#### 2.4.1 Sprite渲染原理

Paper2D的Sprite本质是一个平面Mesh（由4个顶点组成的Quad），配合材质渲染纹理。每个Sprite在场景中生成一个独立的Mesh Component。

**渲染流程**：

```
Sprite资产 → UPaperSpriteComponent → 生成Quad Mesh → 材质绑定纹理 → 渲染到屏幕
```

#### 2.4.2 内置材质类型

| 材质类型 | 渲染方式 | 适用场景 | 本项目使用 |
|----------|----------|----------|-----------|
| Unlit Masked | 无光照，Alpha遮罩 | 像素角色、UI | ✅ 主要使用 |
| Lit Opaque | 有光照，不透明 | 3D物体 | ❌ 不使用 |
| Translucent | 半透明 | 特效、UI | ✅ 部分使用 |

#### 2.4.3 自定义Sprite材质

**SpriteTextureSampler节点**：Paper2D提供专用的材质节点`SpriteTextureSampler`，自动绑定当前Sprite的纹理，无需手动设置纹理参数。

**像素风格自定义材质示例**：

```
Material: M_Sprite_Pixel
  - Shading Model: Unlit
  - Blend Mode: Masked
  - 两面渲染: false（像素风格不需要背面）
  
  节点图:
  SpriteTextureSampler → RGB → Emissive Color
  SpriteTextureSampler → A → Opacity Mask
  
  材质属性:
  - Dither Opacity Mask: false（像素风格不需要抖动）
  - Opacity Mask Clip Value: 0.5
```

**带颜色叠加的Sprite材质**（用于士气变色、受击闪白等）：

```
Material: M_Sprite_ColorOverlay
  - 基于 M_Sprite_Pixel 扩展
  - 额外参数:
    - TintColor (Vector): 颜色叠加
    - TintStrength (Scalar): 叠加强度 0~1
    - FlashWhite (Scalar): 闪白强度 0~1
  
  节点图:
  SpriteTextureSampler → RGB → Lerp(原色, TintColor, TintStrength) → Lerp(结果, White, FlashWhite) → Emissive
```

#### 2.4.4 2D+3D混合渲染方案

**本项目采用分层渲染策略**：

| 层 | 渲染方式 | 内容 | 技术 |
|----|----------|------|------|
| 底层 | 2D渲染 | 战场TileMap | Paper2D TileMap |
| 中层 | 2D渲染 | 角色Sprite | Paper2D Sprite + PaperZD |
| 特效层 | 3D渲染 | 战斗特效 | Niagara Sprite Renderer |
| 顶层 | UI渲染 | HUD/菜单 | UMG/CommonUI |

**2D与3D的Z排序**：

```
场景Z轴布局（俯视角）:
  Z = 0:  TileMap地面层
  Z = 10: TileMap建筑层
  Z = 20: 角色Sprite（按Y轴排序实现前后遮挡）
  Z = 30: 特效层（Niagara）
  Z = 100: UI层（UMG，独立于3D场景）
```

**Sprite排序策略**：

```cpp
// 角色Sprite按Y坐标排序，实现前后遮挡
UPaperFlipbookComponent::SetTranslucentSortPriority(0);
// 在Tick中更新排序
FlipbookComp->SetSortOrder(FMath::FloorToInt(GetActorLocation().Y));
```

## 三、Gameplay Ability System (GAS) 分析

### 3.1 GAS核心概念映射

#### 3.1.1 Ability System Component (ASC)

**挂载目标**：每个小队Actor（BP_Character_Squad）

**ASC初始化**：

```cpp
// BP_Character_Squad.h
UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "AbilitySystem")
UAbilitySystemComponent* AbilitySystemComponent;

// BP_Character_Squad.cpp
AbilitySystemComponent = CreateDefaultSubobject<UAbilitySystemComponent>(TEXT("AbilitySystemComponent"));
AbilitySystemComponent->SetIsReplicated(true);
AbilitySystemComponent->SetReplicationMode(EGameplayEffectReplicationMode::Mixed);
// Mixed模式：玩家控制的Actor用Minimal，AI控制的用Full
```

**ASC的职责**：

| 职责 | 说明 | 本项目应用 |
|------|------|-----------|

#### 3.2.2 战斗结算流程

```
1. 玩家选择攻击目标
2. 客户端发送攻击意图到服务端
3. 服务端验证合法性（距离/状态/回合）
4. 服务端激活GA_AttackMelee
5. GA计算伤害:
   a. 获取攻击方属性（ATK/暴击率/命中率）
   b. 获取防御方属性（DEF/闪避率/护甲类型）
   c. 获取环境修正（地形/天气/士气/羁绊/科技）
   d. 调用S18属性计算引擎计算最终伤害
   e. 创建GE_Damage并应用到目标
6. 服务端广播结果到所有客户端
7. 客户端播放动画+特效+UI反馈
```

#### 3.2.3 Buff/Debuff管理

```
Buff/Debuff通过GE的Duration Policy管理:
- Instant: 即时效果（伤害/治疗）
- Duration: 限时效果（天气/士气/临时Buff）
  - Duration Magnitude: ScalableFloat（可配置回合数）
  - Period: 每回合Tick（如毒伤）
- Infinite: 永久效果（科技/羁绊/装备）
  - 通过RemoveActiveEffectsWithTag手动移除

Buff/Debuff的叠加规则:
- Stacking Type: Aggregate (聚合)
- Stack Limit Count: 配置表定义
- Stack Duration Refresh Policy: RefreshOnSuccessfulApplication
- Stack Period Reset Policy: NeverReset
```

### 3.3 GAS与S18属性计算引擎的融合

#### 3.3.1 融合架构

S18属性计算引擎定义了属性的计算规则，GAS提供了属性容器和修改器系统。两者融合方案：

```
S18定义的计算规则          GAS提供的执行机制
┌──────────────┐          ┌──────────────┐
│ add类型mod   │  ──→     │ ModifierOp:: │
│ (加法修正)    │          │   Add        │
├──────────────┤          ├──────────────┤
│ multiply类型 │  ──→     │ ModifierOp:: │
│ mod(乘法修正) │          │   Multiply   │
├──────────────┤          ├──────────────┤
│ 先加后乘规则 │  ──→     │ Custom       │
│              │          │ Execution    │
├──────────────┤          ├──────────────┤
│ 条件修正     │  ──→     │ Modifier     │
│ (地形/天气)  │          │   Conditional│
└──────────────┘          └──────────────┘
```

#### 3.3.2 自定义ModifierMagnitudeCalculation (MMC)

```cpp
UCLASS()
class UMMC_DamageCalculation : public UGameplayModifierMagnitudeCalculation
{
    GENERATED_BODY()
public:
    virtual float CalculateBaseMagnitude_Implementation(
        const FGameplayEffectSpec& Spec) const override
    {
        // 1. 获取攻击方属性
        float AttackerATK = 0.f;
        GetAttributeCaptured(AttributeATK, Spec, AttackerATK);
        
        float CritRate = 0.f;
        GetAttributeCaptured(AttributeCritRate, Spec, CritRate);
        
        // 2. 获取防御方属性
        float DefenderDEF = 0.f;
        GetAttributeCaptured(AttributeDEF, Spec, DefenderDEF);
        
        float DodgeRate = 0.f;
        GetAttributeCaptured(AttributeDodgeRate, Spec, DodgeRate);
        
        // 3. 调用S18计算引擎
        FS18DamageParams Params;
        Params.AttackPower = AttackerATK;
        Params.DefensePower = DefenderDEF;
        Params.CritRate = CritRate;
        Params.DodgeRate = DodgeRate;
        // ... 设置其他修正因子
        
        float FinalDamage = US18StatEngine::CalculateDamage(Params);
        return -FinalDamage; // 负值表示扣血
    }
};
```

#### 3.3.3 自定义GameplayEffectExecution

{

public:

        const FGameplayEffectCustomExecutionParameters& ExecutionParams,
        OUT FGameplayEffectCustomExecutionOutput& OutExecutionOutput) const override

        // 获取Source和Target的ASC
        UAbilitySystemComponent* SourceASC = ExecutionParams.GetSourceAbilitySystemComponent();
        UAbilitySystemComponent* TargetASC = ExecutionParams.GetTargetAbilitySystemComponent();

        FS18DamageContext Context;
        // ... 填充Context

**战斗流程状态机**：

```

                ↑              │

**详细状态**：

```
BattleInit:
  - 加载战场TileMap
  - 部署玩家小队
  - 部署敌方小队
  - 初始化天气
  - 初始化神力法力值

PlayerTurn:
  - 子状态: SelectSquad → SelectAction → ExecuteAction → NextSquad
  - 玩家依次操作每个未行动的小队
  - 可使用神力
  - 所有小队行动完毕或点击"结束回合"

EnemyTurn:
  - AI行为树控制敌方小队行动
  - 依次执行AI决策
  - 所有敌方小队行动完毕

BattleEnd:
  - 判定胜负
  - 计算星级评价
  - 发放奖励
  - 保存进度

### 4.2 Actor组件架构

#### 4.2.1 组件层级图

```
BP_Character_Squad
├── UPaperFlipbookComponent (Sprite渲染)
│   └── PaperZD AnimInstance (动画状态机)

│   └── UBattleAttributeSet (战斗属性)

├── UBoxComponent (网格碰撞)
├── UChildActorComponent × N (阵型中的单位)
│   └── BP_Actor_Unit
└── UBillboardComponent (选中指示器)
```

#### 4.2.2 组件详细说明

**UPaperFlipbookComponent**：
- 渲染小队Sprite
- 通过PaperZD驱动动画状态机
- 排序模式：按Y轴排序实现前后遮挡
- 材质：M_Sprite_ColorOverlay（支持变色/闪白）

**UAbilitySystemComponent**：
- 复制模式：Mixed（玩家Minimal，AI Full）
- 初始化时机：Possess后
- 技能授予：从DT_Class配置表读取，InitializeSquad时授予

**UWidgetComponent**：
- 渲染头顶HP条和士气条
- 空间模式：Screen（始终面向相机）
- 绘制尺寸：100×30
- 可见性：始终显示友方，敌方选中时显示

**UBoxComponent**：
- 网格碰撞体
- 尺寸：与格子大小匹配（如64×64）
- 碰撞通道：自定义GridObject通道
- 用于鼠标点击检测和网格占用判定

**UChildActorComponent**：
- 阵型中的单位（1~5个）
- 每个Unit是独立的ChildActor
- 纯视觉，不参与战斗逻辑
- 位置由FormationData定义

## 九、网络架构

### 9.1 服务端权威模型

#### 9.1.1 架构原则

| 原则 | 说明 |
|------|------|
| 服务端权威 | 所有战斗逻辑在服务端计算 |
| 客户端预测 | 客户端可预测移动/动画，但以服务端结果为准 |
| 防作弊 | 伤害/概率/抽卡全部服务端计算 |
| 最小复制 | 只复制必要数据，减少带宽 |

#### 9.1.2 客户端-服务端交互

```
客户端                           服务端
  │                                │
  │─── 移动请求(目标格子) ────────→│
  │                                │ 验证合法性
  │                                │ 计算路径
  │                                │ 执行移动
  │←── 移动结果(路径+位置) ───────│
  │                                │
  │─── 攻击请求(目标小队) ────────→│
  │                                │ 验证距离/状态
  │                                │ 计算伤害(S18引擎)
  │                                │ 应用GE_Damage
  │                                │ 检查死亡/士气
  │←── 攻击结果(伤害/暴击/死亡) ──│
  │                                │
  │─── 抽卡请求(池ID+次数) ──────→│
  │                                │ 服务端随机
  │                                │ 计算保底
  │←── 抽卡结果(物品列表) ────────│
```

### 9.2 复制策略

#### 9.2.1 属性复制方式

| 数据类型 | 复制方式 | 理由 |
|----------|----------|------|
| 小队位置 | RepNotify | 位置变化需要客户端执行移动动画 |
| 小队属性(HP等) | RepNotify | 属性变化需要客户端更新UI |
| 战斗结算 | Multicast RPC | 一次性广播给所有客户端 |
| 玩家操作 | Server RPC | 客户端向服务端发送请求 |
| 回合状态 | GameState复制 | 全局状态，自动复制 |
| 天气变化 | GameState复制 | 全局状态，自动复制 |

#### 9.2.2 RepNotify实现

```cpp
// 小队位置RepNotify
UPROPERTY(ReplicatedUsing = OnRep_GridPosition, BlueprintReadOnly)
FIntPoint GridPosition;

UFUNCTION()
void OnRep_GridPosition()
{
    // 客户端收到新位置后，播放移动动画
    PlayMoveAnimation(GridPosition);
    // 更新战场网格显示
    UpdateGridHighlight();
}

// 小队HP RepNotify
UPROPERTY(ReplicatedUsing = OnRep_Health, BlueprintReadOnly)
float Health;

UFUNCTION()
void OnRep_Health()
{
    // 更新头顶HP条
    UpdateHealthBar();
    // 如果HP降低，播放受击效果
    if (Health < PreviousHealth)
    {
        PlayHitEffect();
    }
}
```

#### 9.2.3 RPC使用规范

| RPC类型 | 方向 | 使用场景 | 本项目应用 |
|---------|------|----------|-----------|
| Server RPC | Client→Server | 客户端请求 | 移动/攻击/治疗/抽卡 |
| Multicast RPC | Server→All | 广播结果 | 战斗结算/特效触发 |
| Client RPC | Server→特定Client | 针对性通知 | 错误提示/个人奖励 |

**Server RPC示例**：

```cpp
// 客户端调用，服务端执行
UFUNCTION(Server, Reliable, WithValidation)
void Server_RequestMove(FIntPoint TargetGrid);

bool Server_RequestMove_Validate(FIntPoint TargetGrid)
{
    // 验证请求合法性
    return IsValidGridPosition(TargetGrid) && !bHasActed;
}

void Server_RequestMove_Implementation(FIntPoint TargetGrid)
{
    // 服务端执行移动逻辑
    GridPosition = TargetGrid;
    bHasActed = true;
}
```

## 十、移动端优化

### 10.1 性能挑战

| 挑战 | 严重程度 | 说明 |
|------|----------|------|
| UE5移动端包体大 | 🔴 严重 | UE5默认包体1GB+，需精简到500MB以下 |
| DrawCall过多 | 🔴 严重 | 每个Sprite独立DrawCall，战场20+小队=20+DC |
| 内存占用 | 🟡 中等 | 54职业×270动画，全加载需大量内存 |
| 电量/发热 | 🟡 中等 | UE5渲染管线较重，需精简 |
| 加载时间 | 🟡 中等 | 资源多，首次加载慢 |

### 10.2 优化方案

#### 10.2.1 Sprite Atlas（精灵图集）

**核心策略**：将多个Sprite合并到同一纹理，减少DrawCall和内存碎片。

```
Sprite Atlas组织方式:
├── SA_Units_Warrior    (战士类所有方向+动画帧)
├── SA_Units_Mage       (法师类所有方向+动画帧)
├── SA_Units_Archer     (弓手类所有方向+动画帧)
├── SA_Terrain_Grass    (草地所有变体)
├── SA_Terrain_Forest   (森林所有变体)
├── SA_UI_Icons         (UI图标合集)
├── SA_UI_Common        (UI通用元素)
└── SA_FX_Battle        (战斗特效帧)

Atlas设置:
  - Max Texture Size: 2048×2048
  - Padding: 2px (防止像素渗透)
  - Compression: ASTC 6×6 (移动端) / BC7 (PC)
  - Mipmaps: Disabled
  - Filtering: Nearest
```

**DrawCall优化预期**：

| 优化前 | 优化后 | 说明 |
|--------|--------|------|
| 20+ DC (每Sprite独立) | 5-8 DC (同Atlas合批) | 同Atlas的Sprite可合批渲染 |

#### 10.2.2 Flipbook按需加载

```
加载策略:
1. 进入战斗时，只加载当前战场涉及的职业Flipbook
2. 主界面只加载待机动画
3. 编队界面加载待机+攻击动画（预览用）
4. 未使用的Flipbook及时卸载

实现:
- 使用TSoftObjectPtr<UPaperFlipbook>延迟加载
- 使用FStreamableManager异步加载
- 加载完成后缓存，卸载时清空缓存
```

#### 10.2.3 战场TileMap分块流式加载

```
对于大型战场(>20×15):
1. 将TileMap分为4×4的块
2. 只加载相机可见区域±1块的TileMap
3. 相机移动时动态加载/卸载

对于小型战场(≤20×15):
- 整体加载，无需分块
```

## 十一、Substrate材质系统

### 11.1 UE5.7 Substrate概述

UE5.7默认启用Substrate Materials（前身为Stratum），是UE的新一代材质/光照系统，取代了传统的固定着色模型（Default Lit、Subsurface、Clear Coat等）。

**Substrate核心变化**：
- 材质不再选择固定Shading Model
- 使用SL（Substrate Layer）节点组合材质属性
- 支持多层材质混合
- 更物理准确的光照计算

### 11.2 本项目Substrate应用

#### 11.2.1 2D渲染（主要）— 不使用Substrate

本项目2D渲染使用Unlit材质，不涉及光照计算，Substrate对Unlit材质没有影响。

```
M_Sprite_Pixel:
  - Shading Model: Unlit
  - Substrate影响: 无
  - 材质图: SpriteTextureSampler → Emissive
```

#### 11.2.2 3D特效（次要）— 可选Substrate

3D战斗特效（Niagara Mesh Renderer中的Mesh）可使用Substrate：

```
特效Mesh材质（可选Substrate）:
  - 火焰: Substrate Slab + Emissive + Transmission
  - 冰霜: Substrate Slab + Dielectric + SSS
  - 金属: Substrate Slab + Metal
  - 皮肤: Substrate Slab + Dielectric + SSS

注意：移动端特效建议仍用Unlit，Substrate仅PC端使用
```

#### 11.2.3 CG过场动画 — 使用Substrate

如果项目包含预渲染CG或高质量过场动画，可充分利用Substrate：

```
CG角色材质:
  - 皮肤: Substrate Slab + Dielectric + SSS + Detail Normal
  - 金属盔甲: Substrate Slab + Metal + ClearCoat
  - 布料: Substrate Slab + Dielectric + Fuzz
  - 头发: Substrate Slab + Dielectric + SSS + Tangent Normal
```

### 11.3 像素风格Sprite的Substrate注意事项

```
像素风格Sprite关键设置:
  - Texture Filtering: Nearest (不受Substrate影响)
  - Mipmap: Disabled (不受Substrate影响)
  - 材质: Unlit (不受Substrate影响)
  
结论: Substrate对本项目的2D核心渲染无影响
       仅在3D特效/CG场景中考虑使用
```

## 十二、蓝图 vs C++ 决策

### 12.1 蓝图实现

| 模块 | 具体内容 | 理由 |
|------|----------|------|
| UI逻辑 | 所有WBP的交互逻辑 | UMG天然蓝图友好 |
| 动画状态机 | PaperZD AnimBP | 可视化状态机编辑 |
| 简单技能效果 | 部分GA/GE | 快速原型验证 |
| 关卡配置 | 战场部署、触发器 | 可视化编辑 |
| 原型验证 | 新功能快速原型 | 迭代速度快 |
| 动画通知处理 | AnimNotify回调 | 简单事件响应 |
| 天气系统 | 天气切换逻辑 | 简单状态机 |
| 对话系统 | 剧情对话流程 | 可视化流程编辑 |

### 12.2 C++实现

| 模块 | 具体内容 | 理由 |
|------|----------|------|
| GAS核心 | AttributeSet/GE/GA基类 | 性能+类型安全 |
| 战斗结算引擎 | 伤害计算/回合管理 | 性能热点+服务端权威 |
| 属性计算引擎 | S18公式计算 | 性能热点+复杂逻辑 |
| AI行为树 | 行为树节点/决策逻辑 | 性能+复杂逻辑 |
| 网络复制 | RPC/RepNotify | 网络层需要C++控制 |
| 性能热点 | 网格计算/路径搜索 | A*/BFS算法性能 |
| 自定义MMC | GAS属性计算 | 需要与S18引擎深度集成 |
| 数据结构 | DataTable行定义 | 类型安全+蓝图暴露 |
| 输入处理 | Enhanced Input回调 | 底层输入处理 |

### 12.3 蓝图与C++协作模式

```
C++基类 → 定义接口和核心逻辑
    ↓ 继承
蓝图子类 → 配置具体参数和视觉表现

示例:
  C++: ABP_Character_Squad (定义组件、属性、网络复制)
  BP: BP_Squad_Warrior (配置战士的Sprite、动画、属性值)
  BP: BP_Squad_Mage (配置法师的Sprite、动画、属性值)

C++: UGA_AttackMelee (定义攻击逻辑、伤害计算)
  BP: GA_AttackMelee_Warrior (配置战士攻击参数)
  BP: GA_AttackMelee_Archer (配置弓手攻击参数)
```

## 十六、开发环境配置

### 16.1 必须启用的插件

| 插件 | 类别 | 说明 | 启用方式 |
|------|------|------|----------|
| Paper 2D | 内置-渲染 | 2D渲染核心 | 编辑器→编辑→插件→Paper 2D→启用 |
| Gameplay Ability System | 内置-游戏 | 技能/属性/Buff系统 | 编辑器→编辑→插件→Gameplay Abilities→启用 |
| Enhanced Input | 内置-输入 | 新输入系统 | UE5.7默认启用 |
| CommonUI | 内置-UI | 跨平台UI框架 | 编辑器→编辑→插件→CommonUI→启用 |
| Data Registry | 内置-数据 | 数据管理 | 编辑器→编辑→插件→Data Registry→启用 |
| Niagara | 内置-特效 | 特效系统 | UE5.7默认启用 |
| PaperZD | 第三方 | 2D Animation Blueprint | 手动安装到Plugins目录→启用 |

**插件启用步骤**：

```
1. 创建UE5.7项目（Blank模板）
2. 编辑 → 插件 → 搜索并启用上述内置插件
3. 重启编辑器
4. 下载PaperZD，放入项目Plugins目录
5. 重新生成项目文件
6. 编译项目
```

### 16.2 必须关闭的功能

| 功能 | 关闭方式 | 理由 |
|------|----------|------|
| Nanite | 项目设置→Rendering→Nanite→取消勾选 | 2D不需要，节省GPU和包体 |
| Lumen | 项目设置→Rendering→Global Illumination→None | 2D不需要，节省GPU |
| TSR | 项目设置→Rendering→Anti-Aliasing→FXAA | 2D不需要，节省GPU |
| Virtual Shadow Maps | 项目设置→Rendering→Shadow Map Method→ShadowMap | 2D不需要 |
| Ray Tracing | 项目设置→Rendering→Ray Tracing→取消勾选 | 2D不需要 |
| Ambient Occlusion | 项目设置→Rendering→Ambient Occlusion→取消勾选 | 2D不需要 |
| Auto Exposure | 项目设置→Rendering→Auto Exposure→取消勾选 | 手动控制曝光 |
| Motion Blur | 项目设置→Rendering→Motion Blur→取消勾选 | 2D不需要 |
| Depth of Field | 项目设置→Rendering→Depth of Field→取消勾选 | 2D不需要 |
