## 1. 系统概述

### 1.1 核心定位

日常/周常/主线/成就四类任务系统，驱动玩家每日行为。通过活跃度奖励机制引导玩家体验核心玩法循环，成就系统提供长期追求目标。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 行为驱动 | 任务引导玩家体验核心玩法（战斗/转职/招募/社交） |
| 活跃度闭环 | 日常/周常活跃度奖励驱动每日登录和玩法覆盖 |
| 渐进解锁 | 主线任务跟随章节进度，成就分阶段目标 |
| 事件驱动 | 条件判定基于游戏事件触发，非轮询 |
| 零遗漏 | 奖励可补领，进度不丢失 |

### 1.3 关键参数总览

| 参数 | 值 | 备注 |
|------|----|------|
| 日常任务数 | 8个/天 | 每日5:00刷新 |
| 周常任务数 | 10个/周 | 每周一5:00刷新 |
| 日常活跃度上限 | 120 | 8个任务合计 |
| 周常活跃度上限 | 240 | 10个任务合计 |
| 成就总数 | 约100个 | 分8类 |
| 日常活跃度奖励档位 | 30/60/90/120 | 4档 |
| 周常活跃度奖励档位 | 80/160/240 | 3档 |
| 任务刷新时间 | 5:00 | 与其他系统一致 |

## 4. 日常任务设计

### 4.1 日常任务规则

| 规则 | 说明 |
|------|------|
| 任务数量 | 每日8个，从任务池随机抽取 |
| 刷新时间 | 每日5:00 |
| 刷新行为 | 清空当日所有任务进度，重新抽取 |
| 活跃度上限 | 120（8个任务合计） |
| 未领取奖励 | 刷新时自动通过邮件补发 |

### 4.2 日常活跃度奖励

| 档位 | 所需活跃度 | 奖励 |
|------|-----------|------|
| 1 | 30 | 金币×5000 |
| 2 | 60 | 经验书×5 |
| 3 | 90 | 钻石×50 |
| 4 | 120 | 抽卡券×1 |

### 4.3 日常任务抽取算法

```csharp
public List<TB_DailyQuest> SelectDailyQuests(List<TB_DailyQuest> pool, int count = 8)
{
    int minTotalActivity = 90;
    int maxAttempts = 100;

    for (int attempt = 0; attempt < maxAttempts; attempt++)
    {
        var selected = WeightedRandom.Select(pool, count);
        int totalActivity = selected.Sum(q => q.activity_points);
        if (totalActivity >= minTotalActivity)
        {
            return selected.OrderBy(q => q.sort_order).ToList();
        }
    }

    return pool.OrderBy(q => q.sort_order).Take(count).ToList();
}
```

### 4.4 日常任务刷新流程

```
RefreshDailyQuests(player):
  ┌─────────────────────────────────────────────┐
  │ Step 1: 补发未领取奖励                        │
  │   - 遍历已完成未领奖的日常任务                  │
  │   - 通过邮件补发奖励                          │
  │   - 补发未领取的活跃度档位奖励                 │
  └──────────────────┬──────────────────────────┘
                     ▼
  ┌─────────────────────────────────────────────┐
  │ Step 2: 重置日常数据                          │
  │   - DailyActivityPoints = 0                  │
  │   - ClaimedDailyActivityRewards.Clear()      │
  │   - DailyQuests.Clear()                      │
  └──────────────────┬──────────────────────────┘
                     ▼
  ┌─────────────────────────────────────────────┐
  │ Step 3: 抽取新日常任务                        │
  │   - 从TB_DailyQuest池中抽取8个               │
  │   - 创建QuestData实例，State=InProgress      │
  │   - 设置ExpireTime为次日5:00                  │
  └─────────────────────────────────────────────┘
```

## 6. 主线任务设计

### 6.1 主线任务规则

| 规则 | 说明 |
|------|------|
| 任务来源 | 跟随章节解锁 |
| 每章任务数 | 3个 |
| 解锁条件 | 完成前置任务 + 达到章节进度要求 |
| 不会过期 | 无时间限制 |
| 不可跳过 | 必须按顺序完成 |

### 6.2 主线任务流程

```
CheckMainQuestUnlock(player, chapterId):
  ┌─────────────────────────────────────────────┐
  │ Step 1: 查询章节第一个任务                    │
  │   - pre_quest_id == 0 的任务                  │
  │   - 检查章节进度是否满足                      │
  └──────────────────┬──────────────────────────┘
                     ▼
  ┌─────────────────────────────────────────────┐
  │ Step 2: 解锁任务                             │
  │   - State = Available → InProgress           │
  │   - 添加到PlayerQuestData.MainQuests         │
  └──────────────────┬──────────────────────────┘
                     ▼
  ┌─────────────────────────────────────────────┐
  │ Step 3: 完成后解锁后续任务                    │
  │   - 当前任务State = Completed                │
  │   - 查找next_quest_id对应的任务              │
  │   - 解锁下一个任务                           │
  └─────────────────────────────────────────────┘
```

### 8.3 条件匹配器

```csharp
public class QuestConditionMatcher
{
    public static bool ParamsMatch(string conditionJson, Dictionary<string, object> eventData)
    {
        var condition = JsonUtility.FromJson<QuestCondition>(conditionJson);

        if (condition.@params == null || condition.@params.Count == 0)
            return true;

        foreach (var kvp in condition.@params)
        {
            if (!eventData.ContainsKey(kvp.Key))
                return false;

            if (!eventData[kvp.Key].Equals(kvp.Value))
                return false;
        }

        return true;
    }
}
```

### 8.4 进度更新逻辑

```csharp
public void UpdateProgress(QuestData quest, Dictionary<string, object> eventData)
{
    var condition = JsonUtility.FromJson<QuestCondition>(quest.ConditionJson);

    switch (condition.type)
    {
        case "count":
            quest.ProgressCurrent += 1;
            break;

        case "reach":
            if (eventData.TryGetValue("value", out var val))
            {
                int intValue = Convert.ToInt32(val);
                quest.ProgressCurrent = Math.Max(quest.ProgressCurrent, intValue);
            }
            break;

        case "collect":
            quest.ProgressCurrent += 1;
            break;

        case "complete":
            quest.ProgressCurrent = 1;
            break;
    }

    quest.ProgressCurrent = Math.Min(quest.ProgressCurrent, quest.ProgressTarget);

    if (quest.ProgressCurrent >= quest.ProgressTarget && quest.State == QuestState.InProgress)
    {
        quest.State = QuestState.Completed;
    }
}
```

```csharp
public bool CanClaimActivityReward(PlayerQuestData playerData, int threshold, QuestType type)
{
    int currentActivity = type == QuestType.Daily
        ? playerData.DailyActivityPoints
        : playerData.WeeklyActivityPoints;

    var claimedList = type == QuestType.Daily
        ? playerData.ClaimedDailyActivityRewards
        : playerData.ClaimedWeeklyActivityRewards;

    return currentActivity >= threshold && !claimedList.Contains(threshold);
}
```

### 9.3 邮件补发机制

| 场景 | 处理方式 |
|------|---------|
| 日常任务已完成未领奖 | 刷新时通过邮件补发任务奖励和活跃度奖励 |
| 周常任务已完成未领奖 | 刷新时通过邮件补发 |
| 成就奖励未领奖 | 不过期，下次登录可领 |
| 主线任务奖励未领奖 | 不过期，下次登录可领 |

```csharp
public void CompensateUnclaimedRewards(PlayerQuestData playerData, QuestType type)
{
    var quests = type == QuestType.Daily ? playerData.DailyQuests : playerData.WeeklyQuests;

    foreach (var quest in quests)
    {
        if (quest.State == QuestState.Completed)
        {
            MailService.Instance.SendMail(
                playerData.PlayerId,
                "任务奖励补发",
                $"您未领取的任务「{quest.Title}」奖励已补发",
                quest.RewardJson,
                expiryDays: 30
            );
        }
    }

    var claimedList = type == QuestType.Daily
        ? playerData.ClaimedDailyActivityRewards
        : playerData.ClaimedWeeklyActivityRewards;

    var thresholds = type == QuestType.Daily
        ? new List<int> { 30, 60, 90, 120 }
        : new List<int> { 80, 160, 240 };

    foreach (var threshold in thresholds)
    {
        if (CanClaimActivityReward(playerData, threshold, type))
        {
            var reward = GetActivityReward(type, threshold);
            MailService.Instance.SendMail(
                playerData.PlayerId,
                "活跃度奖励补发",
                $"您未领取的活跃度奖励（{threshold}档）已补发",
                reward,
                expiryDays: 30
            );
        }
    }
}
```
