## 1. 系统概述

### 1.1 核心设计

职业转职系统是单位成长的核心机制。每个单位拥有一个职业，通过战斗积累职业专精（CP），满足条件后可自由转职。

### 1.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 职业总数 | 54 | 覆盖4个阶位 |
| 阶位数 | 4 | 一阶→四阶 |
| 一阶职业数 | 4 | 初始职业 |
| 二阶职业数 | 15 | 中级职业 |
| 三阶职业数 | 25 | 高级职业 |
| 四阶职业数 | 10 | 顶级职业 |
| 转职模式 | 网状自由转职 | 同阶/跨阶均可 |
| 降阶规则 | 退还100%转职资源 | 回到来源职业 |

### 1.3 设计原则

- **网状自由转职**：不限制为线性晋升树，一个职业可晋升为多个上阶职业，同一上阶职业可由多个下阶职业晋升而来
- **降阶退资源**：降阶时全额退还转职消耗的资源，鼓励玩家尝试不同路线
- **属性继承**：转职保留已获得的等级成长，仅更换职业基础值

## 2. 数据结构（C# Class）

### 2.1 ClassData - 职业数据

```csharp
[Serializable]
public class ClassData
{
    public string class_id;           // 职业ID，如 "soldier", "champion"
    public string name;               // 职业中文名，如 "士兵", "冠军勇士"
    public int tier;                  // 阶位 1-4
    public string[] tags;             // 特性标签ID数组，如 ["heavy_armor", "mounted"]
    public string base_stats_json;    // 基础属性JSON，如 {"str":8,"skl":5,"mag":2,"spd":6,"def":5,"res":2,"hp":30,"ldr":3}
    public string passive_skill_id;   // 职业被动技能ID，如 "passive_soldier"
    public string attack_type;        // 攻击类型: "physical" | "magic" | "hybrid"
    public int attack_range;          // 攻击距离: 1=近战, 2=短程, 3=中程, 4=远程, 5=超远程
    public string attack_priority;    // 攻击优先级: "nearest" | "lowest_hp" | "highest_threat" | "healer_first" | "back_row"
    public string attack_pattern;     // 攻击模式: "single" | "aoe_line" | "aoe_cross" | "aoe_area" | "aoe_column"
    public string move_type;          // 移动类型: "walk" | "ride" | "fly"
    public string armor_type;         // 护甲类型: "none" | "light" | "heavy"
    public string size;               // 体型: "small" | "medium" | "large"
}
```

### 2.2 PromotionRule - 晋升规则

```csharp
[Serializable]
public class PromotionRule
{
    public string from_class_id;          // 来源职业ID
    public string to_class_id;            // 目标职业ID
    public int required_str;              // 需求力量
    public int required_skl;              // 需求技巧
    public int required_mag;              // 需求魔力
    public string required_resources_json;// 需求资源JSON，如 {"gold":500,"class_scroll_t2":1}
}
```

### 2.3 UnitClassState - 单位职业状态

```csharp
[Serializable]
public class UnitClassState
{
    public string unit_id;              // 单位唯一ID
    public string current_class_id;     // 当前职业ID
    public int cp_current;              // 当前CP值
    public int cp_max;                  // CP上限（= tier * 100）
    public List<ClassHistoryEntry> class_history; // 职业历史记录
}

[Serializable]
public class ClassHistoryEntry
{
    public string from_class_id;        // 来源职业ID（首次为空字符串）
    public string to_class_id;          // 转入职业ID
    public int timestamp;               // 转职时间戳（游戏内回合数）
    public string resources_consumed_json; // 本次转职消耗的资源JSON
}
```

## 5. CP（职业专精）系统

### 5.1 CP上限

```
cp_max = tier × 100
```

| 阶位 | CP上限 |
|------|--------|
| 一阶 | 100 |
| 二阶 | 200 |
| 三阶 | 300 |
| 四阶 | 400 |

### 5.2 CP获取

每次战斗结束后，参战单位获得CP：

```
cp_gain = base_cp × (1 + tech_bonus) × (1 + talent_bonus)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| base_cp | 10 | 基础CP获取量 |
| tech_bonus | 0.0 ~ 0.5 | 科技加成（如"传统武术"科技+15%=0.15） |
| talent_bonus | 0.0 ~ 0.3 | 天赋加成 |

### 5.3 CP获取规则

- 只有实际参战（进入战场）的单位才获得CP
- 单位在战斗中撤退不获得CP
- CP不可超过cp_max
- CP满后，单位UI显示"可转职"标记

### 5.4 CP重置场景

| 场景 | CP处理 |
|------|--------|
| 转职成功 | cp_current 重置为 0，cp_max 更新为新职业阶位×100 |
| 降阶 | cp_current 重置为 0，cp_max 更新为来源职业阶位×100 |
| 同阶转职 | cp_current 重置为 0，cp_max 不变 |

### 6.2 伪代码

```csharp
public class PromotionService
{
    public PromotionResult TryPromote(UnitClassState unitState, string targetClassId)
    {
        ClassData targetClass = TB_Class.Get(targetClassId);
        ClassData currentClass = TB_Class.Get(unitState.current_class_id);
        PromotionRule rule = TB_ClassPromotion.Find(unitState.current_class_id, targetClassId);

        if (rule == null)
            return PromotionResult.Fail("无此晋升路径");

        if (unitState.cp_current < unitState.cp_max)
            return PromotionResult.Fail("CP未满");

        UnitStats currentStats = GetUnitStats(unitState.unit_id);
        if (currentStats.str < rule.required_str ||
            currentStats.skl < rule.required_skl ||
            currentStats.mag < rule.required_mag)
            return PromotionResult.Fail("属性未达标");

        Resources required = Resources.FromJson(rule.required_resources_json);
        if (!PlayerInventory.HasEnough(required))
            return PromotionResult.Fail("资源不足");

        PlayerInventory.Deduct(required);

        string oldClassId = unitState.current_class_id;
        unitState.current_class_id = targetClassId;

        UpdateStatsPreserveGrowth(unitState.unit_id, currentClass, targetClass);

        LearnPassiveSkill(unitState.unit_id, targetClass.passive_skill_id);

        unitState.class_history.Add(new ClassHistoryEntry
        {
            from_class_id = oldClassId,
            to_class_id = targetClassId,
            timestamp = GameTurn.Current,
            resources_consumed_json = rule.required_resources_json
        });

        unitState.cp_current = 0;
        unitState.cp_max = targetClass.tier * 100;

        return PromotionResult.Success();
    }
}
```

### 6.3 各步骤详细说明

#### 步骤a：检查CP是否满

```
条件：unitState.cp_current >= unitState.cp_max
失败提示："职业专精尚未满级，当前 {cp_current}/{cp_max}"
```

#### 步骤b：检查属性是否达标

```
条件：unit.str >= rule.required_str
   AND unit.skl >= rule.required_skl
   AND unit.mag >= rule.required_mag
失败提示："属性不满足转职要求（需要 STR≥{r} SKL≥{s} MAG≥{m}）"
```

#### 步骤c：检查资源是否足够

```
条件：PlayerInventory.HasEnough(rule.required_resources_json) == true
失败提示："资源不足，需要 {resource_list}"
```

#### 步骤d：扣除资源

```
PlayerInventory.Deduct(Resources.FromJson(rule.required_resources_json))
记录消耗到 class_history.resources_consumed_json
```

#### 步骤e：更改职业

```
unitState.current_class_id = targetClassId
```

#### 步骤f：更新属性（保留成长，更换基础值）

详见第9节"属性变化规则"。

#### 步骤g：习得新被动技能

```
SkillSystem.Learn(unitState.unit_id, targetClass.passive_skill_id)
```

- 新被动技能直接习得，不替换旧被动
- 旧职业的被动技能保留（不遗忘）

#### 步骤h：记录到class_history

```
unitState.class_history.Add(new ClassHistoryEntry {
    from_class_id = oldClassId,
    to_class_id = targetClassId,
    timestamp = GameTurn.Current,
    resources_consumed_json = rule.required_resources_json
})
```

        │
        ▼

  │ CP重置为0         │
  └─────┬────────────┘
        │
        ▼
  ┌──────────────────┐

  └─────┬────────────┘
        │
        ▼

### 7.2 伪代码

```csharp
public DemotionResult TryDemote(UnitClassState unitState)
{
    ClassHistoryEntry lastEntry = null;
    for (int i = unitState.class_history.Count - 1; i >= 0; i--)
    {
        if (unitState.class_history[i].to_class_id == unitState.current_class_id)
        {
            lastEntry = unitState.class_history[i];
            break;
        }
    }

    if (lastEntry == null || string.IsNullOrEmpty(lastEntry.from_class_id))
        return DemotionResult.Fail("当前职业为初始职业，无法降阶");

    Resources consumed = Resources.FromJson(lastEntry.resources_consumed_json);
    PlayerInventory.Refund(consumed);

    ClassData sourceClass = TB_Class.Get(lastEntry.from_class_id);
    ClassData currentClass = TB_Class.Get(unitState.current_class_id);

    unitState.current_class_id = lastEntry.from_class_id;

    UpdateStatsPreserveGrowth(unitState.unit_id, currentClass, sourceClass);

    unitState.cp_current = 0;
    unitState.cp_max = sourceClass.tier * 100;

    unitState.class_history.Remove(lastEntry);

    return DemotionResult.Success();
}
```

### 7.3 降阶规则细节

| 规则 | 说明 |
|------|------|
| 降阶方向 | 只能退回到上一次转职前的职业（来源职业） |
| 资源退还 | 100%退还，按class_history记录的resources_consumed_json原样退还 |
| CP处理 | 重置为0，cp_max按来源职业阶位重新计算 |
| 属性处理 | 按属性变化规则（第9节）恢复 |
| 被动技能 | 降阶后保留已习得的所有被动技能，不遗忘 |
| 历史记录 | 移除对应的class_history条目 |
| 连续降阶 | 可以连续降阶，每次退回一步，直到回到初始职业 |
| 初始职业 | 一阶职业无来源，不可降阶 |

## 9. 属性变化规则

### 9.1 核心公式

转职时保留已获得的等级成长，仅更换职业基础属性差值：

```
new_stat = old_stat - old_class_base + new_class_base
```

### 9.2 详细说明

| 术语 | 含义 |
|------|------|
| old_stat | 单位当前属性值（含基础+成长） |
| old_class_base | 旧职业基础属性值（来自TB_Class.base_stats_json） |
| new_class_base | 新职业基础属性值（来自TB_Class.base_stats_json） |
| growth | 等级成长 = old_stat - old_class_base（隐式保留） |

### 9.3 计算示例

**场景**：士兵(soldier)转职为战士(fighter)

| 属性 | soldier基础 | 当前属性 | 成长(隐式) | fighter基础 | 转职后属性 |
|------|-----------|---------|-----------|-----------|-----------|
| STR | 8 | 12 | 4 | 12 | 16 |
| SKL | 5 | 7 | 2 | 7 | 9 |
| MAG | 2 | 3 | 1 | 2 | 3 |
| SPD | 6 | 8 | 2 | 8 | 10 |
| DEF | 5 | 7 | 2 | 7 | 9 |
| RES | 2 | 3 | 1 | 2 | 3 |
| HP | 30 | 38 | 8 | 40 | 48 |
| LDR | 3 | 4 | 1 | 4 | 5 |

计算过程：
```
STR: 12 - 8 + 12 = 16  (成长4保留，基础8→12)
SKL: 7 - 5 + 7 = 9    (成长2保留，基础5→7)
HP:  38 - 30 + 40 = 48 (成长8保留，基础30→40)
```

### 9.4 降阶属性恢复

降阶时使用相同公式，方向相反：

```
new_stat = old_stat - current_class_base + source_class_base
```

**场景**：战士(fighter)降阶回士兵(soldier)

| 属性 | fighter基础 | 当前属性 | soldier基础 | 降阶后属性 |
|------|-----------|---------|-----------|-----------|
| STR | 12 | 16 | 8 | 12 |
| HP | 40 | 48 | 30 | 38 |

计算过程：
```
STR: 16 - 12 + 8 = 12
HP:  48 - 40 + 30 = 38
```

### 9.5 属性下限保护

```
转职后任意属性 = MAX(转职后属性, 1)
```

属性不会因转职降至0或负数，最低为1。

### 9.6 伪代码

```csharp
public void UpdateStatsPreserveGrowth(string unitId, ClassData oldClass, ClassData newClass)
{
    UnitStats current = GetUnitStats(unitId);
    StatsData oldBase = StatsData.FromJson(oldClass.base_stats_json);
    StatsData newBase = StatsData.FromJson(newClass.base_stats_json);

    current.str = Math.Max(1, current.str - oldBase.str + newBase.str);
    current.skl = Math.Max(1, current.skl - oldBase.skl + newBase.skl);
    current.mag = Math.Max(1, current.mag - oldBase.mag + newBase.mag);
    current.spd = Math.Max(1, current.spd - oldBase.spd + newBase.spd);
    current.def = Math.Max(1, current.def - oldBase.def + newBase.def);
    current.res = Math.Max(1, current.res - oldBase.res + newBase.res);
    current.hp = Math.Max(1, current.hp - oldBase.hp + newBase.hp);
    current.ldr = Math.Max(1, current.ldr - oldBase.ldr + newBase.ldr);

    SetUnitStats(unitId, current);
}
```
