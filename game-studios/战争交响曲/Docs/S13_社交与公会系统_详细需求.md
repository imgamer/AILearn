## 1. 系统概述

### 1.1 核心定位

社交与公会系统为玩家提供好友互动、公会协作、聊天沟通和公会对抗四大模块。通过社交关系增强玩家留存，通过公会战提供长线竞技目标。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 社交驱动留存 | 好友助战/赠送/羁绊加成让玩家每天有理由回来 |
| 公会协作深度 | 公会捐献/商店/战让公会不仅是聊天群 |
| 公平竞技 | 公会战匹配基于公会战力，避免碾压 |
| 安全可控 | 敏感词过滤+举报机制保障聊天环境 |

### 1.3 关键参数总览

| 参数 | 值 | 备注 |
|------|----|------|
| 好友上限 | 50人 | VIP可扩展至80 |
| 好友申请上限 | 20条/天 | 防止骚扰 |
| 公会最大人数 | 30人（1级）~50人（10级） | 随公会等级增长 |
| 创建公会费用 | 500钻石 | - |
| 公会捐献次数 | 3次/天 | 金币/钻石各计 |
| 聊天消息最大长度 | 100字 | 世界频道 |
| 公会战周期 | 每2周一次 | 持续3天 |
| 排行榜刷新 | 每周一00:00 | - |
| 助战借用冷却 | 每个好友1次/天 | - |
| 体力赠送上限 | 10次/天 | - |

## 2. 数据结构定义（C# class）

### 2.1 FriendData — 好友数据

```csharp
public class FriendData
{
    public string friend_id;                        // 好友玩家ID
    public string name;                             // 好友昵称
    public int level;                               // 好友等级
    public long last_login;                         // 最后登录时间(Unix秒)
    public string assist_unit_id;                   // 好友设置的助战单位ID
    public string assist_unit_name;                 // 助战单位名称
    public int assist_unit_level;                   // 助战单位等级
    public int assist_unit_power;                   // 助战单位战力
    public FriendStatus status;                     // 好友在线状态
    public int intimacy;                            // 亲密度(0-100)
    public long last_stamina_gift_time;             // 上次赠送体力时间
    public long last_assist_time;                   // 上次使用助战时间
    public bool has_sent_stamina_today;             // 今天是否已赠送体力
    public bool has_received_stamina_today;         // 今天是否已收到体力
}

public enum FriendStatus
{
    Offline,                                        // 离线
    Online,                                         // 在线
    InBattle,                                       // 战斗中

}
```

## 4. 好友系统

### 4.1 添加好友

```
添加方式：
1. 搜索ID添加：输入对方玩家ID → 发送申请
2. 推荐好友：系统根据等级/战力推荐在线玩家
3. 战后添加：战斗结算界面可添加对手/队友为好友
4. 附近的人：同章节/同关卡玩家推荐

流程：
1. 发送方发起好友申请(FriendRequest)
2. 接收方收到申请通知
3. 接收方选择：
   a. 接受 → 双方成为好友，FriendData双向创建
   b. 拒绝 → 申请删除，发送方无通知
   c. 忽略 → 申请保留24小时后自动过期

限制：
- 好友上限50人(VIP80人)
- 每日申请上限20条
- 不可重复向同一玩家发送申请
- 不可向黑名单中的玩家发送申请
- 对方好友列表已满时不可发送
```

### 4.2 删除好友

```
流程：
1. 在好友列表中选择好友
2. 点击"删除好友"
3. 弹出确认对话框："确定要删除好友XXX吗？"
4. 确认后：
   a. 双方FriendData删除
   b. 亲密度清零
   c. 不可恢复（需重新添加）

限制：
- 不可删除公会内好友（需先退出公会或通过公会管理）
- 删除后24小时内不可再次添加同一玩家
```

### 4.3 好友助战

```
机制：
- 每个玩家可设置1个助战单位（通常是队长）
- 好友的助战单位可在战斗开始前借用
- 每个好友的助战单位每天仅可借用1次
- 助战单位作为临时成员加入小队，战斗结束后消失

助战单位属性：
- 等级/装备/天赋与好友当前状态一致
- 不享受羁绊加成（除非与借用者有羁绊关系）
- 助战单位获得的经验归好友所有（50%经验返还）

流程：
1. 战斗准备界面 → 点击"助战"按钮
2. 显示好友助战列表（按战力排序）
3. 选择一个好友的助战单位
4. 助战单位加入当前小队
5. 战斗结束后：
   a. 助战单位消失
   b. 好友获得50%经验
   c. 标记今日已使用该好友助战
```

### 4.4 好友赠送体力/礼物

```
体力赠送：
- 每日可赠送体力给好友，上限10次/天
- 每次赠送：赠送方消耗0，接收方获得10体力
- 赠送不消耗赠送方的体力
- 一键赠送：可一键向所有今日未赠送的好友赠送
- 接收方需手动领取，未领取的体力24小时后过期

礼物赠送：
- 可赠送特定道具给好友
- 礼物从赠送方背包扣除
- 接收方通过邮件收取
- 每日赠送上限5次
```

### 4.5 好友羁绊加成

```
组队时好友羁绊加成：
- 好友关系：攻击力+2%
- 好友亲密度≥50：攻击力+3%，防御力+2%
- 好友亲密度≥80：攻击力+5%，防御力+3%，暴击率+2%

亲密度增长方式：
| 行为 | 亲密度增长 |
|------|-----------|
| 每日赠送体力 | +2 |
| 使用好友助战 | +3 |
| 好友使用你的助战 | +3 |
| 好友间私聊 | +1/天(上限) |
| 好友组队战斗 | +5/次 |

亲密度衰减：
- 超过7天无互动 → 亲密度-1/天
- 亲密度最低为0，不会为负
```

## 5. 公会系统

### 5.1 创建公会

```
条件：
1. 玩家等级≥10
2. 消耗500钻石
3. 不属于任何公会
4. 公会名称2-12字，不可包含敏感词
5. 同名检测（不可与已有公会重名）

流程：
1. 点击"创建公会"按钮
2. 输入公会名称(2-12字)
3. 选择公会图标
4. 设置加入条件：
   a. 自由加入（自动审批）
   b. 需审批（会长/副会长手动审批）
   c. 不可加入（仅邀请）
5. 设置最低等级要求(0-50)
6. 确认创建 → 扣除500钻石 → 创建公会
7. 创建者自动成为会长
```

### 5.2 加入公会

```
加入方式：
1. 搜索公会名称/ID
2. 推荐公会列表（按等级/人数/战力排序）
3. 会长/副会长邀请

流程（需审批模式）：
1. 玩家点击"申请加入"
2. 填写申请附言(最多50字)
3. GuildJoinRequest发送到公会
4. 会长/副会长审核：
   a. 同意 → 玩家加入公会，成为Member
   b. 拒绝 → 申请删除，玩家收到拒绝通知

流程（自由加入模式）：
1. 玩家点击"加入"
2. 检查等级要求
3. 满足条件直接加入

限制：
- 玩家同时只能属于1个公会
- 退出公会后12小时内不可加入新公会
- 退出公会后贡献值保留50%
- 公会成员已满时不可加入
```

### 5.3 公会等级与福利

```
公会经验来源：
| 来源 | 经验值 |
|------|--------|
| 成员捐献(金币) | 10/次 |
| 成员捐献(钻石) | 50/次 |
| 成员完成日常 | 5/人/天 |
| 公会战胜利 | 200/场 |
| 公会战参与 | 50/场 |

公会福利（由等级决定，见TB_GuildLevel）：

## 7. 公会战系统

### 7.1 公会战周期

```
每2周一次，持续3天：

Day1 - 匹配阶段(Matching)：
  00:00 ~ 12:00  报名期（会长/副会长点击"报名参战"）
  12:00 ~ 14:00  匹配期（系统匹配战力相近的公会）
  14:00 ~ 24:00  准备期（查看对手信息，部署防守）

Day2 - 攻防战阶段(Battle)：
  10:00 ~ 22:00  攻防战（成员派遣小队攻击对方领地）
  22:00 ~ 24:00  战斗统计

Day3 - 结算阶段(Settlement)：
  00:00 ~ 10:00  最终统计
  10:00          公布结果，发放奖励
```

### 7.2 匹配规则

```
匹配算法：
1. 收集所有报名公会
2. 按公会总战力排序
3. 战力差在matching_score_range内的公会优先匹配
4. 匹配公式：
   score_diff = |guild_a.total_power - guild_b.total_power|
   if score_diff <= matching_score_range:
       匹配成功
   else:
       扩大范围，取最接近的公会

5. 匹配结果公布：
   - 对手公会名称/等级/总战力
   - 对手Top5成员信息
```

### 7.3 领地系统

```
领地布局(9个领地)：

┌──────┬──────┬──────┐
│  1   │  2   │  3   │  ← 普通领地
│ 普通  │ 普通  │ 普通  │
├──────┼──────┼──────┤
│  4   │  9   │  5   │  ← 4,5普通 / 9首都
│ 普通  │ 首都  │ 普通  │
├──────┼──────┼──────┤
│  6   │  7   │  8   │  ← 6普通 / 7资源 / 8要塞
│ 普通  │ 资源  │ 要塞  │
└──────┴──────┴──────┘

领地初始状态：全部未占领

领地积分：

| 要塞 | 20 | ×2 |

### 7.4 攻防战详细流程

```
Day2 攻防战流程：

1. 防守部署（Day1 14:00~24:00）：
   a. 会长/副会长分配成员到各领地防守
   b. 每个领地最多3名防守者
   c. 未分配的成员自动部署到首都
   d. 防守者不可更改（锁定至战斗结束）

2. 攻击阶段（Day2 10:00~22:00）：
   a. 每个成员最多攻击3次
   b. 选择对方领地发起攻击
   c. 攻击流程：
      - 选择领地 → 查看防守者信息
      - 编组攻击小队
      - 发起攻击 → 与防守者小队战斗（自动结算）
      - 战斗结果：
        * 胜利 → 占领该领地，获得积分
        * 失败 → 不占领，消耗1次攻击次数

3. 领地争夺规则：
   a. 未占领的领地：直接攻击NPC守卫（战力较低）
   b. 已被对方占领的领地：需击败防守者
   c. 己方已占领的领地：不可攻击
   d. 领地可被反复争夺，最后占领方获得积分

4. 积分计算：
   guild_a_score = SUM(己方占领领地的积分)
   guild_b_score = SUM(对方占领领地的积分)
```

### 7.5 公会战战斗结算

```
单场战斗规则：
- 攻击方选择1支小队
- 防守方使用部署的小队
- 自动战斗结算（使用S01战斗系统）
- 攻击方胜利条件：消灭防守方小队
- 防守方胜利条件：存活任意单位

战斗特殊规则：
- 攻击方有"战意加成"：攻击力+10%（鼓励进攻）
- 防守方有"城墙加成"：防御力+10%（鼓励防守部署）
- 不可使用神力系统（公会战中禁用）
- 不可劝降（公会战中士气系统正常但不可劝降）
```

## 9. 边界条件与测试用例

### 9.1 边界条件

| 场景 | 处理方式 |
