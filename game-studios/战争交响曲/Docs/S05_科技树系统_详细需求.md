## 1. 系统概述

### 1.1 核心设计

科技树系统是阵营全局成长的核心机制。玩家通过阵营评级升级获取科技点，在三大分支（战争学、战术策略、制作产业）中研究科技，获得全军被动加成、条件触发效果和解锁新内容。

### 1.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 分支数 | 3 | 战争学/战术策略/制作产业 |
| 位阶数 | 4 | T1-T4 |
| 科技节点总数 | 36 | 左线13 + 中线11 + 右线12 |
| 阵营评级 | 10级 | 评级1-10 |
| 科技点来源 | 阵营评级升级 | 每次升级获得 |
| 重置消耗 | 钻石 | 首次免费 |

### 1.3 设计原则

- **三大分支并行**：战争学强化单位属性，战术策略强化战斗机制，制作产业强化装备与解锁
- **位阶递进**：T1基础→T2进阶→T3高级→T4终极，需满足前置条件
- **阵营评级驱动**：科技点仅通过评级升级获取，评级由游戏进度决定
- **可重置**：允许玩家重新分配科技点，鼓励尝试不同策略

## 2. 数据结构（C# Class）

### 2.1 TechNode - 科技节点

```csharp
[Serializable]
public class TechNode
{
    public string tech_id;               // 科技ID，如 "war_t1_military_drill"
    public string branch;                // 分支: "war" | "tactics" | "craft"
    public int tier;                     // 位阶 1-4
    public string name;                  // 科技名称
    public string description;           // 科技描述
    public string effect_json;           // 效果JSON，引用TB_TechEffect
    public string[] prerequisite_tech_ids;// 前置科技ID数组（同分支内）
    public int cost_points;              // 学习消耗科技点
    public bool is_unlocked;             // 是否已解锁（满足位阶解锁条件）
    public bool is_learned;              // 是否已学习
}
```

### 2.2 TechState - 科技状态

```csharp
[Serializable]
public class TechState
{
    public List<string> learned_tech_ids;   // 已学科技ID列表
    public int total_points;                // 剩余总科技点
    public int[] points_by_branch;          // 各分支已投入科技点 [war, tactics, craft]
    public int total_learned_by_branch;     // 各分支已学数量（用于解锁判断）
}
```

### 2.3 FactionRank - 阵营评级

```csharp
[Serializable]
public class FactionRank
{
    public int rank_level;           // 评级等级 1-10
    public string rank_name;         // 评级名称
    public int required_score;       // 升级所需分数
    public int tech_points_reward;   // 升级奖励科技点
    public int unlock_tier;          // 解锁的科技位阶
}
```

## 5. 解锁规则

### 5.1 位阶解锁条件

| 目标位阶 | 解锁条件 | 说明 |
|----------|---------|------|
| T1 | 初始解锁 | 游戏开始即可学习 |
| T2 | 同分支T1已学 ≥ 3个 | 需在同分支投入足够T1科技 |
| T3 | 同分支T2已学 ≥ 5个 | 注意：T2总共4-5个，需学完大部分 |
| T4 | 同分支T3已学 ≥ 7个 | 注意：T3总共2-4个，此条件隐含需T2+T3合计≥7 |

> **重要修正**：T4解锁条件"同分支T3已学≥7个"实际含义为**同分支已学科技总数≥7个**（含T1+T2+T3），因为单分支T3最多4个不可能达到7。

### 5.2 解锁判断伪代码

```csharp
public bool IsTierUnlocked(string branch, int targetTier, TechState state)
{
    if (targetTier == 1) return true;

    int learnedInBranch = state.learned_tech_ids
        .Count(id => TB_Tech.Get(id).branch == branch);

    int requiredCount;
    switch (targetTier)
    {
        case 2: requiredCount = 3; break;
        case 3: requiredCount = 5; break;
        case 4: requiredCount = 7; break;
        default: return false;
    }

    return learnedInBranch >= requiredCount;
}
```

### 5.3 阵营评级解锁

阵营评级同时控制科技位阶的硬性上限：

| 阵营评级 | 解锁科技位阶上限 |
|----------|----------------|
| 1-3级 | T1 |
| 4-6级 | T2 |
| 7-9级 | T3 |
| 10级 | T4 |

**综合判断**：科技节点可学习 = 满足同分支已学数量条件 AND 阵营评级达到对应位阶

```csharp
public bool CanLearnTech(TechNode tech, TechState state, int factionRank)
{
    if (tech.is_learned) return false;

    int maxTier = GetMaxTierByRank(factionRank);
    if (tech.tier > maxTier) return false;

    if (!IsTierUnlocked(tech.branch, tech.tier, state)) return false;

    foreach (string prereq in tech.prerequisite_tech_ids)
    {
        if (!state.learned_tech_ids.Contains(prereq)) return false;
    }

    if (state.total_points < tech.cost_points) return false;

    return true;
}
```

### 6.2 评级分数计算


| 击杀数 | 每杀1分 | 战斗中击杀敌军数量 |

### 6.4 伪代码

```csharp
public void UpdateFactionScore(int chaptersCompleted, int threeStarCount, int totalKills)
{
    int score = chaptersCompleted * 100 + threeStarCount * 50 + totalKills;
    FactionData faction = GetFactionData();

    while (faction.rank_level < 10)
    {
        FactionRank nextRank = TB_FactionRank.Get(faction.rank_level + 1);
        if (score >= nextRank.required_score)
        {
            faction.rank_level = nextRank.rank_level;
            faction.tech_points += nextRank.tech_points_reward;
            NotifyRankUp(faction.rank_level, nextRank.tech_points_reward, nextRank.unlock_tier);
        }
        else
        {
            break;
        }
    }

    faction.score = score;
}
```

#### 方式一：全军被动

```csharp
public void ApplyPassiveTechEffects(BattleContext context)
{
    foreach (string techId in TechState.learned_tech_ids)
    {
        TechNode tech = TB_Tech.Get(techId);
        TechEffect effect = TB_TechEffect.GetByTech(techId);

        if (effect.effect_type == "passive")
        {
            switch (effect.target_scope)
            {
                case "all_units":
                    context.global_damage_reduction += GetVal(effect, "damage_reduction");
                    context.global_phys_damage_bonus += GetVal(effect, "phys_damage_bonus");
                    context.global_cp_gain_bonus += GetVal(effect, "cp_gain_bonus");
                    break;
                case "unit_tag:healer":
                    context.tag_stat_bonus["healer"] = MergeStatBonus(
                        context.tag_stat_bonus.GetValueOrDefault("healer"),
                        effect.value);
                    break;
                case "unit_tag:mounted":
                    context.tag_stat_bonus["mounted"] = MergeStatBonus(
                        context.tag_stat_bonus.GetValueOrDefault("mounted"),
                        effect.value);
                    break;
                case "unit_tag:heavy_armor":
                    context.tag_stat_bonus["heavy_armor"] = MergeStatBonus(
                        context.tag_stat_bonus.GetValueOrDefault("heavy_armor"),
                        effect.value);
                    break;
                case "all_equipment":
                    context.equipment_effect_multiplier += GetVal(effect, "equipment_effect_bonus");
                    break;
            }
        }
    }
}
```

## 8. 重置规则

### 8.1 重置条件

| 条件 | 说明 |
|------|------|
| 消耗 | 钻石（首次免费） |
| 次数限制 | 无限制，每次消耗钻石递增 |
| 钻石消耗 | 第1次0，第2次50，第3次100，第4次200...公式：50 × (重置次数 - 1)，最低0 |

### 8.2 重置效果

| 效果 | 说明 |
|------|------|
| 退还科技点 | 全部已消耗科技点退还为可用点数 |
| 移除已学科技 | learned_tech_ids 清空 |
| 重置分支计数 | points_by_branch 归零 |
| 解锁状态重置 | 所有科技 is_learned = false |
| 解锁型效果失效 | 所有通过科技解锁的内容暂时不可用 |

### 8.3 重置流程

```
玩家点击"重置科技树"
       │
       ▼
  ┌──────────────┐    否    ┌──────────────┐
  │ 是否首次重置？ │──────→│ 检查钻石是否足够 │
  └──────┬───────┘       └──────┬─────────┘
         │ 是                    │
         │               ┌──────┴─────────┐
         │               │ 足够           │ 不足
         │               ▼                ▼
         │         扣除钻石          提示"钻石不足"
         │               │
         ▼               ▼
  ┌──────────────────────────┐
  │ 计算退还科技点总额         │
  │ refund = Σ(已学科技.cost) │
  └──────────┬───────────────┘
             │

  ┌──────────────────────────┐
  │ total_points += refund    │
  │ learned_tech_ids.Clear()  │
  │ points_by_branch = [0,0,0]│
  └──────────┬───────────────┘
             │
             ▼
  ┌──────────────────────────┐
  │ 通知各系统刷新解锁状态     │
  │ - 职业系统刷新            │
  │ - 制作系统刷新            │
  │ - 战斗修正刷新            │
  └──────────┬───────────────┘
             │
             ▼

```

### 8.4 重置伪代码

```csharp
public ResetResult ResetTechTree(TechState state, PlayerInventory inventory, int resetCount)
{
    int diamondCost = Math.Max(0, 50 * (resetCount - 1));

    if (diamondCost > 0 && inventory.diamonds < diamondCost)
        return ResetResult.Fail("钻石不足");

    if (diamondCost > 0)
        inventory.diamonds -= diamondCost;

    int refundPoints = 0;
    foreach (string techId in state.learned_tech_ids)
    {
        TechNode tech = TB_Tech.Get(techId);
        refundPoints += tech.cost_points;
        tech.is_learned = false;
    }

    state.total_points += refundPoints;
    state.learned_tech_ids.Clear();
    state.points_by_branch = new int[3];

    RefreshAllSystemUnlocks();

    return ResetResult.Success(refundPoints, diamondCost);
}
```

### 8.5 重置后的影响

| 系统 | 影响 | 处理方式 |
|------|------|---------|
| 职业转职 | 学徒职业可能被锁定（需"军事演习"解锁） | 已转职的单位不回退，但新单位无法转职为学徒 |
| 火器系统 | 暴雪火器限制恢复 | 已有火器单位保留，但暴雪下无法攻击 |
| 制作系统 | 高级配方锁定 | 已制作的装备保留，但无法再制作 |
| 战斗修正 | 被动加成移除 | 下场战斗生效 |
| ZOC | 步兵ZOC移除 | 下场战斗生效 |
