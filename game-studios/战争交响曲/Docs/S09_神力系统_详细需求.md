# S09 神力系统 - 详细需求文档

## 1. 系统概述

神力系统（Nephilim Power）是战争交响曲的核心战术机制之一，在主线第10章通关后解锁。玩家在战场上消耗法力值使用5种神力，对战局产生战略性影响。

| 神力 | 法力消耗 | 目标类型 | 核心效果 |
|------|---------|---------|---------|
| 祝福(Blessing) | 3 | 1友军小队 | 额外行动+治疗刷新 |
| 护盾(Exemplar) | 2 | 1友军小队 | 50%减伤+嘲讽2回合 |
| 传送(Teleport) | 2 | 1友军小队 | 位移5格内不消耗行动 |
| 流星(Meteor) | 4 | 十字形区域 | 75%最大生命伤害(Boss无效) |
| 恐惧(Terror) | 2 | 3×3区域 | 敌人士气-30 |

## 4. 法力值系统

### 4.1 法力值规则

| 规则 | 数值 |
|------|------|
| 每关初始法力 | 5点 |
| 法力上限 | 15点 |
| 击杀敌方小队 | +1法力 |
| 占领建筑 | +1法力 |
| 法力不可跨关继承 | 每关重置为5 |

### 4.2 法力值获取逻辑

```csharp
public class ManaSystem
{
    private int currentMana;
    private const int MAX_MANA = 15;
    private const int INITIAL_MANA = 5;

    public void OnBattleStart()
    {
        currentMana = INITIAL_MANA;
    }

    public void OnEnemySquadKilled()
    {
        AddMana(1);
    }

    public void OnBuildingOccupied()
    {
        AddMana(1);
    }

    private void AddMana(int amount)
    {
        currentMana = Math.Min(currentMana + amount, MAX_MANA);
    }

    public bool CanAfford(int cost)
    {
        return currentMana >= cost;
    }

    public void ConsumeMana(int cost)
    {
        currentMana -= cost;
    }
}
```

### 4.3 法力值UI显示

- 战场左上角显示当前法力/最大法力

- 法力不足时对应神力按钮灰显
- 每次获得法力时屏幕上方浮动 "+1 法力" 提示

## 5. 解锁条件

### 5.1 系统解锁

- 条件：主线第10章通关
- 解锁后：所有5种神力同时可用（无需逐一解锁）
- 解锁提示：通关结算后弹出"神力觉醒"剧情+系统教学

### 5.2 解锁判定逻辑

```csharp
public bool IsNephilimSystemUnlocked(PlayerProgress progress)
{
    return progress.current_chapter > 10
        || progress.chapter_stars.ContainsKey(10);
}
```

### 5.3 第10章之前

- 战场UI不显示神力按钮
- 不显示法力值
- 即使通过修改数据获得法力值也无法使用神力

## 6. 使用流程

### 6.1 操作步骤

```
1. 点击战场空地（非单位占据的格子）
   → 弹出神力菜单（环形/列表式）
2. 选择神力
   → 显示该神力的有效目标范围高亮
   → 法力不足的神力灰显不可选
3. 选择目标
   → 祝福/护盾/传送：点击1个友军小队
   → 流星：点击1个格子（以该格子为中心十字形）
   → 恐惧：点击1个格子（以该格子为中心3×3）
4. 确认
   → 显示效果预览（受影响区域/单位高亮）
   → 确认按钮
5. 执行
   → 扣除法力
   → 播放神力特效
   → 执行效果逻辑
   → 更新战场状态
```

## 7. 各神力详细效果实现

### 7.1 祝福 (Blessing)

```csharp
public void ExecuteBlessing(Squad target)
{
    // 1. 设置目标小队额外行动标记
    target.is_extra_turn = true;

    // 2. 重置治疗次数
    target.heal_count = 0;

    // 3. 刷新法师充能状态
    foreach (var unit in target.units)
    {
        if (unit.unit_class == UnitClass.Mage)
        {
            unit.is_charged = true;
            unit.skill_cooldown = 0;
        }
    }

    // 4. 播放祝福特效
    PlayVFX("blessing", target.position);

    // 5. 额外行动在当前回合结束后触发
    // 由回合管理器在处理完所有正常行动后检查is_extra_turn
}
```

**额外行动规则：**
- 额外行动在当前回合所有小队行动完毕后执行
- 额外行动包含：移动+攻击（完整行动）
- 每个小队每回合最多获得1次额外行动
- 额外行动消耗完毕后 `is_extra_turn` 重置为 false

### 7.2 护盾 (Exemplar)

```csharp
public void ExecuteExemplar(Squad target)
{
    // 1. 创建护盾Buff
    Buff shieldBuff = new Buff
    {
        buff_id = "shield",
        name = "神力护盾",
        duration_type = DurationType.Turns,
        duration_value = 2,  // 2回合（本回合+敌方回合）
        effects = new List<BuffEffect>
        {
            new BuffEffect
            {
                effect_type = BuffEffectType.DamageReduction,
                value = 0.5f  // 50%减伤
            },
            new BuffEffect
            {
                effect_type = BuffEffectType.Taunt,
                value = true  // 嘲讽：敌方优先攻击此小队
            }
        }
    };

    // 2. 添加到目标小队
    target.AddBuff(shieldBuff);

    // 3. 播放护盾特效
    PlayVFX("exemplar_shield", target.position);
}
```

**护盾规则：**
- 持续2回合：施放回合（我方回合）+ 敌方回合，敌方回合结束时到期
- 减伤计算：`final_damage = original_damage * (1 - 0.5)`
- 减伤与其他减伤效果取最大值，不叠加
- 嘲讽：敌方AI在可攻击范围内优先攻击带嘲讽的小队
- 护盾可被驱散类技能移除

### 7.3 传送 (Teleport)

```csharp
public void ExecuteTeleport(Squad target, GridPosition destination)
{
    // 1. 校验目标位置合法性
    if (!IsValidTeleportDestination(target, destination))
    {
        return; // 目标不合法，不执行
    }

    // 2. 修改小队位置
    GridPosition oldPosition = target.position;
    target.position = destination;

    // 3. 更新战场格子占用
    battlefield.MoveSquad(target, oldPosition, destination);

    // 4. 不标记为已行动
    // target.is_acted 保持不变（不设为true）

    // 5. 播放传送特效
    PlayVFX("teleport_out", oldPosition);
    PlayVFX("teleport_in", destination);
}
```

**传送规则：**
- 最大传送距离：5格（曼哈顿距离）
- 目标位置必须为空地（无其他小队、无不可通行地形）
- 传送后小队仍可正常行动（移动+攻击）
- 传送后小队的移动范围从新位置重新计算
- 不可传送至战场边界外

**合法性校验：**
```csharp
private bool IsValidTeleportDestination(Squad squad, GridPosition dest)
{
    // 距离检查
    int distance = ManhattanDistance(squad.position, dest);
    if (distance > 5) return false;

    // 地形检查
    if (!battlefield.IsWalkable(dest)) return false;

    // 占用检查
    if (battlefield.GetSquadAt(dest) != null) return false;

    // 边界检查
    if (!battlefield.IsInBounds(dest)) return false;

    return true;
}
```

### 7.4 流星 (Meteor)

```csharp
public void ExecuteMeteor(GridPosition center)
{
    // 1. 计算十字形区域（中心+上下左右共5格）
    List<GridPosition> affectedCells = new List<GridPosition>
    {
        center,                              // 中心
        new GridPosition(center.x, center.y + 1),  // 上
        new GridPosition(center.x, center.y - 1),  // 下
        new GridPosition(center.x + 1, center.y),  // 右
        new GridPosition(center.x - 1, center.y)   // 左
    };

    // 2. 遍历受影响格子内敌方单位
    foreach (var cell in affectedCells)
    {
        Squad enemySquad = battlefield.GetSquadAt(cell);
        if (enemySquad == null) continue;
        if (enemySquad.faction == Faction.Player) continue; // 跳过友军

        foreach (var unit in enemySquad.units)
        {
            // Boss免疫
            if (unit.is_boss)
            {
                continue;
            }

            // 计算75%最大生命伤害
            int damage = Mathf.CeilToInt(unit.max_hp * 0.75f);
            unit.TakeDamage(damage);
        }
    }

    // 3. 播放流星特效
    PlayVFX("meteor_impact", center);
}
```

**流星规则：**
- 伤害 = `ceil(unit.max_hp * 0.75)`，向上取整
- Boss单位完全免疫（is_boss = true 的单位不受伤害）
- 对友军不造成伤害
- 不触发反击
- 受伤单位正常计算防御减伤（流星伤害为原始伤害，防御仍生效）

**伤害计算修正：**
```csharp
// 流星伤害计算（考虑防御）
int rawDamage = Mathf.CeilToInt(unit.max_hp * 0.75f);
int finalDamage = Mathf.Max(1, rawDamage - unit.defense);
unit.TakeDamage(finalDamage);
```

### 7.5 恐惧 (Terror)

```csharp
public void ExecuteTerror(GridPosition center)
{
    // 1. 计算3×3区域
    List<GridPosition> affectedCells = new List<GridPosition>();
    for (int dx = -1; dx <= 1; dx++)
    {
        for (int dy = -1; dy <= 1; dy++)
        {
            affectedCells.Add(new GridPosition(center.x + dx, center.y + dy));
        }
    }

    // 2. 遍历受影响格子内敌方小队
    foreach (var cell in affectedCells)
    {
        Squad enemySquad = battlefield.GetSquadAt(cell);
        if (enemySquad == null) continue;
        if (enemySquad.faction == Faction.Player) continue;

        // 3. 降低士气
        enemySquad.morale -= 30;

        // 4. 士气下限为0
        enemySquad.morale = Mathf.Max(0, enemySquad.morale);

        // 5. 检查士气崩溃
        if (enemySquad.morale <= 0)
        {
            enemySquad.TriggerMoraleBreak();
        }
    }

    // 6. 播放恐惧特效
    PlayVFX("terror_wave", center);
}
```

**恐惧规则：**
- 士气降低30点，下限为0
- 士气为0时触发士气崩溃（小队逃跑或混乱）
- 对友军无效
- 同一小队在同一回合内被多次恐惧，效果叠加（士气-30×次数）
- Boss小队免疫恐惧效果

## 8. 边界条件与测试用例

### TC-NP-01: 系统未解锁时使用神力

- **前置**: 玩家进度在第9章
- **操作**: 点击战场空地
- **预期**: 不显示神力菜单，无法使用任何神力

### TC-NP-02: 法力不足时选择神力

- **前置**: 当前法力=2，选择流星(4法力)
- **操作**: 点击流星按钮
- **预期**: 流星按钮灰显不可选，提示"法力不足"

### TC-NP-03: 法力值上限限制

- **前置**: 当前法力=14，击杀敌方小队(+1)
- **操作**: 法力增加
- **预期**: 法力变为15（不超过上限），再次击杀不增加

### TC-NP-04: 祝福赋予额外行动

- **前置**: 友军小队A已行动完毕(is_acted=true)
- **操作**: 对A使用祝福(3法力)
- **预期**: A的is_extra_turn=true，本回合结束后A获得额外行动

### TC-NP-05: 护盾减伤与嘲讽

- **前置**: 友军小队A(HP=100)，敌方攻击伤害=40
- **操作**: 对A使用护盾，敌方攻击A
- **预期**: A受到伤害=40×0.5=20，敌方优先攻击A(嘲讽)

### TC-NP-06: 传送距离超限

- **前置**: 友军小队A在(0,0)
- **操作**: 尝试传送至(6,0)（距离6>5）
- **预期**: 目标位置不可选（红色高亮），无法确认

### TC-NP-07: 流星对Boss免疫

- **前置**: Boss单位在十字形区域内，HP=500
- **操作**: 在Boss所在位置使用流星
- **预期**: Boss不受伤害，区域内非Boss敌方单位正常受伤

### TC-NP-08: 恐惧士气叠加与崩溃

- **前置**: 敌方小队士气=50
- **操作**: 对同一小队使用2次恐惧
- **预期**: 第一次士气=20，第二次士气=0(下限)，触发士气崩溃

## 10. 服务器校验要点

1. 每次使用神力需校验：系统是否解锁、法力是否足够、目标是否合法
2. 法力值由服务器维护，客户端不可篡改
3. 神力效果在服务器端计算，客户端仅负责展示
4. 流星伤害需校验目标区域是否在合理范围内
5. 传送目标位置需校验是否为合法格子
6. 每关法力值初始化由服务器控制，不可跨关继承
7. 防作弊：同一回合内同一神力使用次数无硬限制，但法力值自然约束
