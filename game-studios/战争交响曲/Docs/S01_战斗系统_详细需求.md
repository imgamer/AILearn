## 1. 系统概述

### 1.1 核心定位

回合制战术战斗，玩家操控多支小队在网格地图上作战。小队间战斗自动结算，核心策略在战前编队和战中走位。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 编队深度 > 战斗操作 | 战前编组策略是核心乐趣，战斗自动结算 |
| 以护甲为中心的非对称克制 | 不是循环克制，而是功能互补 |
| 士气是隐藏变量 | 连接羁绊、战术、投降多个系统 |
| 攻击顺序是深度策略 | 同序兵种集火溢出，异序兵种分批打击 |

### 1.3 关键参数总览

| 参数 | 值 | 备注 |
|------|----|------|
| 地图尺寸 | 20×15 ~ 40×30 | 可配置 |
| 小队人数上限 | 9人（1队长+8士兵） | - |
| 阵型网格 | 3×5（9主格+6半格） | - |
| 士气范围 | 0 ~ 100 | 初始50 |
| 暴击系数 | 1.5 / 2.0 | 由技能/天赋决定 |
| 随机浮动 | 0.9 ~ 1.1 | Random.Range |
| 掩护减伤上限 | 75% | - |

## 2. 数据结构定义（C# class）

### 2.1 BattleContext — 战场上下文

```csharp
public class BattleContext
{
    public int MapWidth;                        // 地图宽度（格子数）
    public int MapHeight;                       // 地图高度（格子数）
    public int CurrentWeatherId;                // 当前天气ID，关联TB_Weather
    public int WeatherRemainingTurns;           // 天气剩余回合数
    public int TurnNumber;                      // 当前回合数（从1开始）
    public int TurnLimit;                       // 回合上限（0=无限）
    public int NephilimMana;                    // 神力法力值（初始5）
    public BattlePhase Phase;                   // 当前战斗阶段
    public List<SquadState> PlayerSquads;       // 玩家小队列表
    public List<SquadState> EnemySquads;        // 敌方小队列表
    public TileData[,] Tiles;                   // 地图格子二维数组 [y,x]
    public WinCondition WinCondition;           // 胜利条件
    public List<int> DefeatedPlayerSquadIds;    // 已被消灭的玩家小队ID
    public List<int> DefeatedEnemySquadIds;     // 已被消灭的敌方小队ID
}

public enum BattlePhase
{
    BattleInit,
    PlayerTurn,
    PlayerSquadSelect,
    PlayerSquadMove,
    PlayerSquadAction,
    EnemyTurn,
    TurnEndCheck,
    VictoryCheck,
    BattleEnd
}

public class WinCondition
{
    public WinConditionType Type;               // 胜利条件类型
    public List<int> TargetIds;                 // 目标ID列表（Boss小队ID/占领点ID等）
    public int RequiredCount;                   // 需要完成的数量
}

public enum WinConditionType
{
    DefeatAll,                                  // 消灭全部敌人
    DefeatBoss,                                 // 击败指定Boss
    DefeatBossAndOccupyBase,                    // 击败Boss+占领大本营
    SurviveTurns,                               // 坚守N回合
    EscortToTarget,                             // 护送至目标点
    OccupyAllPoints                             // 占领所有据点
}
```

### 2.2 TileData — 地格子数据

```csharp
public class TileData
{
    public int X;                               // 格子X坐标（列）

    public bool IsOccupied;                     // 建筑是否已被占领
    public int OccupiedByFaction;               // 建筑被哪方占领（0=无, 1=玩家, 2=敌方）
    public bool HasTreasure;                    // 是否有宝箱
    public bool IsBlocked;                      // 是否不可通过（墙壁等）
}
```

### 2.3 SquadState — 小队运行时状态

```csharp
public class SquadState
{
    public int SquadId;                         // 小队唯一ID
    public int Faction;                         // 阵营（1=玩家, 2=敌方）
    public int SquadTypeId;                     // 小队类型ID（飞行/轻骑/重骑/重步/轻步/缓慢）
    public int PosX;                            // 地图X坐标
    public int PosY;                            // 地图Y坐标
    public int BaseMovePoints;                  // 基础移动力
    public int RemainingMovePoints;             // 剩余移动力（骑兵攻击后移动用）
    public bool HasActed;                       // 本回合是否已行动
    public int Morale;                          // 士气值（0~100，初始50）
    public int HealCharges;                     // 剩余治疗次数
    public int MaxHealCharges;                  // 最大治疗次数
    public int SwapUsedThisTurn;                // 本回合已使用换位次数（0或1）
    public bool IsDefeated;                     // 是否已被消灭
    public int LeaderUnitId;                    // 队长单位ID
    public List<UnitState> Units;               // 小队内单位列表
    public int FormationWidth;                  // 阵型宽度（固定3）
    public int FormationHeight;                 // 阵型高度（固定3，前/中/后）
    public int AIType;                          // AI类型（0=玩家操控, 1=Normal, 2=Defensive, 3=Elite, 4=Boss）
    public float BossHpRatio;                   // Boss专用：当前血量比例
    public bool HasRaged;                       // Boss专用：是否已进入狂暴
    public bool HasSummoned;                    // Boss专用：是否已召唤援军
}

public enum SquadType
{
    Flying = 0,                                 // 飞行，移动力7
    LightCavalry = 1,                           // 轻骑，移动力7
    HeavyCavalry = 2,                           // 重骑，移动力7
    HeavyInfantry = 3,                          // 重步，移动力6
    LightInfantry = 4,                          // 轻步，移动力6
    Slow = 5                                    // 缓慢（法师/炮兵），移动力5
}
```

### 4.2 各状态详细说明

#### BattleInit（战斗初始化）

| 项目 | 内容 |
|------|------|
| **进入条件** | 玩家点击"开始战斗" |
| **执行逻辑** | 1. 加载TB_ChapterMap配置，初始化地图Tiles数组<br>2. 放置玩家小队到预设位置<br>3. 放置敌方小队到配置位置<br>4. 初始化所有小队的运行时状态（HP满、士气50、移动力满、未行动）<br>5. 计算每队治疗次数 = 队中治疗单位数 × 2<br>6. 初始化神力法力值 = 5<br>7. 设置回合数 = 1<br>8. 根据weather_schedule设置初始天气 |
| **退出条件** | 初始化完成 → 进入PlayerTurn |

#### PlayerTurn（玩家回合）

| 项目 | 内容 |
|------|------|
| **进入条件** | BattleInit完成 或 TurnEndCheck后VictoryCheck未结束 |
| **执行逻辑** | 1. 重置所有玩家小队的HasActed=false<br>2. 重置所有玩家小队的RemainingMovePoints=BaseMovePoints<br>3. 重置所有玩家小队的SwapUsedThisTurn=0<br>4. 恢复骑兵类小队的移动力<br>5. 处理回合开始时的Buff效果<br>6. 检查羁绊士气加成（有羁绊英雄同场 → 士气+5/回合）<br>7. 进入SquadSelect子状态 |
| **退出条件** | 所有玩家小队HasActed=true 或 玩家点击"结束回合" → 进入EnemyTurn |

#### SquadSelect（选择小队）

| 项目 | 内容 |
|------|------|
| **进入条件** | PlayerTurn开始 或 一支小队行动完成但仍有未行动小队 |
| **执行逻辑** | 1. 高亮所有未行动(HasActed=false)的玩家小队<br>2. 等待玩家点击选择<br>3. 点击已行动小队：仅显示信息，不可操作<br>4. 点击未行动小队：选中，进入SquadMove |
| **退出条件** | 选中一支未行动小队 → 进入SquadMove |

#### SquadMove（小队移动）

| 项目 | 内容 |
|------|------|
| **进入条件** | SquadSelect选中一支小队 |
| **执行逻辑** | 1. 以小队当前位置为起点，用BFS计算所有可达格子（考虑移动力、地形消耗、不可通过地形、其他小队占据的格子）<br>2. 蓝色高亮显示可达范围<br>3. 红色高亮显示攻击范围内的敌方小队<br>4. 玩家点击蓝色格子：小队移动到该位置，进入SquadAction<br>5. 玩家可选择不移动直接行动 |
| **退出条件** | 移动完成或跳过移动 → 进入SquadAction |

#### SquadAction（小队行动）

| 项目 | 内容 |
|------|------|
| **进入条件** | SquadMove完成 |
| **执行逻辑** | 弹出行动菜单，选项如下：<br>1. **攻击**：显示攻击范围内的敌方小队，选择目标后执行战斗结算（见第6节），标记HasActed=true<br>2. **治疗**：消耗1次治疗次数，治疗自身或相邻(上下左右4格)友军小队（见第11节），标记HasActed=true<br>3. **换位**：与相邻友军小队交换位置（见第12节），不标记HasActed<br>4. **待机**：不做任何操作，标记HasActed=true<br>5. **劝降**：当目标敌方士气≤30时可用（见投降系统），标记HasActed=true |
| **退出条件** | 执行完攻击/治疗/待机 → 回到SquadSelect检查是否还有未行动小队；执行换位 → 回到SquadSelect（该小队仍可行动） |

#### EnemyTurn（敌方回合）

| 项目 | 内容 |
|------|------|
| **进入条件** | PlayerTurn结束 |
| **执行逻辑** | 1. 按AI类型为每支敌方小队决策（见第13节）<br>2. 依次执行每支敌方小队的移动和行动<br>3. 每支小队行动完成后标记HasActed=true |
| **退出条件** | 所有敌方小队行动完成 → 进入TurnEndCheck |

#### TurnEndCheck（回合结束检查）

| 项目 | 内容 |
|------|------|
| **进入条件** | EnemyTurn结束 |
| **执行逻辑** | 1. 回合数+1<br>2. 检查天气变化：如果当前天气剩余回合=0，根据weather_schedule或随机切换天气<br>3. 天气效果结算：对对应元素单位施加buff/debuff；暴风雪天气检查火器单位是否被禁用<br>4. 天气士气伤害：如热浪，对无防护单位施加morale_damage_per_turn<br>5. 占领建筑奖励结算<br>6. Buff回合数-1，移除过期Buff<br>7. 检查小队全灭情况，从地图移除已消灭小队 |
| **退出条件** | 结算完成 → 进入VictoryCheck |

#### VictoryCheck（胜负判定）

| 项目 | 内容 |
|------|------|
| **进入条件** | TurnEndCheck完成 |
| **执行逻辑** | 1. 检查胜利条件：<br>   - DefeatAll：敌方全灭 → 胜利<br>   - DefeatBoss：指定Boss小队被消灭 → 胜利<br>   - DefeatBossAndOccupyBase：Boss消灭+大本营占领 → 胜利<br>   - SurviveTurns：存活到指定回合 → 胜利<br>   - EscortToTarget：护送小队到达目标点 → 胜利<br>   - OccupyAllPoints：所有据点占领 → 胜利<br>2. 检查失败条件：<br>   - 玩家全灭 → 失败<br>   - 回合数超过TurnLimit → 失败<br>   - 护送目标被消灭 → 失败<br>3. 同时满足多个条件时按优先级处理：失败条件 > 胜利条件 |
| **退出条件** | 胜利或失败 → 进入BattleEnd；未结束 → 回到PlayerTurn |

| **退出条件** | 玩家确认结算 → 返回Home Base |

## 5. 移动系统

### 5.1 移动力计算

```
最终移动力 = 基础移动力 + 地形修正 + 科技修正
```

**基础移动力**（由SquadType决定）：

| SquadType | 基础移动力 |
|-----------|-----------|
| Flying | 7 |
| LightCavalry | 7 |
| HeavyCavalry | 7 |
| HeavyInfantry | 6 |
| LightInfantry | 6 |
| Slow | 5 |

**地形修正**：
- 道路地形：移动力+1（仅对骑兵生效）
- 无其他正向修正，地形通过move_cost消耗移动力

**科技修正**：
- 由科技树配置提供，如"骑兵HP+50"不影响移动力，但某些科技可能增加移动力

**混合编队速度取最慢规则**：
```
小队移动力 = MIN(所有存活单位的移动力)
优先级：Slow(5) < HeavyInfantry(6) = LightInfantry(6) < Cavalry(7) = Flying(7)
特殊：如果小队中有飞行单位且无慢速单位，全队可飞行（忽略地形障碍），但移动力仍取最慢
```

### 5.2 寻路算法

**算法**：BFS（广度优先搜索），考虑地形消耗

**伪代码**：
```
function FindReachableTiles(startX, startY, movePoints, squadType):
    reachable = {}  // Dictionary<Vector2Int, int>  格子坐标 → 剩余移动力
    queue = PriorityQueue()  // 按剩余移动力降序排列
    queue.Enqueue((startX, startY), movePoints)
    reachable[(startX, startY)] = movePoints

    while queue.NotEmpty():
        (cx, cy), remaining = queue.Dequeue()

        for each (nx, ny) in GetFourNeighbors(cx, cy):
            if not InBounds(nx, ny): continue

            terrain = Tiles[ny, nx]
            cost = terrain.move_cost

            // 飞行单位忽略地形障碍
            if squadType == Flying:
                cost = 1  // 飞行单位所有地形消耗1
                // 但不能停在不可停留的地形上（水域等，除非配置允许）
                if not terrain.is_passable_flying: continue
            else:
                if not terrain.is_passable_ground: continue

            newRemaining = remaining - cost
            if newRemaining < 0: continue

            // 不能穿过有敌方小队的格子
            if terrain.IsOccupiedBySquad and terrain.OccupyingFaction != self.Faction:
                continue

            // 可以穿过友军但不能停留在友军格子上
            if terrain.IsOccupiedBySquad and terrain.OccupyingFaction == self.Faction:
                // 允许穿过但不允许停留
                if not reachable.ContainsKey((nx, ny)) or reachable[(nx, ny)] < newRemaining:
                    reachable[(nx, ny)] = newRemaining
                    queue.Enqueue((nx, ny), newRemaining)
                continue

            if not reachable.ContainsKey((nx, ny)) or reachable[(nx, ny)] < newRemaining:
                reachable[(nx, ny)] = newRemaining
                queue.Enqueue((nx, ny), newRemaining)

    // 移除有友军停留的格子（不能停在友军上）
    for each key in reachable.Keys:
        if Tiles[key.y, key.x].IsOccupiedBySquad and Tiles[key.y, key.x].OccupyingFaction == self.Faction:
            reachable.Remove(key)

    return reachable.Keys
```

### 5.3 骑兵攻击后移动

```
1. 骑兵小队移动到目标位置，记录RemainingMovePoints
2. 骑兵执行攻击行动
3. 攻击结算完成后：
   a. 如果RemainingMovePoints > 0：
      - 显示骑兵剩余移动范围
      - 玩家可选择继续移动或待机
   b. 如果RemainingMovePoints <= 0：
      - 无法继续移动，自动待机
4. 骑兵攻击后移动不触发新的攻击
```

### 5.4 飞行单位规则

- 忽略所有地形障碍（墙壁、山脉、水域等）
- 可以飞越敌方小队
- 地形移动消耗统一为1
- 不能停留在配置为不可停留的地形上
- 混合编队中：如果有飞行单位但没有慢速单位，全队获得飞行能力（可飞越障碍），但移动力仍取最慢

### 5.5 移动与攻击范围显示

```
移动范围（蓝色）= FindReachableTiles(当前位置, 移动力, 小队类型)
攻击范围（红色）= 以移动范围每个格子为起点，计算攻击射程内的敌方小队位置
  - 近战(range=1)：相邻4格+对角4格（8方向）
  - 远程(range=2)：曼哈顿距离≤2的所有格子
  - 远程(range=3)：曼哈顿距离≤3的所有格子
  - 高地加成：如果攻击者站在高地(is_high_ground=true)，射程+1
```

### 6.2 攻击类型判定

```
function DetermineAttackType(attackerSquad, defenderSquad, distance):
    attackerHasRanged = attackerSquad.Units.Any(u => u.IsAlive && u.AttackRange > 1)
    attackerHasMelee  = attackerSquad.Units.Any(u => u.IsAlive && u.AttackRange == 1)

    if attackerHasRanged and distance <= MaxRangedRange(attackerSquad):
        // 远程单位先打远程轮次
        // 近战单位打近战轮次
        return MixedAttack
    elif attackerHasMelee and distance == 1:
        return MeleeAttack
    elif attackerHasRanged and distance <= MaxRangedRange(attackerSquad):
        return RangedAttack
    else:
        return CannotAttack
```

### 6.3 目标选择算法

```
function SelectTarget(attacker, defenderSquad):
    aliveUnits = defenderSquad.Units.Where(u => u.IsAlive)

    if attacker.AttackPattern == "backline":
        // 刺客：后排优先
        backRow = aliveUnits.Where(u => u.FormationRow == 2)
        if backRow.Any(): return backRow.OrderBy(u => u.CurrentHP).First()
        midRow  = aliveUnits.Where(u => u.FormationRow == 1)
        if midRow.Any(): return midRow.OrderBy(u => u.CurrentHP).First()
        return aliveUnits.OrderBy(u => u.CurrentHP).First()

    elif attacker.AttackPattern == "horizontal":
        // 横排攻击：选择一排全伤
        // 返回该排所有存活单位
        row = SelectRowWithMostAliveUnits(aliveUnits)
        return aliveUnits.Where(u => u.FormationRow == row).ToList()

    elif attacker.AttackPattern == "vertical":
        // 纵列攻击：选择一列全伤
        col = SelectColWithMostAliveUnits(aliveUnits)
        return aliveUnits.Where(u => u.FormationCol == col).ToList()

    elif attacker.AttackPattern == "aoe":
        // AOE：攻击所有存活单位
        return aliveUnits.ToList()

    else:  // "single"
        if attacker.AttackType == 0:  // 近战
            // 前排优先
            frontRow = aliveUnits.Where(u => u.FormationRow == 0)
            if frontRow.Any(): return frontRow.OrderBy(u => u.CurrentHP).First()
            midRow = aliveUnits.Where(u => u.FormationRow == 1)
            if midRow.Any(): return midRow.OrderBy(u => u.CurrentHP).First()
            return aliveUnits.OrderBy(u => u.CurrentHP).First()
        else:  // 远程
            // 任意目标（最低HP优先）
            return aliveUnits.OrderBy(u => u.CurrentHP).First()
```

## 7. 伤害公式

### 7.1 完整伤害公式

```
基础伤害 = MAX(1, ATK × skill_multiplier - DEF × (1 - armor_penetration))

实际伤害 = 基础伤害
  × (1 + morale_bonus)        // 士气加成
  × (1 + terrain_atk_bonus)   // 地形攻击加成
  × (1 + weather_bonus)       // 天气加成
  × (1 - cover_reduction)     // 掩护减伤，上限75%
  × (1 + bond_bonus)          // 羁绊加成
  × (1 + tech_bonus)          // 科技加成
  × crit_multiplier           // 暴击系数
  × Random.Range(0.9f, 1.1f)  // 随机浮动

最终伤害 = MAX(1, Round(实际伤害))
```

### 7.2 各参数计算方式

#### morale_bonus（士气加成）

```
if 攻击方士气 >= 80:
    morale_bonus = 0.15          // +15%
elif 攻击方士气 <= 20:
    morale_bonus = -0.25         // -25%
else:
    morale_bonus = 0.0
```

#### terrain_atk_bonus（地形攻击加成）

```
terrain_atk_bonus = TB_Terrain[攻击者所在格terrain_id].defense_bonus
// 注意：攻击者站在有利地形时获得加成
// 高地对远程：射程+1（在移动系统中处理），不在此处加算
// 森林对防御方：ranged_dodge_bonus = 0.5（50%闪避远程攻击）
// 森林闪避判定：if 防守方在森林 and 攻击者是远程:
//     if Random.value < 0.5: 命中失败，伤害=0
```

#### weather_bonus（天气加成）

```
weather = TB_Weather[当前weather_id]
if 攻击者.element_id == weather.element_buff:
    weather_bonus = 0.15         // 被增强元素+15%
elif 攻击者.element_id == weather.element_debuff:
    weather_bonus = -0.15        // 被削弱元素-15%
else:
    weather_bonus = 0.0

// 特殊：暴风雪禁用火器
if weather.disable_firearm and 攻击者.DamageType == Firearm:
    if not 攻击者.HasTech("后膛装填"):
        无法攻击，跳过该单位
```

#### cover_reduction（掩护减伤）

```
// 见第9节详细计算
cover_reduction = MIN(0.75, SUM(每个掩护单位贡献))
每个重型单位贡献 = 0.15
```

#### bond_bonus（羁绊加成）

```
// 由外部系统计算后填入UnitState.BondBonus
// 友好：+0%
// 亲密：+3%
// 挚友/恋人：+5%
bond_bonus = 攻击者.BondBonus
```

#### tech_bonus（科技加成）

```
// 由外部系统计算后填入UnitState.TechBonus
// 例如"战术加成"科技：15%伤害换35%减伤
// "充能魔法"：法师首回合技能倍率+50%
tech_bonus = 攻击者.TechBonus
```

#### crit_multiplier（暴击系数）

```
if Random.value < 攻击者.CritRate:
    crit_multiplier = 攻击者.CritMultiplier   // 1.5 或 2.0
    isCrit = true
else:
    crit_multiplier = 1.0
    isCrit = false
```

### 7.3 穿甲率配置表

| DamageType | 名称 | 基础穿甲率 | 特殊修正 |
|------------|------|-----------|---------|
| 0 | 劈砍(剑) | 0% | 无 |
| 1 | 长柄 | 0% | 对骑兵+50%伤害（在skill_multiplier中体现） |
| 2 | 打击(锤) | 0% | 对重甲单位+25%伤害（在skill_multiplier中体现） |
| 3 | 弓箭 | 0% | 对重甲-50%伤害（在skill_multiplier中体现，×0.5） |
| 4 | 十字弓 | 25% | 无 |
| 5 | 火器 | 75% | 无 |
| 6 | 魔法 | 100% | 无 |
| 7 | 龙息 | 100% | 无 |

**穿甲率在公式中的位置**：
```
armor_penetration = 基础穿甲率 + 天赋穿甲加成 + 科技穿甲加成
DEF_effective = DEF × (1 - armor_penetration)
基础伤害 = MAX(1, ATK × skill_multiplier - DEF_effective)
```

**特殊修正实现**：
```
// 长柄对骑兵
if attacker.DamageType == Polearm and target.IsCavalry:
    skill_multiplier *= 1.5

// 打击对重甲

    skill_multiplier *= 1.25

// 弓箭对重甲
if attacker.DamageType == Bow and target.ArmorType == 2:  // 重甲
    skill_multiplier *= 0.5
```

## 9. 阻挡与掩护规则

### 9.1 前排阻挡规则

**阻挡条件**：
1. 目标单位在后排（formation_row == 1 或 2）
2. 目标正前方（同formation_col）有存活的前排单位（formation_row == 0）
3. 攻击类型为近战

**阻挡逻辑**：
```
function ApplyBlock(attacker, originalTarget, attackerSquad, defenderSquad):
    if attacker.AttackType != 0:  // 非近战不触发阻挡
        return originalTarget

    if originalTarget.FormationRow == 0:  // 前排不触发阻挡
        return originalTarget

    // 查找正前方的前排单位
    blocker = defenderSquad.Units.FirstOrDefault(u =>
        u.IsAlive &&
        u.FormationRow == 0 &&
        u.FormationCol == originalTarget.FormationCol
    )

    if blocker != null:
        // 阻挡成立，伤害由前排单位代受
        return blocker
    else:
        // 无阻挡，直接攻击原目标
        return originalTarget
```

### 9.2 半格阻挡规则

阵型为3×5网格，9主格+6半格。半格位于主格之间。

```
半格阻挡规则：
如果后排单位只有一半在前排单位后方（位于半格位置），
阻挡仍然成立。

判断方式：
- 半格位置与相邻两个主格位置有关联
- 如果半格位置关联的任一前排主格有存活单位，阻挡成立
- 阻挡者选择：优先选择同列前排，其次选择相邻列前排
```

### 9.3 掩护规则

**掩护条件**：
1. 掩护单位为重型单位（IsHeavyUnit == true，即重步/重骑）
2. 掩护单位在阻挡范围内（前排位置）
3. 掩护单位存活

**掩护与阻挡的区别**：
- **阻挡**：伤害完全由前排代受，原目标不受伤害
- **掩护**：不代受伤害，但为后方单位提供减伤

**掩护逻辑**：
```
function CalculateCoverReduction(target, defenderSquad):
    // 只对非前排单位生效
    if target.FormationRow == 0:
        return 0.0f

    coverReduction = 0.0f

    // 检查前排的重型单位
    foreach unit in defenderSquad.Units:
        if not unit.IsAlive: continue
        if not unit.IsHeavyUnit: continue
        if unit.FormationRow != 0: continue

        // 判断是否在掩护范围内
        // 同列前排：直接掩护
        // 相邻列前排：如果目标在半格位置也掩护
        colDiff = Mathf.Abs(unit.FormationCol - target.FormationCol)
        if colDiff <= 1:  // 同列或相邻列
            coverReduction += 0.15f  // 每个重型单位贡献15%

    // 上限75%
    return Mathf.Min(0.75f, coverReduction)
```

### 9.4 阻挡与掩护的交互

```
完整流程：
1. 攻击者选择目标
2. 判断是否触发阻挡：
   a. 如果触发阻挡 → 实际受击者变为前排阻挡单位
   b. 如果不触发阻挡 → 实际受击者仍为原目标
3. 计算掩护减伤：
   a. 对实际受击者计算cover_reduction
   b. 如果实际受击者是前排（因阻挡），掩护不生效（前排不享受掩护）
   c. 如果实际受击者是后排（无阻挡），掩护生效
4. 将cover_reduction代入伤害公式
```

### 9.5 掩护叠加示例

| 场景 | 前排重型单位数 | cover_reduction | 实际减伤 |
|------|-------------|----------------|---------|
| 1个重步在前排同列 | 1 | 15% | 15% |
| 2个重步在前排（同列+相邻列） | 2 | 30% | 30% |
| 3个重步在前排全覆盖 | 3 | 45% | 45% |
| 5个重步在前排 | 5 | 75% | 75%（上限） |

## 10. 反击规则

### 10.1 反击条件

| 条件 | 说明 |
|------|------|
| 被近战攻击时可反击 | 近战单位被近战攻击时 |
| 被远程攻击时可反击 | 远程单位被远程攻击时 |
| 被击败的单位不能反击 | CurrentHP <= 0 的单位 |
| 反击范围限制 | 近战反击仅限距离1，远程反击需在射程内 |

### 10.2 反击判定流程

```
function TryCounterAttack(attacker, defender, attackerSquad, defenderSquad):
    // 1. 检查防守方是否存活
    if not defender.IsAlive:
        return false

    // 2. 检查反击条件
    canCounter = false

    if attacker.AttackType == 0 and defender.AttackType == 0:
        // 近战攻击近战单位 → 可反击
        canCounter = true
    elif attacker.AttackType == 1 and defender.AttackType == 1:
        // 远程攻击远程单位 → 可反击
        canCounter = true
    elif attacker.AttackType == 0 and defender.AttackType == 1:
        // 近战攻击远程单位 → 远程单位不可反击（不在射程内）
        canCounter = false
    elif attacker.AttackType == 1 and defender.AttackType == 0:
        // 远程攻击近战单位 → 近战单位不可反击（不在射程内）
        canCounter = false

    if not canCounter:
        return false

    // 3. 特殊天赋检查（如"防止反击"天赋）
    if attacker.HasTalent("防止反击"):
        return false

    // 4. 执行反击
    counterDamage = CalculateDamage(defender, attacker, isCounterAttack: true)
    // 反击伤害 = 正常伤害 × 50%
    counterDamage = MAX(1, Round(counterDamage * 0.5f))

    attacker.CurrentHP -= counterDamage
    if attacker.CurrentHP <= 0:
        attacker.IsAlive = false

    return true
```

### 10.3 反击伤害计算

```
反击伤害 = CalculateDamage(反击者, 原攻击者) × 0.5

// CalculateDamage使用完整伤害公式（第7节）
// 但反击时不触发再次反击（防止无限循环）
```

| 地形限制 | 双方目标位置必须可通过 |

### 12.2 换位执行流程

```
function ExecuteSwap(squadA, squadB):
    // 1. 检查条件
    if squadA.SwapUsedThisTurn >= 1:
        return false  // 本回合已使用换位
    if squadB.SwapUsedThisTurn >= 1:
        return false  // 对方本回合已使用换位
    if not IsAdjacent(squadA, squadB):
        return false  // 不相邻
    if not IsPassable(squadB.PosX, squadB.PosY, squadA):
        return false  // 目标位置不可通过
    if not IsPassable(squadA.PosX, squadA.PosY, squadB):
        return false  // 目标位置不可通过

    // 2. 交换位置
    tempX = squadA.PosX
    tempY = squadA.PosY
    squadA.PosX = squadB.PosX
    squadA.PosY = squadB.PosY
    squadB.PosX = tempX
    squadB.PosY = tempY

    // 3. 更新地图占据信息
    Tiles[squadA.PosY, squadA.PosX].OccupyingSquadId = squadA.SquadId
    Tiles[squadA.PosY, squadA.PosX].OccupyingFaction = squadA.Faction
    Tiles[squadB.PosY, squadB.PosX].OccupyingSquadId = squadB.SquadId
    Tiles[squadB.PosY, squadB.PosX].OccupyingFaction = squadB.Faction

    // 4. 标记换位使用
    squadA.SwapUsedThisTurn = 1
    squadB.SwapUsedThisTurn = 1

    // 5. 不标记HasActed（双方仍可行动）
    return true

function IsAdjacent(a, b):
    dx = Mathf.Abs(a.PosX - b.PosX)
    dy = Mathf.Abs(a.PosY - b.PosY)
    return (dx + dy) == 1  // 曼哈顿距离=1，仅上下左右
```

## 13. 敌方AI

### 13.1 NormalAI — 普通AI

**行为优先级**：
1. 攻击威胁最低的可攻击目标
2. 如果无法攻击，向最近的敌人移动
3. 如果无法移动，待机

```
NormalAI_Decide(squad, battleContext):
    // 1. 获取所有可攻击目标
    attackableTargets = GetAttackableTargets(squad, battleContext)

    if attackableTargets.Count > 0:
        // 按威胁等级排序，攻击威胁最低的
        attackableTargets.Sort((a, b) => CalcThreatLevel(a) - CalcThreatLevel(b))
        target = attackableTargets[0]

        // 移动到最佳攻击位置
        bestPos = FindBestAttackPosition(squad, target, battleContext)
        MoveTo(squad, bestPos)
        Attack(squad, target)
    else:
        // 向最近的敌人移动
        nearestEnemy = FindNearestEnemy(squad, battleContext)
        if nearestEnemy != null:
            MoveToward(squad, nearestEnemy, battleContext)
        // 待机
        squad.HasActed = true
```

### 13.2 DefensiveAI — 防守AI

**行为优先级**：
1. 占据有利地形（高地、建筑、森林）
2. 不主动出击
3. 被攻击时反击（正常反击规则）
4. 如果有敌人进入攻击范围，攻击

```
DefensiveAI_Decide(squad, battleContext):
    // 1. 检查是否已在有利地形
    currentTile = battleContext.Tiles[squad.PosY, squad.PosX]
    isOnGoodTerrain = currentTile.is_high_ground or
                      currentTile.TerrainId == 3 or  // 森林
                      currentTile.BuildingType > 0     // 建筑

    // 2. 如果不在有利地形，移动到最近的有利地形
    if not isOnGoodTerrain:
        bestTerrain = FindNearestBeneficialTerrain(squad, battleContext)
        if bestTerrain != null:
            MoveTo(squad, bestTerrain)

    // 3. 检查攻击范围内是否有敌人
    attackableTargets = GetAttackableTargets(squad, battleContext)
    if attackableTargets.Count > 0:
        // 只攻击进入范围的敌人，不主动追击
        target = attackableTargets.OrderBy(t => t.CurrentHP).First()
        Attack(squad, target)
    else:
        // 待机，不主动出击
        squad.HasActed = true
```

### 13.3 EliteAI — 精英AI

**行为优先级**：
1. 优先攻击我方队长
2. 其次攻击低血目标
3. 主动追击

```
EliteAI_Decide(squad, battleContext):
    // 1. 获取所有可攻击目标
    attackableTargets = GetAttackableTargets(squad, battleContext)

    if attackableTargets.Count > 0:
        // 优先级1：我方队长
        leaders = attackableTargets.Where(t => t.Units.Any(u => u.IsLeader and u.IsAlive))
        if leaders.Count > 0:
            target = leaders.OrderBy(t => CalcThreatLevel(t)).First()
        else:
            // 优先级2：低血目标
            target = attackableTargets.OrderBy(t => t.Units.Sum(u => u.CurrentHP)).First()

        bestPos = FindBestAttackPosition(squad, target, battleContext)
        MoveTo(squad, bestPos)
        Attack(squad, target)
    else:
        // 主动追击最近的敌方队长或低血目标
        priorityTarget = FindNearestEnemyLeader(squad, battleContext)
        if priorityTarget == null:
            priorityTarget = FindNearestLowHPEnemy(squad, battleContext)
        if priorityTarget != null:
            MoveToward(squad, priorityTarget, battleContext)
        squad.HasActed = true
```

### 13.4 BossAI — Boss AI

**多阶段行为**：

| 血量阶段 | 行为 |
|---------|------|
| >75% | 正常攻击，优先攻击威胁最高目标 |
| 50%~75% | 使用技能（AOE/特殊攻击），优先攻击密集区域 |
| <50% | 狂暴：攻击力+30%，移动力+1 |
| <25% | 召唤援军（1次），继续狂暴 |

```
BossAI_Decide(squad, battleContext):
    hpRatio = squad.Units.First(u => u.IsLeader).CurrentHP /
              (float)squad.Units.First(u => u.IsLeader).MaxHP

    // 阶段4：血量<25%，召唤援军
    if hpRatio < 0.25 and not squad.HasSummoned:
        SummonReinforcements(squad, battleContext)
        squad.HasSummoned = true

    // 阶段3：血量<50%，狂暴
    if hpRatio < 0.50 and not squad.HasRaged:
        squad.HasRaged = true
        // 全队攻击力+30%
        foreach unit in squad.Units:
            if unit.IsAlive:
                unit.ATK = Mathf.RoundToInt(unit.ATK * 1.3f)
        // 移动力+1
        squad.BaseMovePoints += 1
        squad.RemainingMovePoints += 1

    // 攻击决策
    attackableTargets = GetAttackableTargets(squad, battleContext)

    if attackableTargets.Count > 0:
        if hpRatio > 0.75:
            // 阶段1：正常攻击，优先威胁最高
            target = attackableTargets.OrderByDescending(t => CalcThreatLevel(t)).First()
            Attack(squad, target)
        elif hpRatio > 0.50:
            // 阶段2：使用技能
            // 优先攻击单位最密集的区域
            target = FindDensestEnemyCluster(squad, battleContext)
            UseSkill(squad, target)
        else:
            // 阶段3/4：狂暴攻击
            target = attackableTargets.OrderByDescending(t => CalcThreatLevel(t)).First()
            Attack(squad, target)
    else:
        // 向最近敌人移动
        nearestEnemy = FindNearestEnemy(squad, battleContext)
        if nearestEnemy != null:
            MoveToward(squad, nearestEnemy, battleContext)
        squad.HasActed = true
```

### 13.5 威胁等级计算

```
function CalcThreatLevel(squad):
    threatLevel = 0
    foreach unit in squad.Units:
        if not unit.IsAlive: continue
        classMultiplier = GetClassMultiplier(unit.ClassId)
        qualityMultiplier = GetQualityMultiplier(unit.Quality)
        threatLevel += unit.Level * classMultiplier * qualityMultiplier

    threatLevel += squad.ArtifactValue / 100.0f
    return threatLevel

// 职业系数：一阶1.0 / 二阶1.5 / 三阶2.2 / 四阶3.5
// 品质系数：N=1.0 / R=1.1 / SR=1.3 / SSR=1.6
```

## 14. 边界条件与异常处理

### 14.1 小队全灭

```
触发条件：小队内所有单位IsAlive == false
处理逻辑：
1. 从地图移除该小队（Tiles上清除占据信息）
2. 加入DefeatedPlayerSquadIds或DefeatedEnemySquadIds
3. 触发士气变化：
   - 被消灭方：同阵营所有小队士气-10
   - 消灭方：同阵营所有小队士气+10
4. 如果被消灭方队长阵亡：同阵营所有小队额外士气-20
5. 检查胜负条件
```

### 14.2 回合数超限

```
触发条件：TurnNumber > TurnLimit（TurnLimit > 0时）
处理逻辑：
1. 判定玩家失败
2. 显示"回合数耗尽"提示
3. 进入BattleEnd（失败结算）
```

### 14.3 同时满足多个胜负条件

```
优先级（从高到低）：
1. 失败条件 > 胜利条件
2. 玩家全灭 > 回合数超限 > 护送目标死亡
3. 胜利条件内部优先级：
   DefeatBossAndOccupyBase > DefeatBoss > DefeatAll >
   OccupyAllPoints > SurviveTurns > EscortToTarget

处理逻辑：
1. 先检查所有失败条件
2. 任一失败条件满足 → 判负
3. 再检查胜利条件
4. 按优先级检查，第一个满足的 → 判胜
```

### 14.4 网络断线重连

```
处理逻辑：
1. 每个完整回合结束时，保存战场快照（BattleContext序列化）
2. 断线重连时：
   a. 加载最近一次保存的完整回合快照
   b. 如果断线发生在玩家回合中：
      - 恢复到该回合开始状态
      - 已完成行动的小队标记HasActed=true
   c. 如果断线发生在敌方回合中：
      - 恢复到该回合开始状态
      - 重新执行敌方AI
3. 快照保存频率：每个回合结束时
4. 快照内容：BattleContext完整序列化（JSON/Protobuf）
```

### 14.5 其他边界条件

| 场景 | 处理方式 |
|------|---------|
| 伤害计算结果≤0 | MAX(1, damage)，最低1点伤害 |
| 士气超出范围 | Clamp到[0, 100] |
| 治疗量超过MaxHP | CurrentHP = MaxHP，不溢出 |
| 移动到不可停留格子 | BFS中排除，不可选 |
| 攻击范围内无目标 | 攻击选项灰化，不可选 |
| 所有玩家小队已行动 | 自动进入EnemyTurn |
| 敌方小队被包围（2~4队） | 包围加成：2队+5/3队+10/4队+15士气伤害 |
| 天气切换时正在战斗 | 战斗结算完成后切换天气 |
| 火器单位在暴风雪中被禁用 | 跳过该单位攻击，不参与攻击顺序 |
| 骑兵攻击后移动到有宝箱的格子 | 自动开启宝箱 |
