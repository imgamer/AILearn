# S11 天赋与装备神器系统 — 详细需求文档

> 版本：v1.0 | 日期：2026-05-28
> 本文档为零歧义、可追踪、可直接落地的系统需求规格，程序拿到手即可编码。

---

## 一、系统概述

### 1.1 系统定位

| 系统 | 核心职责 | 生效范围 | 交互关系 |
|------|---------|---------|---------|
| 天赋系统 | 为单位提供个性化战斗能力 | 单位级 | 影响伤害计算、防御计算、暴击判定、容量计算 |
| 装备系统 | 为单位提供属性加成与职业特化 | 单位级 | 影响属性面板、容量占用、职业适配 |
| 神器系统 | 为小队提供全局修正效果 | 小队级 | 影响全队战斗表现、占用小队容量 |

### 1.2 系统间数据流

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  天赋系统    │     │  装备系统    │     │  神器系统    │
│ (单位级)    │     │ (单位级)    │     │ (小队级)    │
└──────┬──────┘     └──────┬──────┘     └──────┬──────┘
       │                   │                   │
       │  TalentHandler    │  StatBonus        │  SquadEffect
       │  注入战斗管线     │  叠加到属性面板   │  注入小队修正
       ▼                   ▼                   ▼
┌──────────────────────────────────────────────────────┐
│                   战斗计算管线                        │
│  伤害计算 → 防御计算 → 暴击判定 → 容量计算 → 士气   │
└──────────────────────────────────────────────────────┘
```

### 1.3 需求ID命名规则

| 前缀 | 系统 |
|------|------|
| TAL- | 天赋系统 |
| EQP- | 装备系统 |
| ART- | 神器系统 |
| QLT- | 品质系统 |
| TEL- | 天赋效果处理器 |

## 二、数据结构（C# class）

### 2.1 TalentData — 天赋数据

```csharp
public enum TalentCategory
{
    Damage,   // 伤害类
    Defense,  // 防御类
    Crit,     // 暴击类
    Utility,  // 功能类
    Capacity, // 容量类
    Leader    // 队长类
}

public enum TalentRarity
{
    Common,    // 普通
    Uncommon,  // 稀有
    Rare,      // 精良
    Epic       // 史诗
}

[Serializable]
public class TalentData
{
    public string talent_id;           // 天赋ID，如 "anti_cavalry"
    public string name;                // 显示名称，如 "反骑兵"
    public TalentCategory category;    // 天赋类别
    public string description;         // 描述文本
    public string effect_json;         // 效果参数JSON，供TalentHandler解析
    public bool is_leader_only;        // 是否仅队长生效
    public TalentRarity rarity;        // 稀有度
}
```

**effect_json 结构规范**：

每个天赋的 `effect_json` 为 JSON 字符串，包含该天赋效果所需的全部参数。TalentHandler 根据参数执行对应逻辑。

```json
// 示例：anti_cavalry
{
    "target_type": "cavalry",
    "damage_bonus": 0.35,
    "trigger_phase": "damage_calc"
}

// 示例：berserker
{
    "damage_per_dead": 0.25,
    "trigger_phase": "damage_calc",
    "count_source": "squad_dead_count"
}

// 示例：undying
{
    "trigger_phase": "on_fatal_damage",
    "hp_after": 1,
    "max_triggers_per_battle": 1
}

// 示例：cost_plus_2
{
    "capacity_modifier": 2,
    "leader_exempt": true,
    "trigger_phase": "capacity_calc"
}
```

### 2.2 EquipmentData — 装备数据

```csharp
public enum EquipSlot
{
    Weapon,    // 武器
    Armor,     // 护甲
    Accessory, // 饰品
    Boots,     // 鞋子
    Amulet,    // 护符
    Mount      // 坐骑
}

public enum EquipQuality
{
    White,   // 白
    Green,   // 绿
    Blue,    // 蓝
    Purple,  // 紫
    Orange   // 橙
}

public enum AttackType
{
    Slash,      // 劈砍
    Pierce,     // 穿刺(长柄)
    Blunt,      // 打击
    Bow,        // 弓箭
    Crossbow,   // 十字弓
    Firearm,    // 火器
    Magic,      // 魔法
    DragonBreath // 龙息
}

public enum ArmorType
{
    Light,  // 轻甲
    Medium, // 中甲
    Heavy   // 重甲
}

[Serializable]
public class StatBonus
{
    public int str_bonus;
    public int skl_bonus;
    public int mag_bonus;
    public int ldr_bonus;
    public int hp_bonus;
    public int armor_bonus;
    public int move_bonus;      // 移动力加成
    public int dodge_bonus;     // 闪避率加成(千分比)
    public int crit_bonus;      // 暴击率加成(千分比)
}

[Serializable]
public class EquipmentData
{
    public string equip_id;              // 装备ID，如 "wpn_iron_sword"
    public string name;                  // 显示名称
    public EquipSlot slot;               // 装备部位
    public EquipQuality quality;         // 品质
    public string stat_bonus_json;       // 属性加成JSON(StatBonus序列化)
    public AttackType? attack_type;      // 武器攻击类型(仅weapon)
    public ArmorType? armor_type;        // 护甲类型(仅armor)
    public List<string> class_restriction; // 职业限制列表(空=无限制)
    public int level_requirement;        // 等级需求
    public string special_effect_json;   // 特殊效果JSON(饰品/护符)
}
```

### 5.2 天赋随机抽取算法

```csharp
public class TalentRoller
{
    // 单位生成时调用
    public List<string> RollInitialTalents(
        List<TalentData> pool,      // 可用天赋池(排除leader类)
        int minCount,                // 最小数量=1
        int maxCount)                // 最大数量=2
    {
        int count = Random.Range(minCount, maxCount + 1);
        var results = new List<string>();
        var usedCategories = new HashSet<TalentCategory>();

        for (int i = 0; i < count; i++)
        {
            // 过滤：排除已选同类(容量类按talent_id去重)
            var candidates = pool.Where(t =>
            {
                if (t.category == TalentCategory.Capacity)
                    return !results.Contains(t.talent_id);
                return !usedCategories.Contains(t.category);
            }).ToList();

            if (candidates.Count == 0) break;

            // 按稀有度加权随机
            var selected = WeightedRandomByRarity(candidates);
            results.Add(selected.talent_id);
            usedCategories.Add(selected.category);
        }

        return results;
    }

    // 天赋书赋予时调用
    public string RollTalentFromBook(
        List<TalentData> pool,       // 可用天赋池
        TalentRarity maxRarity,      // 天赋书允许的最高稀有度
        List<string> existingTalentIds) // 已有天赋
    {
        var existingCategories = pool
            .Where(t => existingTalentIds.Contains(t.talent_id))
            .Select(t => t.category)
            .ToHashSet();

        var candidates = pool.Where(t =>
        {
            if (t.rarity > maxRarity) return false;
            if (existingTalentIds.Contains(t.talent_id)) return false;
            if (t.category == TalentCategory.Capacity)
                return true; // 容量类按id去重，已在上方排除
            if (existingCategories.Contains(t.category)) return false;
            return true;
        }).ToList();

        if (candidates.Count == 0) return null;

        return WeightedRandomByRarity(candidates).talent_id;
    }
}
```

### 6.3 TalentHandler 注册与调度

```csharp
public class TalentSystem
{
    private Dictionary<string, TalentHandler> _handlers
        = new Dictionary<string, TalentHandler>();

    private Dictionary<BattlePhase, List<TalentHandler>> _phaseHandlers
        = new Dictionary<BattlePhase, List<TalentHandler>>();

    // 初始化：从TB_Talent加载所有天赋，创建对应Handler
    public void Initialize(List<TalentData> talents)
    {
        foreach (var t in talents)
        {
            var handler = CreateHandler(t);
            _handlers[t.talent_id] = handler;

            var phase = ParsePhase(t.effect_json);
            if (!_phaseHandlers.ContainsKey(phase))
                _phaseHandlers[phase] = new List<TalentHandler>();
            _phaseHandlers[phase].Add(handler);
        }
    }

    // 战斗管线调用：在对应阶段触发所有相关Handler
    public void ProcessPhase(BattlePhase phase, TalentContext context)
    {
        if (!_phaseHandlers.ContainsKey(phase)) return;

        // 获取当前单位/小队拥有的天赋
        var activeTalents = GetActiveTalents(context);
        var handlers = _phaseHandlers[phase]
            .Where(h => activeTalents.Contains(h.talent_id));

        foreach (var handler in handlers)
        {
            handler.Apply(context);
        }
    }

    // 判断天赋是否激活(队长天赋需校验)
    private List<string> GetActiveTalents(TalentContext context)
    {
        var unit = context.attacker;
        var talents = unit.talent_ids;

        return talents.Where(tid =>
        {
            var data = TB_Talent.Get(tid);
            if (data.is_leader_only && !unit.is_leader)
                return false;
            return true;
        }).ToList();
    }
}
```

```csharp
public class BerserkerHandler : TalentHandler
{
    private float _damagePerDead;

    public override void Apply(TalentContext ctx)
    {
        int deadCount = ctx.attacker_squad.dead_count;
        ctx.damage *= (1 + _damagePerDead * deadCount);
    }

// 不死
public class UndyingHandler : TalentHandler
{
    private int _triggersLeft;

    public override void Apply(TalentContext ctx)
    {
        if (_triggersLeft <= 0) return;
        if (ctx.damage >= ctx.attacker.current_hp)
        {
            ctx.damage = ctx.attacker.current_hp - 1;
            _triggersLeft--;
        }
    }
```

```csharp
public StatBonus CalculateEquipBonus(UnitEquipment equip, int enhanceLevel)
{
    var total = new StatBonus();

    var slotFields = new (EquipSlot slot, string id)[]
    {
        (EquipSlot.Weapon, equip.weapon_id),
        (EquipSlot.Armor, equip.armor_id),
        (EquipSlot.Accessory, equip.accessory_id),
        (EquipSlot.Boots, equip.boots_id),
        (EquipSlot.Amulet, equip.amulet_id),
        (EquipSlot.Mount, equip.mount_id),
    };

    foreach (var (slot, instanceId) in slotFields)
    {
        if (string.IsNullOrEmpty(instanceId)) continue;

        var instance = GetInstance(instanceId);
        var data = TB_Equipment.Get(instance.equip_id);
        var baseBonus = JsonUtility.FromJson<StatBonus>(data.stat_bonus_json);
        var qualityData = TB_Quality.Get((int)instance.quality);

        float multiplier = qualityData.stat_multiplier
            * (1 + GetEnhanceRate(instance.quality) * instance.enhance_level);

        total.str_bonus   += (int)(baseBonus.str_bonus * multiplier);
        total.skl_bonus   += (int)(baseBonus.skl_bonus * multiplier);
        total.mag_bonus   += (int)(baseBonus.mag_bonus * multiplier);
        total.ldr_bonus   += (int)(baseBonus.ldr_bonus * multiplier);
        total.hp_bonus    += (int)(baseBonus.hp_bonus * multiplier);
        total.armor_bonus += (int)(baseBonus.armor_bonus * multiplier);
        total.move_bonus  += baseBonus.move_bonus; // 移动力不受品质倍率影响
        total.dodge_bonus += (int)(baseBonus.dodge_bonus * multiplier);
        total.crit_bonus  += (int)(baseBonus.crit_bonus * multiplier);
    }

    return total;
}
```

### 8.5 神器效果实现

```csharp
public abstract class ArtifactHandler
{
    public string artifact_id { get; protected set; }
    public abstract void Apply(ArtifactContext context);
}

public class ArtifactContext
{
    public SquadData squad;
    public ref float damage;
    public ref float damage_reduction;
    public ref float morale;
    public ref float crit_chance;
    public ref float heal_multiplier;
    public BattlePhase phase;
    public UnitData trigger_unit;
}
```

神器效果通过 `ArtifactSystem` 在战斗管线中注入，逻辑与天赋系统类似，区别在于：
- 天赋：遍历当前行动单位的天赋列表
- 神器：遍历当前行动单位所属小队的神器列表

```csharp
public class ArtifactSystem
{
    private Dictionary<string, ArtifactHandler> _handlers
        = new Dictionary<string, ArtifactHandler>();

    public void ProcessPhase(BattlePhase phase, SquadData squad, ArtifactContext ctx)
    {
        foreach (var artifactId in squad.artifact_ids)
        {
            if (_handlers.TryGetValue(artifactId, out var handler))
            {
                handler.Apply(ctx);
            }
        }
    }
}
```

### 9.2 战斗掉落规则

```csharp
public class DropTable
{
    public EquipQuality RollDropQuality(int chapterDifficulty)
    {
        // 基础权重随章节难度偏移
        // 章节1-10: 白70% 绿20% 蓝8% 紫2% 橙0%
        // 章节11-20: 白40% 绿30% 蓝20% 紫8% 橙2%
        // 章节21-30: 白20% 绿25% 蓝30% 紫18% 橙7%
        var weights = GetWeightsByChapter(chapterDifficulty);
        return WeightedRandom(weights);
    }

    public EquipmentInstance RollDropEquip(int chapterDifficulty)
    {
        var quality = RollDropQuality(chapterDifficulty);
        var candidates = TB_Equipment.GetAll()
            .Where(e => (EquipQuality)e.quality == quality)
            .ToList();

        var selected = WeightedRandomByField(candidates, e => e.drop_weight);
        return new EquipmentInstance
        {
            instance_id = GenerateId(),
            equip_id = selected.equip_id,
            quality = quality,
            enhance_level = 0,
            owner_unit_id = ""
        };
    }
}
```

## 附录B：容量计算完整公式

```
小队总容量 = 队长LDR × 2 + 队长等级 × 0.5

单位占用容量 = 基础占用(8)
             - 忠诚加成(0~3)
             + 天赋容量修正(SUM of capacity_modifier from capacity类天赋)
             (若单位为队长且天赋有leader_exempt=true，则该天赋不计算)

队长天赋容量修正 = SUM of capacity_modifier from leader类天赋
                   (仅对匹配target_tag的单位生效)

神器占用容量 = SUM(神器品质对应容量)

小队已用容量 = SUM(所有单位占用容量) + SUM(神器占用容量)

校验: 小队已用容量 ≤ 小队总容量
```
