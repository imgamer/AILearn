## 1. 系统概述

### 1.1 核心概念

小队（Squad）是战场上的基本行动单位。每个小队由 **1 名队长（Leader）+ 最多 8 名士兵（Member）** 组成，并携带若干遗物（Artifact）。小队在 3×5 阵型网格中排列，阵型决定了战斗中的受击顺序与掩护关系。

### 1.2 阵型网格结构

```
        Col0    Col1    Col2
Back   [B0,0]  [B0,1]  [B0,2]     ← 后排（3主格）
Mid    [M0,0]  [M0,1]  [M0,2]     ← 中排（3主格）
Front  [F0,0]  [F0,1]  [F0,2]     ← 前排（3主格）

半格位置（6个）：
  Front与Mid之间：[F-M,0]  [F-M,1]  [F-M,2]
  Mid与Back之间： [M-B,0]  [M-B,1]  [M-B,2]
```

- **主格**：9 个（3排 × 3列），每个格子可放置 1 个单位
- **半格**：6 个（排间过渡位置），仅用于特殊技能效果或临时位移，不可作为常规部署位
- **前排（Front）**：最靠近敌人的一排，承受近战攻击的第一线
- **中排（Mid）**：中间缓冲排
- **后排（Back）**：最远离敌人的一排，远程/法师推荐位置

### 1.3 关键约束

| 约束项 | 规则 |
|--------|------|
| 小队人数 | 1（队长）~ 9（队长+8士兵） |
| 阵型格子 | 9 主格，单位只能放在主格 |
| 容量限制 | used_capacity ≤ total_capacity |
| 队长必须 | 小队至少有队长，队长不可移除 |
| 阵型完整性 | 前排有单位才能放后排（可配置为警告模式） |

### 2.2 FormationData - 阵型数据

```csharp
/// <summary>
/// 阵型数据，描述9个主格的部署情况
/// </summary>
[Serializable]
public class FormationData
{
    // 9个主格，按 row-major 排列：Front[0-2], Mid[0-2], Back[0-2]
    public List<FormationSlot> slots;

    public FormationData()
    {
        slots = new List<FormationSlot>(9);
        for (int i = 0; i < 9; i++)
        {
            slots.Add(new FormationSlot
            {
                row = (FormationRow)(i / 3),
                col = i % 3,
                unit_id = null,
                is_half_slot = false
            });
        }
    }

    public FormationSlot GetSlot(FormationRow row, int col)
    {
        int index = (int)row * 3 + col;
        if (index < 0 || index >= slots.Count) return null;
        return slots[index];
    }

    public bool SetUnit(FormationRow row, int col, string unitId)
    {
        var slot = GetSlot(row, col);
        if (slot == null) return false;
        slot.unit_id = unitId;
        return true;
    }

    public FormationSlot FindUnitSlot(string unitId)
    {
        return slots.Find(s => s.unit_id == unitId);
    }

    public List<FormationSlot> GetRow(FormationRow row)
    {
        return slots.FindAll(s => s.row == row);
    }

    public List<FormationSlot> GetColumn(int col)
    {
        return slots.FindAll(s => s.col == col);
    }

    public bool HasUnitInRow(FormationRow row)
    {
        return slots.Exists(s => s.row == row && !string.IsNullOrEmpty(s.unit_id));
    }
}
```

### 2.4 SquadCapacityInfo - 容量信息

```csharp
/// <summary>
/// 小队容量信息，用于UI显示和容量校验
/// </summary>
public class SquadCapacityInfo
{
    public float total_capacity;           // 总容量
    public float used_capacity;            // 已用容量
    public float remaining_capacity;       // 剩余容量 = total - used
    public float leader_ldr;               // 队长统率值
    public int leader_level;               // 队长等级
    public List<UnitCostEntry> member_cost_list;   // 各成员容量消耗
    public List<ArtifactCostEntry> artifact_cost_list; // 各遗物容量消耗

    public bool IsOverCapacity => used_capacity > total_capacity;

    public void Recalculate()
    {
        used_capacity = 0f;
        foreach (var m in member_cost_list)
            used_capacity += m.cost;
        foreach (var a in artifact_cost_list)
            used_capacity += a.cost;
        remaining_capacity = total_capacity - used_capacity;
    }
}

[Serializable]
public class UnitCostEntry
{
    public string unit_id;
    public float base_cost;           // 基础消耗（8）
    public float loyalty_bonus;       // 忠诚减免（0~3）
    public float talent_reduction;    // 天赋减免（0~2）
    public float leader_talent_red;   // 队长天赋减免（0~2）
    public float cost;                // 最终消耗 = base - loyalty - talent - leader_talent
}

[Serializable]
public class ArtifactCostEntry
{
    public string artifact_id;
    public int quality;    // 品质等级：0白/1绿/2蓝/3紫/4橙
    public float cost;     // 消耗 = quality + 1（白1/绿2/蓝3/紫4/橙5）
}
```

## 4. 小队容量计算

### 4.1 总容量计算

```
total_capacity = leader.LDR * 2 + leader.level * 0.5
```

| 队长统率(LDR) | 队长等级 | 总容量 |
|---------------|----------|--------|
| 5 | 1 | 10.5 |
| 5 | 10 | 15.0 |
| 8 | 1 | 16.5 |
| 8 | 10 | 21.0 |
| 10 | 20 | 25.0 |

### 4.2 单位容量消耗

```
unit_cost = base_cost - loyalty_bonus - talent_reduction - leader_talent_reduction
```

| 参数 | 范围 | 说明 |
|------|------|------|
| base_cost | 固定8 | 所有单位基础消耗 |
| loyalty_bonus | 0~3 | 单位忠诚度减免，忠诚度≥80时+1，≥90时+2，≥95时+3 |
| talent_reduction | 0~2 | 单位自身天赋减免 |
| leader_talent_reduction | 0~2 | 队长天赋对成员的减免 |

**单位消耗范围：8（无减免）~ 1（最大减免7）**

### 4.3 遗物容量消耗

```
artifact_cost = quality_cost
```

| 品质 | quality值 | 消耗 |
|------|-----------|------|
| 白（Common） | 0 | 1 |
| 绿（Uncommon） | 1 | 2 |
| 蓝（Rare） | 2 | 3 |
| 紫（Epic） | 3 | 4 |
| 橙（Legendary） | 4 | 5 |

### 4.4 容量校验逻辑

```csharp
public class SquadCapacityChecker
{
    public static SquadCapacityInfo Calculate(SquadData squad, UnitDataManager unitMgr)
    {
        var info = new SquadCapacityInfo();
        var leader = unitMgr.GetUnit(squad.leader_unit_id);

        // 计算总容量
        info.leader_ldr = leader.stats.LDR;
        info.leader_level = leader.level;
        info.total_capacity = info.leader_ldr * 2f + info.leader_level * 0.5f;

        // 计算成员消耗
        info.member_cost_list = new List<UnitCostEntry>();
        foreach (var unitId in squad.member_list)
        {
            var unit = unitMgr.GetUnit(unitId);
            var entry = new UnitCostEntry
            {
                unit_id = unitId,
                base_cost = 8f,
                loyalty_bonus = CalcLoyaltyBonus(unit.loyalty),
                talent_reduction = unit.GetTalentCostReduction(),
                leader_talent_red = leader.GetLeaderTalentCostReduction(unit)
            };
            entry.cost = entry.base_cost - entry.loyalty_bonus
                       - entry.talent_reduction - entry.leader_talent_red;
            entry.cost = Math.Max(entry.cost, 1f); // 最低消耗1
            info.member_cost_list.Add(entry);
        }

        // 计算遗物消耗
        info.artifact_cost_list = new List<ArtifactCostEntry>();
        foreach (var artId in squad.artifact_list)
        {
            var artifact = unitMgr.GetArtifact(artId);
            var entry = new ArtifactCostEntry
            {
                artifact_id = artId,
                quality = (int)artifact.quality,
                cost = (int)artifact.quality + 1
            };
            info.artifact_cost_list.Add(entry);
        }

        info.Recalculate();
        return info;
    }

    private static float CalcLoyaltyBonus(int loyalty)
    {
        if (loyalty >= 95) return 3f;
        if (loyalty >= 90) return 2f;
        if (loyalty >= 80) return 1f;
        return 0f;
    }
}
```

## 5. 小队类型判定规则

### 5.1 判定算法

```csharp
public class SquadTypeResolver
{
    public static string ResolveSquadType(SquadData squad, UnitDataManager unitMgr)
    {
        // 收集所有单位（含队长）
        var allUnits = new List<Unit> { unitMgr.GetUnit(squad.leader_unit_id) };
        foreach (var uid in squad.member_list)
            allUnits.Add(unitMgr.GetUnit(uid));

        // 统计各 move_type 数量
        var moveCounts = new Dictionary<MoveType, int>();
        foreach (var unit in allUnits)
        {
            var mt = unit.GetEffectiveMoveType();
            if (!moveCounts.ContainsKey(mt))
                moveCounts[mt] = 0;
            moveCounts[mt]++;
        }

        // 特殊规则：有飞行单位时全队可飞
        if (moveCounts.ContainsKey(MoveType.Flying) && moveCounts[MoveType.Flying] > 0)
            return "flying_squad";

        // 特殊规则：有慢速单位时全队减速
        if (moveCounts.ContainsKey(MoveType.Slow) && moveCounts[MoveType.Slow] > 0)
            return "slow_squad";

        // 占多数的 move_type 决定 squad_type
        var maxCount = moveCounts.Values.Max();
        var majorityTypes = moveCounts
            .Where(kv => kv.Value == maxCount)
            .Select(kv => kv.Key)
            .ToList();

        // 平局时取更慢的类型
        // 优先级（速度从慢到快）：Slow > Normal > Fast > Flying
        var priorityOrder = new List<MoveType>
            { MoveType.Slow, MoveType.Normal, MoveType.Fast, MoveType.Flying };

        MoveType resolvedType = majorityTypes[0];
        foreach (var mt in priorityOrder)
        {
            if (majorityTypes.Contains(mt))
            {
                resolvedType = mt;
                break;
            }
        }

        return MoveTypeToSquadType(resolvedType);
    }

    private static string MoveTypeToSquadType(MoveType mt)
    {
        return mt switch
        {
            MoveType.Slow   => "slow_squad",
            MoveType.Normal => "normal_squad",
            MoveType.Fast   => "fast_squad",
            MoveType.Flying => "flying_squad",
            _ => "normal_squad"
        };
    }
}
```

### 5.2 判定示例

| 小队组成 | move_type统计 | 结果 |
|----------|---------------|------|
| 1队长(Normal)+4步兵(Normal)+2弓手(Normal) | Normal:7 | normal_squad |
| 1队长(Normal)+2步兵(Normal)+3轻骑(Fast) | Normal:3, Fast:3 | normal_squad（平局取慢） |
| 1队长(Normal)+1步兵(Normal)+6轻骑(Fast) | Normal:2, Fast:6 | fast_squad |

### 6.2 拖拽与吸附规则

```csharp
public class FormationEditor
{
    private FormationData formation;
    private SquadCapacityInfo capacityInfo;
    private bool allowEmptyFrontWarning; // 是否允许空前排（仅警告）

    public enum PlaceResult
    {
        Success,
        SlotOccupied,
        CavalryMustFrontOrMid,
        NoFrontUnitWarning,
        OverCapacity,
        InvalidSlot
    }

    public PlaceResult TryPlaceUnit(
        string unitId,
        FormationRow targetRow,
        int targetCol,
        Unit unit,
        out string warningMsg)
    {
        warningMsg = null;
        var slot = formation.GetSlot(targetRow, targetCol);

        // 1. 格子已被占用
        if (slot.IsOccupied)
            return PlaceResult.SlotOccupied;

        // 2. 骑兵只能放前排/中排
        var unitType = unit.GetUnitType();
        if (unitType.is_cavalry && targetRow == FormationRow.Back)
        {
            warningMsg = "骑兵单位只能部署在前排或中排";
            return PlaceResult.CavalryMustFrontOrMid;
        }

        // 3. 前排检查：后排/中排放置时，前排是否有单位
        if (targetRow == FormationRow.Back && !formation.HasUnitInRow(FormationRow.Mid)
            && !formation.HasUnitInRow(FormationRow.Front))
        {
            if (allowEmptyFrontWarning)
            {
                warningMsg = "警告：前排无单位，后排单位将直接暴露在攻击下";
                // 允许放置但警告
            }
            else
            {
                warningMsg = "前排必须有单位才能放置后排单位";
                return PlaceResult.NoFrontUnitWarning;
            }
        }

        if (targetRow == FormationRow.Mid && !formation.HasUnitInRow(FormationRow.Front))
        {
            if (allowEmptyFrontWarning)
            {
                warningMsg = "警告：前排无单位，中排单位将直接暴露在攻击下";
            }
            else
            {
                warningMsg = "前排必须有单位才能放置中排单位";
                return PlaceResult.NoFrontUnitWarning;
            }
        }

        // 4. 放置单位
        formation.SetUnit(targetRow, targetCol, unitId);
        return PlaceResult.Success;
    }

    public bool CanSave(out string errorMsg)
    {
        errorMsg = null;

        // 容量校验
        if (capacityInfo.IsOverCapacity)
        {
            errorMsg = $"小队容量超限：已用{capacityInfo.used_capacity:F1}/{capacityInfo.total_capacity:F1}";
            return false;
        }

        // 至少有队长
        if (string.IsNullOrEmpty(formation.FindUnitSlot(capacityInfo.leader_ldr.ToString())?.unit_id))
        {
            // 队长必须在阵型中
            // 此检查通过 leader_unit_id 在阵型中存在来验证
        }

        return true;
    }
}
```

### 6.4 自动布阵

```csharp
public class AutoFormation
{
    public static void AutoArrange(FormationData formation, List<Unit> units)
    {
        // 清空阵型
        foreach (var slot in formation.slots)
            slot.unit_id = null;

        var frontUnits = new List<Unit>();
        var midUnits = new List<Unit>();
        var backUnits = new List<Unit>();

        foreach (var unit in units)
        {
            var ut = unit.GetUnitType();
            if (ut.is_cavalry || ut.armor_type == ArmorType.Heavy)
                frontUnits.Add(unit);
            else if (ut.is_ranged || ut.attack_type == AttackType.Ranged)
                backUnits.Add(unit);
            else
                midUnits.Add(unit);
        }

        // 按优先级填充：前排 → 中排 → 后排
        FillRow(formation, FormationRow.Front, frontUnits, midUnits, backUnits);
        FillRow(formation, FormationRow.Mid, midUnits, backUnits, frontUnits);
        FillRow(formation, FormationRow.Back, backUnits, frontUnits, midUnits);
    }

    private static void FillRow(
        FormationData formation,
        FormationRow row,
        List<Unit> preferred,
        List<Unit> secondary1,
        List<Unit> secondary2)
    {
        int col = 0;
        var pool = new List<Unit>(preferred);
        pool.AddRange(secondary1);
        pool.AddRange(secondary2);

        foreach (var unit in pool)
        {
            if (col >= 3) break;
            // 骑兵不放后排
            if (row == FormationRow.Back && unit.GetUnitType().is_cavalry)
                continue;
            formation.SetUnit(row, col, unit.unit_id);
            col++;
        }
    }
}
```

## 7. 阵型战斗规则

### 7.1 前排阻挡 - 受击顺序

近战攻击的受击顺序由前排到后排逐层递进：

```
攻击方 → 前排存活单位? → 是 → 攻击前排
                      → 否 → 中排存活单位? → 是 → 攻击中排
                                            → 否 → 攻击后排
```

```csharp
public class FormationTargetResolver
{
    public static List<FormationSlot> GetMeleeTargets(FormationData formation)
    {
        if (formation.HasUnitInRow(FormationRow.Front))
            return formation.GetRow(FormationRow.Front)
                .FindAll(s => s.IsOccupied);
        if (formation.HasUnitInRow(FormationRow.Mid))
            return formation.GetRow(FormationRow.Mid)
                .FindAll(s => s.IsOccupied);
        return formation.GetRow(FormationRow.Back)
            .FindAll(s => s.IsOccupied);
    }
}
```

### 7.2 掩护减伤

重步兵/重骑兵为后方同列单位提供掩护减伤：

```csharp
public class CoverSystem
{
    public const float COVER_REDUCTION_PER_UNIT = 0.15f; // 每单位15%
    public const float COVER_REDUCTION_MAX = 0.75f;       // 上限75%

    public static float CalculateCoverReduction(
        FormationData formation,
        FormationSlot targetSlot)
    {
        // 只有中排/后排单位享受掩护
        if (targetSlot.row == FormationRow.Front)
            return 0f;

        float totalReduction = 0f;
        int col = targetSlot.col;

        // 检查前方所有排中同列的重步/重骑
        var rowsToCheck = targetSlot.row == FormationRow.Back
            ? new[] { FormationRow.Mid, FormationRow.Front }
            : new[] { FormationRow.Front };

        foreach (var row in rowsToCheck)
        {
            var slot = formation.GetSlot(row, col);
            if (slot != null && slot.IsOccupied)
            {
                var unit = UnitManager.Instance.GetUnit(slot.unit_id);
                var ut = unit.GetUnitType();
                if (ut.armor_type == ArmorType.Heavy &&
                    (ut.unit_type_id == "heavy_infantry" || ut.unit_type_id == "heavy_cavalry"))
                {
                    totalReduction += COVER_REDUCTION_PER_UNIT;
                }
            }
        }

        return Math.Min(totalReduction, COVER_REDUCTION_MAX);
    }
}
```

**掩护示例：**

| 后排目标 | 前方同列重步 | 中排同列重步 | 总减伤 |
|----------|-------------|-------------|--------|
| 后排弓手 | 1重步 | 1重步 | 30% |
| 后排弓手 | 1重步 | 无 | 15% |
| 后排弓手 | 无 | 1重步 | 15% |
| 中排法师 | 1重骑 | - | 15% |
| 前排重步 | - | - | 0%（前排不享受掩护） |

### 7.3 刺客绕后

刺客/盗贼的攻击从后排开始：

```csharp
public static List<FormationSlot> GetAssassinTargets(FormationData formation)
{
    // 优先级：后排 → 中排 → 前排（与近战相反）
    if (formation.HasUnitInRow(FormationRow.Back))
        return formation.GetRow(FormationRow.Back)
            .FindAll(s => s.IsOccupied);
    if (formation.HasUnitInRow(FormationRow.Mid))
        return formation.GetRow(FormationRow.Mid)
            .FindAll(s => s.IsOccupied);
    return formation.GetRow(FormationRow.Front)
        .FindAll(s => s.IsOccupied);
}
```

## 8. 边界条件

### 8.1 空小队

| 场景 | 处理方式 |
|------|----------|
| 小队无任何单位 | 不允许创建，至少需要队长 |
| 只有队长 | 合法，squad_type 由队长 move_type 决定 |
| 队长被移除 | 不允许，队长不可移除，只能替换 |

### 8.2 超容量

| 场景 | 处理方式 |
|------|----------|
| 添加成员后超容量 | 拒绝添加，提示"容量不足" |
| 遗物升级后超容量 | 允许保留但标记为超容量，战斗前必须调整 |
| 队长更换导致容量下降 | 允许保留但标记为超容量，下次编辑时强制调整 |

### 8.3 阵型异常

| 场景 | 处理方式 |
|------|----------|
| 阵型无前排 | 战斗中所有单位视为暴露，受近战攻击无阻挡 |
| 阵型所有格子空 | 不允许进入战斗 |
| 1x2体型单位 | 占据同列相邻两排的主格（前排+中排或中排+后排） |
| 两个单位交换位置 | 支持拖拽交换，需重新校验骑兵位置规则 |

### 8.4 战斗中阵型变化

| 场景 | 处理方式 |
|------|----------|
| 前排单位死亡 | 后排自动不前移，受击顺序自动调整 |
| 全部前排死亡 | 中排变为新的"前排"受击 |
| 只剩1个单位 | 该单位直接承受所有攻击 |
| 1x2体型单位半身死亡 | 整体移除，不留残体 |
