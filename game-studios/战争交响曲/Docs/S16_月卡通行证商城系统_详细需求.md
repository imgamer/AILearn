# S16 月卡/通行证/商城系统 - 详细需求文档

## 1. 系统概述

月卡/通行证/商城系统是F2P手游核心商业化落地模块，包含月卡、终身卡、赛季通行证、商城礼包四大子系统，占游戏收入目标30%以上。

| 子系统 | 定位 | 目标收入占比 | 核心价值 |
|--------|------|-------------|---------|
| 月卡 | 低门槛持续留存 | 8% | 每日登录激励 |
| 终身卡 | 一次性高价值付费 | 5% | 长期钻石来源 |
| 赛季通行证 | 赛季深度参与 | 10% | 活跃度+付费转化 |
| 商城礼包+充值 | 即时消费+钻石供给 | 7%+ | 付费覆盖面 |

## 4. 月卡系统

### 4.1 购买流程

```
PurchaseMonthlyCard(player_id):
    1. 获取月卡配置 card = TB_MonthlyCard["monthly_card_001"]
    2. 获取玩家月卡状态 state = GetMonthlyCardState(player_id)
    3. 创建支付订单
       order = CreateOrder(player_id, "monthly_card_001", card.price_cny)
    4. 发起第三方支付（见第9节支付流程）
    5. 支付成功回调后：
       a. 发放立即奖励
          AddDiamond(player_id, card.immediate_diamond)  // +300钻石
       b. 更新月卡状态
          IF state.is_active:
              // 叠加购买：延长到期日
              new_expire = ParseDate(state.expire_date).AddDays(card.duration_days)
              state.expire_date = new_expire.ToString()
          ELSE:
              // 首次购买或过期后重新购买
              state.purchase_date = Now().ToString()
              state.expire_date = Now().AddDays(card.duration_days).ToString()
              state.is_active = true
              state.daily_claimed_dates.Clear()
       c. 保存状态
          SaveMonthlyCardState(state)
    6. 记录购买
       SavePurchaseRecord(order.transaction_id, player_id, "monthly_card_001", card.price_cny)
```

### 4.2 每日领取流程

```
ClaimDailyMonthlyCard(player_id):
    1. 获取月卡状态 state = GetMonthlyCardState(player_id)
    2. 校验月卡是否生效
       IF NOT state.is_active OR ParseDate(state.expire_date) <= Now():
           state.is_active = false
           SaveMonthlyCardState(state)
           RETURN Error("月卡已过期")
    3. 校验今日是否已领取
       today = Now().ToString("yyyy-MM-dd")
       IF state.daily_claimed_dates.Contains(today):
           RETURN Error("今日已领取")
    4. 发放每日奖励
       card = TB_MonthlyCard["monthly_card_001"]
       AddDiamond(player_id, card.daily_diamond)  // +100钻石
    5. 记录领取
       state.daily_claimed_dates.Add(today)
       SaveMonthlyCardState(state)
    6. RETURN Success(daily_diamond=100)
```

### 4.3 月卡到期检查

```
CheckMonthlyCardExpiry(player_id):
    state = GetMonthlyCardState(player_id)
    IF state.is_active AND ParseDate(state.expire_date) <= Now():
        state.is_active = false
        SaveMonthlyCardState(state)
        // 推送通知：月卡已到期
        PushNotification(player_id, "monthly_card_expired")
```

### 4.4 每日领取入口

- 主界面顶部状态栏显示月卡图标+红点提示（有未领取时）
- 点击图标进入月卡详情页，显示剩余天数和领取按钮
- 每日5:00服务器时间刷新领取状态

## 6. 赛季通行证系统

### 6.1 赛季周期

- 每赛季约6周（42天）
- 赛季切换时重置通行证等级和经验
- 赛季结束后未领取的奖励通过邮件补发

### 6.2 经验获取

| 来源 | 单次经验 | 每日上限 | 说明 |
|------|---------|---------|------|
| 日常任务完成 | +10 | 50 | 每日5个日常任务 |
| 周常任务完成 | +30 | 210 | 每周7个周常任务 |
| 竞技场胜利 | +5 | 25 | 每日5次胜利 |
| 公会战参与 | +20 | 140 | 每周7次公会战 |

- 每日最大经验获取：50+25=75
- 每周最大经验获取：75×7+210+140=875
- 6周总经验上限：875×6=5250
- 50级总经验需求：100×50=5000
- 活跃玩家可满级，轻度玩家约35~40级

### 6.3 升级流程

```
AddBattlePassExp(player_id, exp_amount):
    1. 获取通行证状态 state = GetBattlePassState(player_id)
    2. 获取通行证配置 bp = TB_BattlePass[state.season_id]
    3. 校验赛季是否进行中
       IF Now() < ParseDate(bp.start_time) OR Now() > ParseDate(bp.end_time):
           RETURN Error("赛季未进行中")
    4. 校验是否已满级
       IF state.current_level >= bp.max_level:
           RETURN  // 已满级，经验溢出不记录
    5. 增加经验
       state.exp_current += exp_amount
    6. 检查升级
       WHILE state.exp_current >= bp.exp_per_level AND state.current_level < bp.max_level:
           state.exp_current -= bp.exp_per_level
           state.current_level += 1
           // 推送升级通知
           PushBattlePassLevelUp(player_id, state.current_level)
    7. 保存状态
       SaveBattlePassState(state)
```

### 6.4 奖励领取流程

```
ClaimBattlePassReward(player_id, level, is_premium_line):
    1. 获取通行证状态 state = GetBattlePassState(player_id)
    2. 获取通行证配置 bp = TB_BattlePass[state.season_id]
    3. 校验等级是否已达到
       IF level > state.current_level:
           RETURN Error("尚未达到该等级")
    4. 校验是否已领取
       IF is_premium_line:
           IF state.claimed_premium_levels.Contains(level):
               RETURN Error("该等级付费线奖励已领取")
           IF NOT state.is_premium:
               RETURN Error("未购买付费通行证")
       ELSE:
           IF state.claimed_free_levels.Contains(level):
               RETURN Error("该等级免费线奖励已领取")
    5. 获取奖励配置
       bp_level = TB_BattlePassLevel.Where(b => b.season_id == state.season_id && b.level == level)
       reward_id = is_premium_line ? bp_level.premium_reward_id : bp_level.free_reward_id
    6. 发放奖励
       GrantReward(player_id, reward_id)
    7. 记录领取
       IF is_premium_line:
           state.claimed_premium_levels.Add(level)
       ELSE:
           state.claimed_free_levels.Add(level)
       SaveBattlePassState(state)
```

### 6.5 购买付费线

```
PurchaseBattlePassPremium(player_id):
    1. 获取通行证状态 state = GetBattlePassState(player_id)
    2. 获取通行证配置 bp = TB_BattlePass[state.season_id]
    3. 校验是否已购买
       IF state.is_premium:
           RETURN Error("已购买付费通行证")
    4. 创建支付订单
       order = CreateOrder(player_id, "bp_premium_" + state.season_id, bp.premium_price_cny)
    5. 发起第三方支付
    6. 支付成功回调后：
       state.is_premium = true
       SaveBattlePassState(state)
       // 自动补发已达到等级的付费线奖励（不自动领取，仅标记可领取）
       PushNotification(player_id, "battle_pass_premium_activated")
```

### 6.6 快速升级购买

```
PurchaseQuickLevel(player_id, target_level):
    1. 获取通行证状态 state = GetBattlePassState(player_id)
    2. 获取通行证配置 bp = TB_BattlePass[state.season_id]
    3. 校验目标等级
       IF target_level <= state.current_level OR target_level > bp.max_level:
           RETURN Error("目标等级无效")
    4. 计算价格
       levels_to_buy = target_level - state.current_level
       total_price = levels_to_buy * bp.quick_level_price_cny  // 每级6元=600分
    5. 创建支付订单
       order = CreateOrder(player_id, "bp_quick_" + state.season_id, total_price)
    6. 发起第三方支付
    7. 支付成功回调后：
       state.current_level = target_level
       state.exp_current = 0
       SaveBattlePassState(state)
```

### 7.7 礼包触发检测

```
CheckShopItemAvailability(player_id, item_id):
    item = TB_ShopItem[item_id]
    state = GetPlayerShopState(player_id)

    // 检查限购次数
    purchased_count = state.GetPurchasedCount(item_id)
    IF item.purchase_limit > 0 AND purchased_count >= item.purchase_limit:
        RETURN false

    // 检查触发条件
    condition = ParseJson(item.available_condition)
    SWITCH condition.type:
        CASE "first_day":
            RETURN (Now() - player.create_time).TotalHours <= 24
        CASE "trigger":
            SWITCH condition.event:
                CASE "first_class_change":
                    RETURN player.HasCompletedClassChange(3)
                CASE "stage_fail_3":
                    RETURN player.GetConsecutiveFails() >= 3
                CASE "got_ssr_faction":
                    RETURN player.HasSSRInFaction(condition.faction_id)
        CASE "always":
            RETURN true
        DEFAULT:
            RETURN false
```

## 9. 支付流程

### 9.1 通用支付流程

```
客户端                    服务器                      第三方
  |                        |                           |
  |--- 请求创建订单 ------>|                           |
  |                        |--- 生成transaction_id     |
  |                        |--- 保存订单(pending)      |
  |<-- 返回订单信息 -------|                           |
  |                        |                           |
  |--- 调起支付SDK ---------------------------------->|
  |                        |                           |
  |<-- 支付完成回调 ----------------------------------|
  |                        |                           |
  |--- 上报支付凭证 ------>|                           |
  |                        |--- 向第三方验证凭证 ------>|
  |                        |<-- 验证结果 --------------|
  |                        |--- 幂等性检查             |
  |                        |--- 发放奖励               |
  |                        |--- 更新订单(completed)    |
  |<-- 返回发放结果 -------|                           |
```

### 9.2 iOS支付流程

```
iOSPurchase(player_id, product_id):
    1. 客户端调用StoreKit发起购买
       SKPaymentQueue.default().add(SKPayment(product: sk_product))
    2. StoreKit回调支付完成，获取receipt
    3. 客户端将receipt发送至服务器
    4. 服务器验证receipt
       url = "https://buy.itunes.apple.com/verifyReceipt"  // 生产环境
       // url = "https://sandbox.itunes.apple.com/verifyReceipt"  // 沙盒环境
       response = HTTPPost(url, {"receipt-data": receipt, "password": shared_secret})
    5. 解析验证结果
       IF response.status == 0:
           // 验证成功
           transaction_id = response.latest_receipt_info[0].transaction_id
       ELSE:
           RETURN Error("receipt验证失败: " + response.status)
    6. 幂等性检查
       IF ExistsPurchaseRecord(transaction_id):
           RETURN 已有结果（不重复发放）
    7. 发放奖励
    8. 保存PurchaseRecord(transaction_id, ..., status=completed)
```

### 9.3 Android支付流程

```
AndroidPurchase(player_id, product_id, channel):
    1. 根据渠道选择SDK
       SWITCH channel:
           CASE "google":
               调用Google Play Billing Library
           CASE "huawei":
               调用华为IAP SDK
           CASE "xiaomi":
               调用小米支付SDK
           // ... 其他渠道
    2. 客户端发起购买，获取orderId + purchaseToken
    3. 客户端将purchaseToken发送至服务器
    4. 服务器向对应渠道验证
       // Google Play示例
       url = "https://androidpublisher.googleapis.com/androidpublisher/v3/applications/{packageName}/purchases/products/{productId}/tokens/{token}"
       response = HTTPGet(url, headers=OAuth2Token)
    5. 验证成功后幂等性检查+发放奖励
```

### 9.4 幂等性保障

```
IdempotentGrant(transaction_id, player_id, item_id, rewards):
    // 分布式锁
    lock = AcquireLock("pay_lock_" + transaction_id, timeout=30s)
    IF NOT lock.acquired:
        RETURN Error("订单处理中，请勿重复请求")

    TRY:
        // 检查是否已处理
        existing = QueryPurchaseRecord(transaction_id)
        IF existing != null AND existing.status == completed:
            RETURN existing  // 已处理，返回原结果

        // 发放奖励（事务内）
        BEGIN_TRANSACTION:
            GrantRewards(player_id, rewards)
            UpdatePurchaseRecord(transaction_id, status=completed)
        COMMIT_TRANSACTION
    FINALLY:
        ReleaseLock(lock)
```

### 9.5 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 支付成功但客户端未收到回调 | 服务器定时查询未完成订单（每5分钟），主动补发 |
| receipt验证失败 | 标记订单failed，客户端提示"支付异常，请联系客服" |
| 重复receipt提交 | 幂等性检查，返回原结果不重复发放 |
| 退款 | 标记订单refunded，扣除已发放物品（如不可扣则标记欠款） |
| 订单超时未支付 | 30分钟后自动关闭pending订单 |
