## 1. 系统概述

羁绊系统是角色间关系深度机制，影响战斗士气和属性加成。特定角色对可通过剧情选择进入恋爱路线，最终影响结局走向。

**核心设计原则：**
- 羁绊关系是长期养成内容，不可速成
- 恋爱路线具有排他性，选择后不可更改
- 羁绊效果在战斗中自动生效，无需手动激活
- 羁绊对话采用视觉小说形式，增强沉浸感

**系统流程：**
```
角色配对出场/互动 → 获取羁绊经验 → 羁绊等级提升 → 解锁对话/技能/恋爱选择
     ↓                                                          ↓
  战斗中生效加成                                          恋人路线锁定 → 结局变化
```

## 2. 数据结构

### 2.1 BondPair - 羁绊对

```csharp
[Serializable]
public class BondPair
{
    public int hero_id_a;              // 角色A的ID（较小值放前，保证唯一性）
    public int hero_id_b;              // 角色B的ID
    public int bond_level;             // 当前羁绊等级 0~4
    public int bond_exp;               // 当前等级内累计经验
    public bool is_romance_available;  // 该对是否可恋爱（配置表决定）
    public bool is_romance_locked;     // 是否已锁定恋人路线（选择后不可逆）
    public int daily_exp_gained;       // 今日已获取经验（每日重置）
    public string last_daily_reset;    // 上次每日重置日期 "yyyy-MM-dd"

    public int GetKey()
    {
        int min = Math.Min(hero_id_a, hero_id_b);
        int max = Math.Max(hero_id_a, hero_id_b);
        return min * 10000 + max;
    }
}
```

## 4. 羁绊经验获取

### 4.1 经验来源与数值

| 来源 | 经验值 | 条件 | 说明 |
|------|--------|------|------|
| 同场战斗 | +10 | 两人同时出场且战斗胜利 | 失败+5，逃跑+0 |

| 专属事件 | +30 | 完成羁绊事件 | 一次性 |

| 特殊剧情 | +50 | 主线强制触发 | 极少数 |

### 4.2 每日上限机制

```csharp
public class BondExpManager
{
    private const int DAILY_EXP_CAP = 100;

    public bool CanGainExp(BondPair pair, int amount)
    {
        if (pair.daily_exp_gained >= DAILY_EXP_CAP)
            return false;
        if (pair.daily_exp_gained + amount > DAILY_EXP_CAP)
            return true; // 允许部分获取
        return true;
    }

    public int GetActualExpGain(BondPair pair, int requestedAmount)
    {
        int remaining = DAILY_EXP_CAP - pair.daily_exp_gained;
        return Math.Min(requestedAmount, Math.Max(remaining, 0));
    }

    public void CheckDailyReset(BondPair pair)
    {
        string today = DateTime.Now.ToString("yyyy-MM-dd");
        if (pair.last_daily_reset != today)
        {
            pair.daily_exp_gained = 0;
            pair.last_daily_reset = today;
        }
    }
}
```

### 4.3 经验与等级关系

```
等级0 陌生: exp_required=0    (起始等级)
等级1 友好: exp_required=100  (累计100)
等级2 亲密: exp_required=300  (累计300)
等级3 挚友: exp_required=600  (累计600)
等级4 恋人: exp_required=1000 (累计1000，仅可恋爱对)
```

**升级逻辑：**
- `bond_exp` 为当前等级内的累计经验（非全局累计）
- 升级时 `bond_exp` 减去 `exp_required`，溢出部分保留
- 恋人等级需要先做出恋爱选择才能升级

```csharp
public void AddExp(BondPair pair, int amount)
{
    int actual = GetActualExpGain(pair, amount);
    if (actual <= 0) return;

    pair.bond_exp += actual;
    pair.daily_exp_gained += actual;

    BondLevelConfig currentConfig = GetLevelConfig(pair.bond_level);
    BondLevelConfig nextConfig = GetLevelConfig(pair.bond_level + 1);

    while (nextConfig != null && pair.bond_exp >= nextConfig.exp_required)
    {
        if (nextConfig.is_romance_tier && !pair.is_romance_locked)
        {
            // 恋人等级需要先锁定，经验暂存不升级
            pair.bond_exp = nextConfig.exp_required;
            TriggerRomanceChoiceEvent(pair);
            return;
        }

        pair.bond_exp -= nextConfig.exp_required;
        pair.bond_level = nextConfig.level;
        OnLevelUp(pair, nextConfig);

        nextConfig = GetLevelConfig(pair.bond_level + 1);
    }
}
```

### 5.2 战斗中效果生效逻辑

```csharp
public class BondCombatEffect
{
    public static void ApplyBondEffects(Unit unit, List<BondPair> activeBonds, BattleContext context)
    {
        foreach (var bond in activeBonds)
        {
            if (!IsUnitInBond(unit, bond)) continue;

            BondPair otherUnit = GetBondPartner(unit, bond, context);
            if (otherUnit == null || !context.IsUnitOnField(otherUnit)) continue;

            BondLevelConfig config = GetLevelConfig(bond.bond_level);

            // 士气加成
            context.AddMorale(unit.team_id, config.morale_bonus);

            // 属性加成
            if (config.stat_bonus_percent > 0)
            {
                float multiplier = config.stat_bonus_percent / 100f;
                unit.temp_stats.atk = (int)(unit.base_stats.atk * (1 + multiplier));
                unit.temp_stats.def = (int)(unit.base_stats.def * (1 + multiplier));
                unit.temp_stats.spd = (int)(unit.base_stats.spd * (1 + multiplier));
                unit.temp_stats.mag = (int)(unit.base_stats.mag * (1 + multiplier));
            }

            // 专属技能
            if (!string.IsNullOrEmpty(config.unlock_skill_id))
            {
                unit.AddBondSkill(config.unlock_skill_id);
            }
        }
    }
}
```

### 5.3 属性加成计算规则

- 加成基于**基础属性**计算，不与其他百分比加成叠加（乘法叠加）
- 多个羁绊对的属性加成**可叠加**（如角色A同时与B、C有羁绊）
- 士气加成**不叠加**，取最高值
- 临时加成，仅在同场战斗中生效，战斗结束后移除

## 6. 恋爱路线

### 6.1 恋爱条件

```
条件1: is_romance_available = true（配置表决定）
条件2: bond_level >= 2（亲密及以上）
条件3: 主角性别匹配 romance_require_protagonist_gender
条件4: is_romance_locked = false（未锁定其他恋人路线）
条件5: 主角同时只能锁定1条恋人路线
```

### 6.2 恋爱选择流程

```
1. 羁绊达到亲密(2)后，触发 RomanceChoice 事件
2. 玩家在对话中选择：
   - 选项A："我对你有超越战友的感情" → 进入恋人路线
   - 选项B："我们是最好的伙伴" → 维持挚友路线
3. 选择A后：
   - is_romance_locked = true
   - 如果主角已有恋人，弹出确认："你已有恋人，确认更换吗？"
   - 确认更换后，旧恋人羁绊降级为友好(1)，is_romance_locked=false
   - 新恋人羁绊升级为恋人(4)
4. 选择B后：
   - 继续挚友路线，不再出现恋爱选项
   - 可在羁绊界面手动重新触发选择（仅1次机会）
```

### 6.3 主角性别影响

| 主角性别 | 可恋爱对象 | 说明 |

| female | hero_id: 1002, 1003, 1004, 1009, 1011 | 男性角色 |

- 非主角角色间的羁绊不受性别限制

### 7.3 对话系统规则

- 每段对话 3~10 句
- 选择分支最多 2~3 个选项
- 选择后设置 flag，影响后续事件和结局
- 对话可配置立绘表情变化
- 背景图可按场景配置
- 对话播放期间不可操作战斗

- 已播放的一次性对话可在"羁绊回忆"中重看（只读，不可重新选择）

## 9. 边界条件与测试用例

### 测试用例 1：每日经验上限

**场景：** 羁绊对已获取90经验，再次同场战斗(+10)  
**预期：** 实际获取10经验中的10（达到上限100），当日后续获取为0  
**验证：** `daily_exp_gained == 100`，后续调用 `CanGainExp` 返回 false

### 测试用例 2：等级升级经验溢出

**场景：** 友好等级，当前经验80，完成事件+30  
**预期：** 经验变为110，超过100升级阈值，升级后经验为10（110-100），等级变为亲密(2)  
**验证：** `bond_level == 2`，`bond_exp == 10`

### 测试用例 3：恋人等级未选择不可升级

**场景：** 亲密等级，经验达到300（满足恋人升级条件），但未做出恋爱选择  
**预期：** 经验锁定在300，不自动升级为恋人，触发恋爱选择事件  
**验证：** `bond_level == 2`，`bond_exp == 300`，恋爱选择事件已触发

### 测试用例 4：主角性别不匹配不可恋爱

**场景：** 主角性别为male，尝试与hero_id=1002（要求female）建立恋人关系  
**预期：** 恋爱选择事件不触发，只能走挚友路线  
**验证：** `RomanceChoice` 事件不出现，可正常升级至挚友(3)

### 测试用例 5：更换恋人路线

**场景：** 主角已与1002锁定恋人，后与1003触发恋爱选择并确认更换  
**预期：** 与1002的羁绊降级为友好(1)，`is_romance_locked=false`；与1003升级为恋人(4)，`is_romance_locked=true`  
**验证：** 1002的 `bond_level==1`，1003的 `bond_level==4`

### 测试用例 6：不可恋爱对无法到达恋人等级

**场景：** `is_romance_available=false` 的角色对（如1002-1003）经验超过挚友上限  
**预期：** 最高等级为挚友(3)，经验不再增长（或溢出后不再升级）  
**验证：** `bond_level <= 3`

### 测试用例 7：战斗失败经验获取

**场景：** 羁绊对两人同场但战斗失败  
**预期：** 获取5经验（胜利为10的一半）  
**验证：** `bond_exp += 5`

### 测试用例 8：多羁绊对属性叠加

**场景：** 角色A同时与B（亲密+3%）和C（挚友+5%）有羁绊，三人同场  
**预期：** A获得属性加成为 3%+5%=8%，B获得3%，C获得5%  
**验证：** A的 `temp_stats` 基于8%加成计算

### 测试用例 9：一次性事件重复触发防护

**场景：** 已完成的 `is_one_time=true` 事件再次满足条件  
**预期：** 事件不再触发  
**验证：** `is_completed==true` 的事件被跳过

### 测试用例 10：每日重置跨天

**场景：** 昨日 `daily_exp_gained=100`，今日首次获取经验  
**预期：** `daily_exp_gained` 重置为0，正常获取经验  
**验证：** `last_daily_reset` 更新为今日日期，`daily_exp_gained` 从0开始累加
