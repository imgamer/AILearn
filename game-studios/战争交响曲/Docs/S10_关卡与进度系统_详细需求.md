# S10 关卡与进度系统 - 详细需求文档

## 1. 系统概述

关卡与进度系统是游戏的核心推进框架，包含30章主线关卡、困难模式、体力系统、3星评价机制和阵营评级系统。

| 模块 | 说明 |
|------|------|
| 主线关卡 | 30章，难度递增，含6个卡点 |
| 困难模式 | 通关主线后解锁，敌方属性×1.5 |
| 体力系统 | 上限120，每6分钟恢复1点 |
| 3星评价 | 通关/占领所有建筑/回合限制 |
| 阵营评级 | 累计进度评级，解锁科技点位阶 |

## 4. 难度曲线公式

### 4.1 敌方属性计算公式

```
// chapter: 当前章节编号(1~30)
// base_xxx: TB_Chapter中配置的基础值

敌方HP = base_hp * (1 + chapter * 0.12)^2
敌方ATK = base_atk * (1 + chapter * 0.10)^1.8
敌方DEF = base_def * (1 + chapter * 0.08)^1.6
```

### 4.2 我方预期等级

```
我方预期等级 = chapter * 1.6 + 1
```

### 4.5 C# 实现

```csharp
public static class DifficultyCurve
{
    public static float CalculateEnemyHP(int chapter, int baseHP)
    {
        return baseHP * Mathf.Pow(1 + chapter * 0.12f, 2f);
    }

    public static float CalculateEnemyATK(int chapter, int baseATK)
    {
        return baseATK * Mathf.Pow(1 + chapter * 0.10f, 1.8f);
    }

    public static float CalculateEnemyDEF(int chapter, int baseDEF)
    {
        return baseDEF * Mathf.Pow(1 + chapter * 0.08f, 1.6f);
    }

    public static int ExpectedPlayerLevel(int chapter)
    {
        return Mathf.CeilToInt(chapter * 1.6f + 1);
    }

    public static float ApplyDifficultyMultiplier(float value, string difficultyId)
    {
        var difficulty = TB_Difficulty[difficultyId];
        return value * difficulty.hp_multiplier;
    }
}
```

## 5. 卡点设计（6个难度跳跃点）

### 5.1 卡点总览

| 卡点 | 章节 | 名称 | 核心机制 | 设计意图 |
|------|------|------|---------|---------|
| 1 | 第5章 | 铁蹄初鸣 | 首次出现骑兵敌人 | 迫使玩家学习反骑兵策略 |
| 2 | 第10章 | 重甲壁垒 | 首次出现重甲敌人 | 解锁神力系统，引入破甲需求 |
| 3 | 第15章 | 龙之降临 | 首次出现龙系敌人 | 迫使玩家培养对空/魔法单位 |
| 4 | 第20章 | 暗潮汹涌 | Boss多波次战斗 | 考验续航与资源管理 |
| 5 | 第25章 | 神罚 | 敌方使用神力 | 对称机制，迫使玩家理解神力反制 |
| 6 | 第30章 | 终焉之战 | 最终Boss | 综合考验所有机制 |

### 5.2 各卡点详细设计

#### 卡点1: 第5章 - 铁蹄初鸣

- **新敌人**: 骑兵（移动3格，冲锋伤害×1.5）
- **敌方配置**: 2队骑兵 + 3队步兵
- **地图特征**: 开阔平原，无建筑可守
- **教学提示**: "骑兵冲锋威力巨大，利用地形和枪兵可以有效克制"
- **预期卡点时长**: 1~2天
- **推荐战力**: 我方等级≈9，需要至少1队枪兵

#### 卡点2: 第10章 - 重甲壁垒

- **新敌人**: 重甲兵（防御×2，移动1格，弱魔法）
- **敌方配置**: 1队重甲 + 2队弓兵 + 2队步兵
- **地图特征**: 城堡攻防战，敌方据守建筑
- **教学提示**: "重甲单位防御极高，但魔法攻击可以无视部分防御"
- **通关后解锁**: 神力系统
- **预期卡点时长**: 2~3天
- **推荐战力**: 我方等级≈17，需要法师单位

#### 卡点3: 第15章 - 龙之降临

- **新敌人**: 飞龙（空中单位，近战无法攻击，弱弓箭/魔法）
- **敌方配置**: 1队飞龙 + 2队骑兵 + 2队步兵
- **地图特征**: 山地地形，高海拔优势
- **教学提示**: "飞龙翱翔天际，只有弓箭和魔法能触及它们"
- **预期卡点时长**: 2~3天
- **推荐战力**: 我方等级≈25，需要弓箭手/法师

#### 卡点4: 第20章 - 暗潮汹涌

- **新机制**: Boss多波次战斗（3波敌人依次出现）
- **敌方配置**: Boss + 3波增援（每3回合一波）
- **地图特征**: 封闭竞技场，无退路
- **设计要点**: 考验玩家续航能力、神力使用时机、资源管理
- **预期卡点时长**: 3~5天
- **推荐战力**: 我方等级≈33，需要治疗单位和合理神力使用

#### 卡点5: 第25章 - 神罚

- **新机制**: 敌方使用神力（每3回合使用1次）
- **敌方神力池**: 流星、恐惧（随机选择）
- **敌方配置**: 2队重甲 + 2队法师 + 1队骑兵
- **设计要点**: 迫使玩家理解神力效果，学会分散站位
- **预期卡点时长**: 3~5天
- **推荐战力**: 我方等级≈41，需要灵活走位

#### 卡点6: 第30章 - 终焉之战

- **最终Boss**: 3阶段Boss战
  - 阶段1: 人形（HP=10000，标准战斗）
  - 阶段2: 变身（HP=15000，每2回合使用神力）
  - 阶段3: 终极形态（HP=20000，全屏AOE+召唤增援）
- **设计要点**: 综合考验所有系统（兵种克制、神力、站位、资源管理）
- **预期卡点时长**: 5~7天
- **推荐战力**: 我方等级≈49，全系统精通

## 6. 3星评价系统

### 6.1 评价规则

| 星级 | 条件 | 说明 |
|------|------|------|
| ★ | 通关 | 满足胜利条件 |
| ★★ | 占领所有建筑 | 地图上所有可占领建筑均被我方控制 |
| ★★★ | 回合限制 | 通关时使用回合数≤指定阈值 |

### 6.2 星级判定逻辑

```csharp
public StarResult CalculateStars(BattleResult result, ChapterData chapter)
{
    int stars = 0;
    var conditions = ParseStarConditions(chapter.star_conditions_json);

    // 星1：通关（基础条件）
    if (result.is_victory)
    {
        stars = 1;
    }
    else
    {
        return new StarResult { stars = 0, details = "未通关" };
    }

    // 星2：占领所有建筑
    var occupyCondition = conditions.Find(c => c.condition_type == StarConditionType.occupy_all);
    if (occupyCondition != null && result.all_buildings_captured)
    {
        stars = 2;
    }

    // 星3：回合限制
    var turnCondition = conditions.Find(c => c.condition_type == StarConditionType.turn_limit);
    if (turnCondition != null && result.turns_used <= turnCondition.threshold)
    {
        stars = 3;
    }

    return new StarResult { stars = stars };
}
```

### 6.3 3星解锁扫荡

- 条件：某章节获得3星评价

- 限制：每日扫荡次数=3次/章

```csharp
public bool CanSweep(int chapterId, PlayerProgress progress)

}
```

### 7.4 体力系统实现

```csharp
public class StaminaSystem
{
    private int currentStamina;
    private const int MAX_STAMINA = 120;
    private const int RECOVER_INTERVAL_SEC = 360; // 6分钟=360秒
    private long lastRecoverTimestamp;

    public void OnLogin()
    {
        long now = GetCurrentTimestamp();
        long elapsed = now - lastRecoverTimestamp;
        int recovered = (int)(elapsed / RECOVER_INTERVAL_SEC);
        currentStamina = Math.Min(currentStamina + recovered, MAX_STAMINA);
        lastRecoverTimestamp += recovered * RECOVER_INTERVAL_SEC;
    }

    public int GetStaminaCost(int chapterId)
    {
        if (chapterId <= 10) return 6;
        if (chapterId <= 20) return 10;
        return 15;
    }

    public int GetHardModeStaminaCost(int chapterId)
    {
        return Mathf.CeilToInt(GetStaminaCost(chapterId) * 1.5f);
    }

    public bool ConsumeStamina(int amount)
    {
        if (currentStamina < amount) return false;
        currentStamina -= amount;
        return true;
    }

    public bool PurchaseStamina(int purchaseCount)
    {
        if (purchaseCount > 3) return false;
        int[] costs = { 50, 100, 150 };
        int diamondCost = costs[purchaseCount - 1];
        if (!Player.HasDiamond(diamondCost)) return false;
        Player.ConsumeDiamond(diamondCost);
        currentStamina = Math.Min(currentStamina + 60, MAX_STAMINA + 60); // 可超上限
        return true;
    }
}
```

## 8. 阵营评级系统

### 8.1 评级分数计算

```
评级分数 = 累计通关章节数 × 100 + 累计3星数 × 50
```

**示例：**
- 通关15章，其中10章3星 → 分数 = 15×100 + 10×50 = 2000
- 通关30章，全部3星 → 分数 = 30×100 + 30×50 = 4500

### 8.2 评级等级

| 等级 | 所需分数 | 解锁内容 |
|------|---------|---------|
| 1 | 0 | 基础科技树 |
| 2 | 300 | 科技树第2层 |
| 3 | 800 | 科技树第3层 |
| 4 | 1500 | 科技树第4层 |
| 5 | 2500 | 科技树第5层 |
| 6 | 3500 | 困难模式 |
| 7 | 4500 | 全科技解锁 |

### 8.3 评级计算实现

```csharp
public class FactionRankSystem
{
    public static int CalculateFactionScore(PlayerProgress progress)
    {
        int completedChapters = progress.chapter_stars.Count(kv => kv.Value > 0);
        int totalStars = progress.chapter_stars.Values.Sum();
        return completedChapters * 100 + totalStars * 50;
    }

    public static int CalculateFactionRank(int score)
    {
        int[] thresholds = { 0, 300, 800, 1500, 2500, 3500, 4500 };
        int rank = 1;
        for (int i = thresholds.Length - 1; i >= 0; i--)
        {
            if (score >= thresholds[i])
            {
                rank = i + 1;
                break;
            }
        }
        return rank;
    }
}
```

## 11. 边界条件与测试用例

### TC-CP-01: 体力不足时进入关卡

- **前置**: 玩家体力=5，第1章消耗6体力
- **操作**: 点击进入第1章
- **预期**: 提示"体力不足"，弹出购买体力/等待恢复选项

### TC-CP-02: 首次通关获得3星

- **前置**: 第1章未通关，3星条件为占领所有建筑+10回合内
- **操作**: 首次通关，占领所有建筑，8回合完成
- **预期**: 获得3星，奖励：钻石50+30+50=130，碎片×10

### TC-CP-03: 重复通关只获得基础奖励

- **前置**: 第1章已3星
- **操作**: 再次通关
- **预期**: 仅获得经验+金币，不再获得钻石奖励

### TC-CP-04: 3星解锁扫荡

- **前置**: 第1章已3星
- **操作**: 点击扫荡
- **预期**: 扣除6体力，直接获得通关奖励，不进入战斗

### TC-CP-05: 扫荡次数耗尽

- **前置**: 第1章今日已扫荡3次
- **操作**: 点击扫荡
- **预期**: 提示"今日扫荡次数已用完"

### TC-CP-06: 困难模式未解锁

- **前置**: 玩家未通关主线30章
- **操作**: 尝试进入困难模式
- **预期**: 困难模式入口锁定，提示"通关主线后解锁"

### TC-CP-07: 困难模式属性倍率

- **前置**: 第10章普通模式敌方HP=484
- **操作**: 进入第10章困难模式
- **预期**: 敌方HP=484×1.5=726

### TC-CP-08: 体力购买钻石不足

- **前置**: 玩家钻石=30，购买体力需50钻石
- **操作**: 点击购买体力
- **预期**: 提示"钻石不足"，不扣除钻石

### TC-CP-09: 体力购买次数耗尽

- **前置**: 今日已购买3次体力
- **操作**: 点击购买体力
- **预期**: 提示"今日购买次数已用完"

### TC-CP-10: 章节解锁条件未满足

- **前置**: 第5章未通关
- **操作**: 尝试进入第6章
- **预期**: 第6章显示锁定，提示"通关第5章后解锁"

## 13. 服务器校验要点

1. 每次进入关卡需校验：体力是否足够、章节是否解锁、今日次数是否超限
2. 战斗结算需校验：胜利条件是否真实达成、星级条件是否满足
3. 体力恢复由服务器定时推送，客户端不可篡改
4. 扫荡功能需校验3星记录真实性
5. 困难模式解锁需校验主线通关进度
6. 素材关每日次数由服务器重置，不可客户端伪造
7. 阵营评级分数由服务器计算，客户端仅展示
8. 防作弊：战斗结果需服务端验证，不可通过修改客户端跳过战斗
